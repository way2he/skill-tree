# 项目编码规范
## 安全规则
- 禁止在前端代码中使用`eval()`函数
- 所有用户输入必须进行XSS过滤
- SQL查询必须使用参数化查询,禁止字符串拼接

## 代码风格
- 使用TypeScript,所有变量必须有明确的类型定义
- 函数命名采用驼峰式,类命名采用帕斯卡式
- 每个函数必须有JSDoc注释,说明参数和返回值类型

## 架构规范
- 遵循单一职责原则,每个文件不超过500行
- 组件拆分粒度控制在单一功能维度
- 禁止在组件内部直接调用API,必须通过Service层

## 错误处理
- 所有异步操作必须有try-catch错误处理
- 错误信息必须包含上下文信息,禁止使用模糊的错误提示
