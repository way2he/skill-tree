## ❓ 问题1：什么是 CoT（思维链）？

### 📝 详细解答

**CoT = Chain-of-Thought = 思维链**

简单说：**让大模型把思考过程一步步说出来，而不是直接给出答案**。

#### 为什么 CoT 这么有效？

| 传统方式（直接给答案） | CoT 方式（分步思考） |
|------------------------|---------------------|
| 大模型凭「直觉」回答 | 大模型进行「理性」推理 |
| 一步错，步步错 | 每一步都可以验证 |
| 复杂问题正确率低 | 复杂问题正确率提升 50%+ |

#### CoT 的 3 个层级

| 层级 | 名称 | 做法 | 适用场景 |
|------|------|------|---------|
| L1 | 零样本 CoT | 只加一句话："Let's think step by step" | 简单问题 |
| L2 | 少样本 CoT | 给 1-2 个完整的「思考过程 + 答案」示例 | 中等问题 |
| L3 | 自一致性 | 生成 5 个不同的推理路径，投票选出现最多的答案 | 复杂问题 |

#### 生活中的类比

就像做数学题：
- ❌ 坏学生：直接写答案 = 大模型直接输出
- ✅ 好学生：先写已知条件，再写解题步骤，最后写答案 = CoT

## ❓ 问题2：什么是大模型的「涌现能力」？

### 📝 详细解答

**涌现（Emergence）= 量变到质变**

#### 定义
当模型参数规模、训练数据量达到某个**阈值**之后，模型突然拥有了一些之前没有的能力，而且这些能力不是线性增长的，是**突然出现**的。

#### 经典的涌现能力

| 能力 | 涌现阈值 | 说明 |
|------|---------|------|
| **思维链推理** | ~10B 参数 | 参数不够时，加 CoT 反而更差；超过阈值后，CoT 效果显著提升 |
| **上下文学习** | ~10B 参数 | Few-shot 才开始真正有效 |
| **数学推理** | ~60B 参数 | 能做复杂的加减乘除、应用题 |
| **指令遵循** | ~7B-10B 参数 | 能听懂人的指令并执行 |

#### 非常关键的一张图（记住！）

```
模型能力
  ↑
  │       ╭───── ← 涌现阈值
  │      ╱
  │     ╱
  │    ╱
  │   ╱
  │  ╱
  │╱
  └───────────────────→ 参数规模
       小模型          大模型
```

在阈值之前：能力线性缓慢增长
在阈值之后：能力突然爆发式增长

#### 为什么会有涌现？（3 个主流解释）

1. **任务的本质是多步推理**
   - 很多任务看似简单，实际上需要多步推理
   - 小模型只能做单步推理，大模型才能做多步
   - CoT 正好把多步过程显式化了

2. **数据分布的长尾效应**
   - 简单的模式在少量数据中就能学到
   - 复杂的模式藏在数据的长尾里
   - 只有看到足够多的数据，才能学到这些复杂模式

3. **Grokking（顿悟现象）**
   - 训练前期：模型好像什么都没学会，正确率一直很低
   - 训练到某个点：突然顿悟，正确率飙升到 99%+
   - 就像人学某个东西，突然某一天开窍了一样

#### 面试高频问法

> Q：为什么 7B 模型做 CoT 效果不好，13B 就突然好了很多？
>
> A：因为 10B 参数左右就是思维链推理的涌现阈值！

## ❓ 问题3：Ollama 调用参数详解

### 📝 详细解答

Ollama 的 API 参数和 OpenAI 基本对齐，下面逐一详解：

#### 完整参数列表

```python
payload: dict[str, Any] = {
    "model": self.model,           # 模型名称
    "prompt": full_prompt,         # 用户输入的提示词
    "system": self.system_prompt,  # 系统提示词
    "temperature": 0.3,            # 温度参数
    "format": "json",              # 输出格式
    "stream": False,               # 是否流式输出
    "max_tokens": 2048,            # 最大输出令牌数
    "top_p": 0.9,                  # 核采样参数
    "presence_penalty": 0.0,       # 存在惩罚
    "frequency_penalty": 0.0,      # 频率惩罚
    "num_ctx": 4096,               # ✅ 上下文窗口大小！
    "num_predict": 2048,           # 预测的最大 token 数
    "stop": ["\n\n", "用户:"],      # 停止词
    "seed": 42,                    # 随机种子，可复现
}
```

---

#### ⭐ 重点：能设置上下文吗？能！

**答案：可以，用 `num_ctx` 参数！**

```python
payload = {
    "model": "qwen2:7b",
    "prompt": "...",
    "num_ctx": 8192,  # ✅ 设置上下文窗口大小为 8k
}
```

##### 关于 `num_ctx` 的注意事项：

| 问题 | 答案 |
|------|------|
| 默认值是多少？ | 4096（4k） |
| 最大能设多大？ | 看模型的上下文窗口，比如 qwen2 是 128k |
| 设越大越好吗？ | 不是！越大越占显存，推理越慢。够用就行 |
| 设太小会怎样？ | 超长输入会被截断，丢失信息 |

##### 常见的上下文大小选择：

| 场景 | 推荐 num_ctx |
|------|-------------|
| 简单问答、闲聊 | 2048 - 4096 |
| 文档总结、长对话 | 8192 - 16384 |
| 整本书、超长文档 | 32768 - 128k |

---

#### 其他重要参数详解

##### 1. `temperature` 温度
- 范围：0.0 - 2.0
- **0.0 = 最确定**：每次输出几乎一样，适合需要精确答案的场景（代码生成、事实问答）
- **1.0 = 平衡**：默认值，创造力和确定性平衡
- **>1.0 = 更发散**：适合创意写作、头脑风暴，但容易胡说八道

##### 2. `top_p` 核采样
- 范围：0.0 - 1.0
- 0.1 = 只从前 10% 概率的词里选，很保守
- 0.9 = 从前 90% 概率的词里选，更多样
- **最佳实践**：`temperature` 和 `top_p` 只改一个，不要同时改两个

##### 3. `format: "json"`
- 强制模型输出 JSON 格式
- **重要**：即使加了这个，也还是要用 Pydantic 校验！模型还是可能输出非法 JSON
- 建议配合系统提示词一起用："你只能输出合法的 JSON 格式，不要输出任何其他内容"

##### 4. `presence_penalty` vs `frequency_penalty`
两个都是惩罚重复，但侧重点不同：

| 参数 | 作用 | 常用值 |
|------|------|-------|
| `presence_penalty` | **是否出现过** → 只要出现过一次就惩罚 | -2.0 到 +2.0 |
| `frequency_penalty` | **出现频率** → 出现次数越多，惩罚越重 | -2.0 到 +2.0 |

**使用场景**：
- 写小说，不想重复用词 → 两个都设 0.5-1.0
- 摘要生成，不想老是重复"本文主要介绍了..." → 设 0.3
- 代码生成，需要精确 → 设 0

##### 5. `seed` 随机种子
- 设一个固定的数字（比如 42），相同的输入会得到相同的输出
- 用于可复现、测试场景
- **注意**：只是尽量接近，不是 100% 完全相同（模型内部有一定的随机性）

## ❓ 问题4：什么是 A/B 测试？

### 📝 详细解答

#### 定义
A/B 测试 = **对照实验**
- 把用户随机分成两组
- A 组用旧方案（对照组）
- B 组用新方案（实验组）
- 对比两组的数据，看新方案是不是真的更好

#### 为什么需要 A/B 测试？

因为：**你觉得好 ≠ 真的好**

例子：
- 你觉得新提示词写得很完美 → 实际效果反而更差
- 你觉得这个颜色更好看 → 实际转化率下降 30%

**人的直觉非常不可靠，数据才是客观的！**

---

#### 在大模型/Agent 开发中的典型 A/B 测试场景

| 测试什么 | A 组（旧） | B 组（新） | 看什么指标 |
|---------|-----------|-----------|----------|
| 提示词 | 旧提示词 | 新优化的提示词 | 正确率、格式符合率 |
| 模型 | GPT-3.5 | Qwen2-7B | 正确率、速度、成本 |
| 温度 | 0.7 | 0.3 | 答案正确率、重复率 |
| CoT 策略 | 无 CoT | 有 CoT | 复杂问题正确率 |

---

#### A/B 测试的 4 个关键步骤

```
第1步：明确目标和指标
  ↓
第2步：随机分组，保证公平
  ↓
第3步：运行实验，收集数据
  ↓
第4步：统计检验，判断是否真的更好
```

##### 第1步：明确可量化的指标
❌ 坏指标："效果更好"、"用户体验提升"（无法量化）
✅ 好指标：
- 输出 JSON 格式合法率从 85% 提升到 95%
- 数学题正确率从 60% 提升到 80%
- 平均响应时间从 3s 降低到 1.5s
- 每次调用成本从 0.01 元降低到 0.002 元

##### 第2步：随机分组的重要性
- ✅ 必须真正随机，不能按地区、年龄、用户等级分
- ✅ 两组样本量要足够大，不能 A 组 10 个，B 组 10 个就下结论
- ✅ 同时运行，不能 A 组今天测，B 组明天测

##### 第3步：运行实验
- 流量切分：比如 50% 用户走 A，50% 走 B
- 时间：至少跑 3-7 天，避免偶然因素
- 不要中途改实验条件

##### 第4步：统计显著性检验
不是 A 组正确率 60%，B 组 65%，就说 B 更好！
需要做**统计检验**（如 t 检验、卡方检验），看这个差异是真的有效果，还是只是随机波动。

**判断标准：p 值 < 0.05 = 统计显著，差异是真实的**

---

#### 常见的 A/B 测试陷阱

| 陷阱 | 说明 |
|------|------|
| **偷看实验** | 实验才跑了 1 天就看结果，看到有差异就停止 → 很容易误判 |
| **多重比较** | 同时测 10 个指标，总有一个看起来有差异 → 需要做校正 |
| **样本量不够** | 每组只有几十个样本，结果不可信 |
| **辛普森悖论** | 整体看 A 好，拆分成子群体看 B 好 → 要分层分析 |

## ❓ 问题5：什么是梯度消失？

### 📝 详细解答

#### 什么是梯度？
梯度 = **导数 = 变化的方向和大小**

训练神经网络时：
> 计算损失函数对每个参数的梯度 → 沿着梯度反方向更新参数 → 一步步降低损失

梯度 = "该怎么改参数，才能让损失变小"

---

#### 什么是梯度消失？

**梯度消失 = 深层网络中，靠近输入层的梯度变得无限小，几乎等于 0**

结果：
- 靠近输出的层还能正常更新
- 靠近输入的层几乎不动，学不到东西
- 深层网络训练不动，效果还不如浅层网络

```
输入层 → 隐藏层1 → 隐藏层2 → ... → 隐藏层10 → 输出层
  ↓          ↓          ↓                 ↓
梯度=0.0001  梯度=0.001  梯度=0.01       梯度=0.1
  ↓
学不动！
```

---

#### 为什么会发生梯度消失？（数学原理）

因为反向传播时，梯度是**链式相乘**的：

```
梯度(第1层) = 梯度(第2层) × W2 × sigmoid'(x2)
            = 梯度(第3层) × W3 × sigmoid'(x3) × W2 × sigmoid'(x2)
            = ...
            = 一大堆小于 1 的数相乘
            = 越乘越小，最后趋近于 0
```

##### 两个主要原因：

| 原因 | 说明 |
|------|------|
| **激活函数饱和** | Sigmoid 导数最大值是 0.25，大部分区域导数接近 0，经过多层相乘，指数级衰减 |
| **权重初始化不当** | 权重初始化太小，W < 1，越乘越小 |

---

#### Transformer 中为什么梯度消失不明显？

因为 Transformer 有几个关键设计，缓解了梯度消失：

##### 1. **残差连接（Residual Connection）

```
x → 注意力 → 加 x → 前馈网络 → 加 x → 输出
    ↓
梯度可以直接通过残差连接传回，不会因为多层就消失！
```

残差连接 = 梯度的「高速公路」，梯度可以直接从输出层传回到输入层

##### 2. **LayerNorm**
- 把每一层的输出都归一化到均值 0，方差 1
- 避免数值越来越大或越来越小，梯度保持稳定

##### 3. **更好的初始化方法**
- He 初始化、Xavier 初始化
- 让每一层的输出和梯度的方差保持在合适的范围

##### 4. **Attention 机制本身**
- Attention 是乘法交互，梯度可以直接通过注意力权重传递
- 不像 RNN 那样需要一步步反向传播

---

#### 梯度消失 vs 梯度爆炸

| 现象 | 原因 | 表现 |
|------|------|------|
| **梯度消失** | 很多小于1的数相乘，越来越小 | 深层学不动，loss 不下降 |
| **梯度爆炸** | 很多大于1的数相乘，越来越大 | loss 变成 NaN，参数飞掉 |

**解决方法对比**：
- 梯度消失：残差连接、换 ReLU 激活、LayerNorm、调整初始化
- 梯度爆炸：梯度裁剪（Gradient Clipping）、学习率不要太大
