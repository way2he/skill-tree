### 第1题：思维链 CoT 原理（8分）

#### ✅ 标准答案

思维链（Chain-of-Thought）是让大模型 **把思考过程一步步说出来，而不是直接给出答案的提示词技术。

**核心3个得分点：
1. **问题分解**：复杂问题需要多步推理，CoT强迫模型把大问题拆成多个小问题，每个小问题只解决一步
2. **中间验证**：每一步思考都可以验证对错，避免一步错步步错
3. **涌现能力**：CoT是大模型的涌现能力之一，**参数量越大，CoT效果越明显**

#### 🧠 背诵口诀
> **一说二拆三涌现，CoT就是分步算**

#### 💯 打分对照
- 答出「分步思考」→ 4分
- 答出「问题分解+中间验证」→ 6分
- 答出「涌现能力，参数量越大效果越好」→ 8分（满分）

### 第2题：Few-shot vs Zero-shot（8分）

#### ✅ 标准答案

三者核心区别：**给模型多少个示例来学习新任务**。

| 类型 | 示例数 | 定义 | 应用场景 |
|------|--------|------|----------|
| **Zero-shot** | 0个 | 不给示例，只给指令，直接完成任务 | 通用问答、简单分类 |
| **One-shot** | 1个 | 只给1个示例，学习完成任务 | 特定格式输出 |
| **Few-shot** | 3-5个 | 给3-5个示例，学习完成任务 | 复杂格式、精确分类 |

#### 🧠 背诵口诀
> **零看办，一照来，五照搬**

#### 💯 打分对照
- 答出三者定义区别 → 6分
- 各举一个合理场景 → +2分 = 8分（满分）

### 第3题：提示注入攻击（8分）

#### ✅ 标准答案

**什么是提示注入？**
攻击者通过构造特殊输入，**让大模型忽略原始的系统提示，转而执行攻击者的恶意指令。

**3种常见攻击手法 + 防护方法**：

| 攻击手法 | 防护方法 |
|----------|---------|
| **1. 指令覆盖** |  | 输入过滤，检测关键词；系统提示加固 |
| **2. 角色扮演绕过** | 角色白名单，禁止危险角色；输出审核 |
| **3. 编码绕过** | 输入解码后再检测，多轮校验 |

#### 🧠 背诵口诀
> **一覆盖二扮演三编码，过滤加固加审核**

#### 💯 打分对照
- 答出提示注入定义 → 2分
- 答出3种攻击手法 → 3分
- 答出对应3种防护 → 3分
- 合计 8分（满分）

### 第4题：注意力机制本质（8分）

#### ✅ 标准答案

**注意力机制的本质（一句话必背）**：
> **动态加权平均**——根据输入内容的重要性，动态给不同部分分配权重，加权求和得到输出。

**Q、K、V分别是什么？
- **Q** Query 查询向量 → 我要找什么
- **K** Key 键向量 → 我有什么
- **V** Value 值向量 → 实际的内容

**核心公式（写出来加分）**：
```
Attention(Q, K, V) = softmax(QK^T / √d_k) * V
```

#### 🧠 背诵口诀
> **动加权，Q找K，V是内容**

#### 💯 打分对照
- 本质说对「动态加权平均」→ 3分
- Q/K/V各说对含义 → 各1分，共3分
- 解释清楚关系/写出公式 → +2分 = 8分（满分）

### 第5题：多头注意力（8分）

#### ✅ 标准答案

**单头局限性**：一次只能**关注一种语义关系**，就像人只能从一个角度看问题，有盲区。

**为什么多头有效（3个核心）**：
1. **多视角关注**：每个头学习不同注意力模式（语法、语义、指代等）
2. **并行捕捉**：多个头并行计算，同时捕捉多种关系，效率更高
3. **表达能力更强**：多个头结果拼接，表达能力指数级提升

**关键数字（记住加分）**：Transformer base = 8头，每头64维，总512维

#### 🧠 背诵口诀
> **单头偏，多头全，八个法官一起判**

#### 💯 打分对照
- 答出单头局限性（只能关注一种关系）→ 3分
- 答出多头2个优势 → 3分
- 类比生动/说出具体数字 → +2分 = 8分（满分）

### 第6题：JSON 格式强制输出（20分）

#### ✅ 标准答案

**第一部分：系统提示词（8分）**

```
你是一个专业的代码审查专家。

【输出要求 - 必须严格遵守！】
1. 只输出纯 JSON 字符串，不要输出任何其他内容
2. 不要输出 markdown 代码块标记（```json 或 ```）
3. 不要输出任何解释性文字、说明、前缀后缀
4. 必须严格符合以下 JSON Schema：

{
  "type": "object",
  "properties": {
    "review_result": {"type": "string", "enum": ["approve", "reject"]},
    "confidence": {"type": "number", "minimum": 0, "maximum": 1},
    "comments": {"type": "string"},
    "suggestions": {"type": "array", "items": {"type": "string"}}
  },
  "required": ["review_result", "confidence", "comments", "suggestions"]
}

【示例输出】
{"review_result": "approve", "confidence": 0.95, "comments": "代码质量良好", "suggestions": ["添加单元测试"]}

重要提醒：如果输出了 JSON 以外的任何内容，你将被扣分！
```

**第二部分：Pydantic校验函数（8分）**

```python
from pydantic import BaseModel, Field, ValidationError
from typing import List

class CodeReview(BaseModel):
    review_result: str = Field(..., pattern=r"^(approve|reject)$")
    confidence: float = Field(..., ge=0.0, le=1.0)
    comments: str = Field(..., min_length=5)
    suggestions: List[str]

def validate_json(json_str: str):
    try:
        json_str = json_str.strip()
        if json_str.startswith("```json"):
            json_str = json_str[7:]
        if json_str.startswith("```"):
            json_str = json_str[3:]
        if json_str.endswith("```"):
            json_str = json_str[:-3]
        return CodeReview.model_validate_json(json_str.strip())
    except ValidationError as e:
        print(f"校验失败: {e}")
        return None
```

**第三部分：自动重试机制（4分）**

```python
import asyncio

async def generate_with_retry(generate_func, max_retries=3, backoff=1.0):
    for attempt in range(max_retries):
        result = validate_json(await generate_func())
        if result:
            return result
        if attempt < max_retries - 1:
            await asyncio.sleep(backoff * (2 ** attempt))
    return None
```

#### 🧠 背诵口诀
> **一角色二禁止三Schema，四示例五后果，Pydantic加重试**

#### 💯 打分对照
- 系统提示词完整（5要素）→ 8分
- Pydantic模型+校验正确 → 8分
- 重试机制正确 → 4分
- 合计 20分（满分）

### 第7题：CoT 提示词优化（20分）

#### ✅ 标准答案

**优化后的完整提示词**：

```
你是一个经验丰富的小学数学老师，擅长用清晰的步骤讲解数学问题。
你的讲解风格是：耐心、细致、逻辑清晰，让每个学生都能听懂。

【答题要求 - 必须严格遵守！】
请按照以下 6 步格式回答数学问题，一步都不能少：
1. 【理解题目】：用自己的话复述题目，明确已知条件和要求解的问题
2. 【设立变量】：定义未知数，说明每个变量代表什么
3. 【列方程】：根据题意列出方程式
4. 【解方程】：一步步展示计算过程，每一步都要写清楚
5. 【验证答案】：把答案代回原题，验证是否正确
6. 【最终答案】：用一句话给出最终答案

【示例 - 正确的答题格式】
问题：小明有5个苹果，小红的苹果是小明的3倍，两人共有多少个苹果？

【理解题目】：已知小明有5个苹果，小红的苹果数量是小明的3倍，求两人一共有多少个苹果
【设立变量】：设两人一共有 x 个苹果
【列方程】：x = 5 + 5 × 3
【解方程】：x = 5 + 15 = 20
【验证答案】：20 - 5 = 15，15 ÷ 5 = 3，符合题意，正确
【最终答案】：两人一共有 20 个苹果

现在请解答这个问题：
商店里有苹果和香蕉，苹果的数量是香蕉的2倍，总共有15个水果，问苹果有几个？
```

**优化的4个核心要点（各5分）**：
1. 角色设定：明确的身份 + 风格说明 → 5分
2. 思维链CoT：明确要求分步思考 → 5分
3. 结构化输出：统一的输出格式 → 5分
4. Few-shot示例：完整的示例展示 → 5分

#### 🧠 背诵口诀
> **角色加CoT，结构加示例**

#### 💯 打分对照
- 4个要点每个5分，共20分（满分）

### 第8题：Agent 安全防护系统设计（20分）

#### ✅ 标准答案

**整体设计理念：四层防护，纵深防御**

```
┌─────────────────────────────────────────────────┐
│ 输入层 → 推理层 → 输出层 → 工具调用层 │
│ 第一道闸   第二道闸   第三道闸   最后一道闸 │
└─────────────────────────────────────────────────┘
```

**第一层：输入层防护（5分）**
1. 注入检测：关键词匹配 + 语义检测
2. 输入净化：特殊字符过滤 + 编码检测
3. 风险分级：低/中/高风险分级处理

**第二层：推理层防护（5分）**
1. 系统提示加固：前后夹击 + 特权分隔
2. 思维链监控：发现异常思考立即中断
3. 角色白名单：只允许扮演预设角色

**第三层：输出层防护（5分）**
1. 敏感信息脱敏：手机号、身份证、密钥等
2. 内容审核：违规 + 风险内容检测
3. 格式校验：不符合预期格式就重试

**第四层：工具调用层防护（5分）**
1. 权限最小化：每个工具只给必要权限
2. 参数白名单：危险参数直接拦截
3. 二次确认：高风险操作必须用户确认
4. 审计日志：每次调用完整记录，不可删除

**加分项（说出来直接满分）**：红蓝对抗、零信任原则、灰度发布

#### 🧠 背诵口诀
> **入推工出，四层防护**

#### 💯 打分对照
- 四层防护每说清楚一层得5分
- 每层能说出2个以上具体措施得满分
- 说出红蓝对抗/零信任等加分项 → 直接20分满分！
