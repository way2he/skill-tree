## 一、代码生成的完整流程

### 完整流程图

```
用户需求
  ↓
需求理解与澄清（可选）
  ↓
方案设计
  ↓
代码生成
  ↓
代码审查（可选）
  ↓
输出可执行代码
```

### 每个步骤的说明

| 步骤 | 作用 | 为什么需要 |
|------|------|-----------|
| **需求理解** | 确保理解用户真实意图 | 需求模糊会导致代码不符合预期 |
| **方案设计** | 先想清楚怎么做，再写代码 | 直接写代码容易走弯路 |
| **代码生成** | LLM生成实际代码 | 核心环节 |
| **代码审查** | 检查代码质量和安全性 | 防止bug和安全问题 |

## 二、代码生成的Prompt设计

### 1. 基础Prompt模板

```python
prompt = """你是一个专业的Python开发工程师。请根据以下需求生成高质量可执行代码。

需求：
{user_requirement}

要求：
1. 代码必须完整可执行
2. 包含必要的注释
3. 遵循PEP 8规范
4. 处理异常情况
5. 包含使用示例

请直接输出代码，不要其他解释。"""
```

### 2. 进阶Prompt模板（带思考）

```python
prompt = """你是一个专业的Python开发工程师。请根据以下需求生成高质量可执行代码。

需求：
{user_requirement}

请按以下步骤思考并输出：

【思考过程】
1. 理解需求：这个需求要解决什么问题？
2. 方案设计：用什么技术方案？整体架构是什么？
3. 关键实现：有哪些难点？如何解决？
4. 异常处理：可能出现什么错误？如何处理？

【代码实现】
(在此处写完整可执行代码)

【使用示例】
(在此处写使用示例)

【注意事项】
(在此处写注意事项)
"""
```

### 3. 带示例的Few-shot Prompt

```python
prompt = """你是一个专业的Python开发工程师。请根据以下需求生成高质量可执行代码。

示例1：
需求：写一个函数，输入两个数，返回它们的和
代码：
```python
def add(a, b):
    \"\"\"计算两个数的和\"\"\"
    return a + b

# 使用示例
print(add(1, 2))  # 输出: 3
```

示例2：
需求：写一个函数，输入一个列表，返回列表中的最大值
代码：
```python
def find_max(numbers):
    \"\"\"找到列表中的最大值\"\"\"
    if not numbers:
        raise ValueError("列表不能为空")
    return max(numbers)

# 使用示例
print(find_max([1, 5, 3, 9, 2]))  # 输出: 9
```

现在请处理以下需求：
需求：{user_requirement}

请直接输出代码，格式同上。"""
```

## 三、提高代码生成质量的技巧

### 1. 明确需求

**❌ 不好的需求：**
```
写一个数据分析程序
```

**✅ 好的需求：**
```
写一个Python程序，功能如下：
1. 读取data.csv文件（包含id, name, age, score四列）
2. 计算平均分、最高分、最低分
3. 按分数从高到低排序
4. 输出到result.csv文件
5. 包含异常处理（文件不存在、格式错误等）
```

### 2. 指定技术栈

**示例：**
```
请使用以下技术栈：
- Web框架：FastAPI
- 数据库：SQLAlchemy + SQLite
- 数据验证：Pydantic
- 不要用Django
```

### 3. 给出输入输出示例

**示例：**
```
输入：
[1, 2, 3, 4, 5]

输出：
{
    "sum": 15,
    "mean": 3.0,
    "max": 5,
    "min": 1
}
```

### 4. 分步骤生成复杂代码

对于复杂需求，不要指望一次生成完整代码：

```
步骤1：先生成核心数据结构
步骤2：再生成核心算法
步骤3：然后生成API接口
步骤4：最后整合测试
```

## 四、代码生成的最佳实践

### 1. 要求生成测试

```
请在代码中包含单元测试（使用pytest）
```

### 2. 要求处理边界情况

```
请考虑以下边界情况：
- 空输入
- 超大输入
- 异常输入
```

### 3. 要求生成文档

```
请为每个函数添加docstring，包含：
- 功能说明
- 参数说明
- 返回值说明
- 异常说明
- 使用示例
```

### 4. 安全性考虑

```
请确保代码安全性：
- 不要执行任意代码
- 验证所有输入
- 不要硬编码敏感信息
```

## 五、常见问题与解决方案

### 问题1：生成的代码不完整

**解决方案：**
```
请继续完成上面的代码，确保完整可执行。
```

### 问题2：生成的代码有bug

**解决方案：**
```
这个代码运行时报错：{error_message}
请修复这个问题。
```

### 问题3：生成的代码不符合预期

**解决方案：**
```
这个代码没有满足我的需求，我想要的是：{clarification}
请重新生成。
```
