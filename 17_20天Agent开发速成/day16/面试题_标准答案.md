## 第 1 题答案

### 问题
请详细描述代码解释器的完整流程，以及每个环节的作用是什么？

### 标准答案

**完整流程图：**

```
用户需求
    ↓
代码生成（LLM）
    ↓
沙箱执行（安全隔离）
    ↓
结果检查
    ↓
成功？ ──是──→ 返回结果
    ↓否
报错分析
    ↓
自动修正
    ↓
次数超限？ ──是──→ 放弃/返回最终版本
    ↓否
重新执行（回到沙箱执行）
```

**每个环节的作用：**

| 环节 | 作用 | 为什么需要 |
|------|------|-----------|
| **代码生成** | 用LLM把自然语言需求转成代码 | 自动化生成，节省时间 |
| **沙箱执行** | 在安全隔离环境中运行代码 | 防止恶意代码破坏系统 |
| **结果检查** | 判断执行是否成功 | 确定是否需要修正 |
| **报错分析** | 理解错误原因和位置 | 为修复提供依据 |
| **自动修正** | 根据错误信息修改代码 | 自动化修复，提高成功率 |

## 第 2 题答案

### 问题
执行沙箱有哪些核心安全措施？为什么需要这些措施？

### 标准答案

**核心安全措施：**

| 措施 | 作用 | 为什么需要 |
|------|------|-----------|
| **Docker隔离** | 容器化运行，与宿主隔离 | 防止代码访问宿主资源 |
| **资源限制** | 限制CPU/内存/磁盘/进程数 | 防止资源耗尽攻击 |
| **网络禁用** | 禁止网络访问 | 防止数据泄露、恶意访问 |
| **只读文件系统** | 文件系统只读，用tmpfs | 防止篡改系统文件 |
| **最小权限** | 以最低权限运行 | 减少攻击面 |
| **代码预检查** | 执行前先静态分析 | 阻止明显的危险代码 |

**示例：**
```python
# 没有沙箱的危险代码
import os
os.system("rm -rf /")  # 删除整个系统！

# 有沙箱：Docker隔离、网络禁用、只读文件系统
# 上面的代码无法造成实质性破坏
```

## 第 3 题答案

### 问题
Python常见的错误类型有哪些？分别举一个例子，并说明如何分析和修复？

### 标准答案

**常见错误类型：**

| 类型 | 示例 | 分析和修复 |
|------|------|-----------|
| **SyntaxError** | `if True print()` | 看行号，检查语法（冒号、括号、缩进） |
| **NameError** | `print(unknown)` | 检查变量名拼写、作用域、是否导入 |
| **TypeError** | `1 + "2"` | 检查类型，做类型转换 |
| **IndexError** | `[][0]` | 检查索引范围、列表是否为空 |
| **KeyError** | `{}['a']` | 检查字典键，用get()方法 |
| **AttributeError** | `None.foo()` | 检查对象类型、属性名拼写 |

## 第 4 题答案

### 问题
自动修正的反思回路如何设计？如何提高修复成功率？

### 标准答案

**反思回路设计：**

```
执行代码
    ↓
捕获错误信息
    ↓
分析错误原因
    ↓
反思：为什么会错？怎么修？
    ↓
修正代码
    ↓
重新执行
    ↓
成功？ ──是──→ 完成
    ↓否
次数超限？ ──是──→ 放弃
    ↓否
回到分析错误原因
```

**提高成功率的策略：**

1. **渐进式修复**：不要试图一次全修好，分步来
2. **利用历史经验**：记录成功和失败案例
3. **测试驱动**：先写测试，再修代码
4. **限制修正范围**：每次只改一个问题
5. **提供更多上下文**：给LLM更多信息

## 第 5 题答案

### 问题
AST（抽象语法树）是什么？在代码理解中有什么应用场景？

### 标准答案

**AST定义：**
AST = Abstract Syntax Tree，源代码的抽象语法结构的树状表示。

**示例：**
代码 `x = 1 + 2` 的AST：
```
Assign
├─ Name(x)
└─ BinOp(Add)
   ├─ Num(1)
   └─ Num(2)
```

**应用场景：**

| 场景 | 说明 |
|------|------|
| **代码检查** | lint工具，检查代码规范、安全问题 |
| **代码转换** | 自动重构，代码格式化 |
| **代码统计** | 统计函数数、类数、复杂度等 |
| **代码生成** | 模版引擎，从AST生成代码 |

**代码示例：**
```python
import ast

# 解析代码
code = "x = 1 + 2"
tree = ast.parse(code)
print(ast.dump(tree, indent=2))
```

## 第 6 题答案

### 问题
代码生成的Prompt如何设计才能提高成功率？

### 标准答案

**Prompt设计原则：**

1. **明确需求**：不要模糊，要具体
2. **给出示例**：Few-shot learning
3. **要求思考**：先想再写（Chain-of-Thought）
4. **指定规范**：PEP8、注释、异常处理
5. **提供输入输出**：给示例数据

**好的Prompt模板：**
```
你是一个专业的Python开发工程师。请根据以下需求生成高质量可执行代码。

需求：
{user_requirement}

要求：
1. 代码必须完整可执行
2. 包含必要的注释
3. 遵循PEP 8规范
4. 处理异常情况
5. 包含使用示例

请先思考实现方案，然后输出代码。
```

**对比：**
- ❌ 不好："写一个数据处理程序"
- ✅ 好："写一个Python程序读取data.csv（name,age,score列），计算平均分，输出到result.csv，包含异常处理"
