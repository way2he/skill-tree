## 面试高频问题（今天学的）

### 必背 3 题

1. **代码解释器的完整流程是什么？**
   - 用户需求 → 代码生成 → 沙箱执行 → 结果检查
   - 如果失败：报错分析 → 自动修正 → 重新执行
   - 关键：安全沙箱、迭代修复

2. **执行沙箱的安全措施有哪些？**
   - Docker隔离
   - 资源限制（CPU/内存/磁盘/进程数）
   - 网络禁用
   - 只读文件系统 + tmpfs
   - 最小权限
   - 代码预检查

3. **自动修正的反思回路如何设计？**
   - 执行 → 分析错误 → 反思原因 → 修正代码 → 重新执行
   - 最大尝试次数限制
   - 渐进式修复、经验学习

### 进阶 3 题

4. AST在代码理解中有什么作用？
5. 如何提高代码生成的质量？
6. 沙箱的性能和安全如何平衡？
