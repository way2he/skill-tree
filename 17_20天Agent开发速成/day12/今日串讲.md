## 核心知识点回顾

### 1. 4 种架构模式

```
顺序模式：A → B → C（线性流程）
并行模式：A、B、C 同时（独立任务）
层次模式：管理者 + 工作者（复杂协调）
混合模式：组合多种模式（最灵活）
```

### 2. CrewAI 四件套

| 概念 | 一句话 |
|------|--------|
| **Agent** | "我是谁，我要做什么"（角色+目标+背景） |
| **Task** | "具体做什么，输出什么"（描述+预期+执行者） |
| **Crew** | "团队成员有哪些，任务有哪些" |
| **Process** | "团队怎么协作"（顺序/分层/异步） |

### 3. 好 Agent 的三要素

```
✅ Role（清晰的角色）："资深产品经理"（不是"助手"）
✅ Goal（明确的目标）："输出PRD"（不是"帮忙"）
✅ Backstory（丰富的背景）：经历+专长+风格+信念
```

### 4. 任务依赖设计

```
顺序依赖：context=[task_a]
并行汇总：context=[task_a, task_b, task_c]
混合依赖：复杂的 context 组合
```

## 常见问题解答

### Q1: 什么时候用单 Agent，什么时候用多 Agent？

**判断标准**：
- 任务简单、单一技能 → 单 Agent
- 任务复杂、需要多种技能 → 多 Agent
- 想并行提速 → 多 Agent
- 想模拟真实团队协作 → 多 Agent

### Q2: CrewAI 和 LangChain 是什么关系？

**答案**：
- **LangChain**：大模型应用开发框架（更底层）
- **CrewAI**：多 Agent 协作框架（更专注）
- 可以一起用：CrewAI 底层可以用 LangChain 的组件

### Q3: 多 Agent 成本会不会很高？

**答案**：
- 确实会高一些（调用次数多）
- 但质量和效率也会提升
- 可以优化：
  - 简单任务用 GPT-3.5，复杂用 GPT-4
  - 合理设计任务，避免重复
  - 用缓存（CrewAI 有 cache 参数）
