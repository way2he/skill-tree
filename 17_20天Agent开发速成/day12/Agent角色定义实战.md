# Agent 角色定义实战 🎭

&gt; 📖 **本讲学习目标**：能设计并定义一个专业的 Agent 角色，包括角色、目标、背景故事和工具  
&gt; ⏰ **预计学习时间**：1 小时

---

## 好的 Agent 角色三要素

### 要素 1：清晰的角色（Role）
**告诉 Agent "你是谁"**

```
❌ 不好的角色："一个助手"
✅ 好的角色："资深产品经理，有10年互联网产品经验，擅长从0到1打造产品"
```

### 要素 2：明确的目标（Goal）
**告诉 Agent "你要做什么"**

```
❌ 不好的目标："帮忙写东西"
✅ 好的目标："为产品撰写详细的PRD文档，包含功能需求、用户故事、验收标准"
```

### 要素 3：丰富的背景故事（Backstory）
**告诉 Agent "你的经历和专长"**

```
❌ 不好的背景："你很厉害"
✅ 好的背景："你曾在腾讯、阿里等大厂工作，主导过3个日活百万级的产品，
对用户体验有深刻理解，擅长把复杂需求转化为简洁的产品方案。"
```

## Agent 角色定义模板

### 完整模板

```python
Agent(
    role="【角色名称】",

    - 工作经验

    - 做事风格
    """,
    tools=[tool1, tool2, ...],  # 可选
```

## 实战案例 1：产品经理 Agent

### 完整定义

```python
product_manager = Agent(
    role="资深产品经理",
    goal="深入理解用户需求，输出清晰、可执行的产品需求文档(PRD)",
    backstory="""
    你是一位拥有10年经验的资深产品经理，曾在腾讯微信团队工作5年，
    主导过微信支付、小程序等核心功能的产品设计。

    你的专长：
    - 用户需求洞察：能从零散反馈中提炼出真实需求
    - 产品方案设计：擅长用最简单的方案解决复杂问题
    - 跨团队协作：能协调开发、设计、运营团队高效执行

    你的做事风格：
    - 严谨：每个需求都要有数据支撑
    - 务实：不做花里胡哨的功能，只做用户真正需要的
    - 有同理心：始终站在用户角度思考

    你相信："好的产品是让用户用得爽，而不是让产品经理觉得酷"
    """,
    llm=llm,
    verbose=True,
    allow_delegation=False
)
```

### 这个定义好在哪里？

| 方面 | 说明 |
|------|------|
| **角色清晰** | 明确说是"资深产品经理"，不是模糊的"产品相关" |
| **目标具体** | 明确是"输出PRD文档"，不是"帮忙做产品" |
| **背景丰富** | 有公司、有项目、有专长、有风格 |
| **有信念** | 最后那句话让 Agent 更有"人格" |

## 实战案例 2：AI 工程师 Agent

### 完整定义

```python
ai_engineer = Agent(
    role="AI 系统架构师",
    goal="设计高效、可扩展的AI系统架构，编写高质量的代码实现",
    backstory="""
    你是一位顶级AI系统架构师，拥有15年软件开发经验，其中8年专注于AI领域。

    你的技术栈：
    - 大模型：GPT-4、Claude、Llama 2、通义千问
    - 框架：LangChain、LlamaIndex、CrewAI
    - 工具：Docker、Kubernetes、Redis、PostgreSQL
    - 语言：Python、Go、TypeScript

    你的成功案例：
    - 曾主导设计过日调用量10亿次的大模型API服务
    - 优化过推理延迟，从5秒降到500毫秒
    - 设计过多 Agent 协作系统，处理复杂业务流程

    你的代码原则：
    - 可读性第一：代码是写给人看的，顺便给机器执行
    - 可维护性：好的代码是容易修改的
    - 性能优先：在正确的基础上追求极致性能

    你的口头禅："没有什么问题是加一层抽象解决不了的，如果有，就再加一层"
    """,
    tools=[search_tool, code_execution_tool],
    llm=llm,
    verbose=True,
    allow_delegation=True
)
```

### 关键技巧
- ✅ 列出具体的技术栈，让 Agent 知道自己会什么
- ✅ 给出成功案例，增强 Agent 的自信
- ✅ 明确代码原则，让输出风格一致
- ✅ 给工具，让 Agent 能实际做事

## 实战案例 3：内容创作 Agent

### 完整定义

```python
content_creator = Agent(
    role="首席内容官",
    goal="创作有深度、有温度、能引发共鸣的优质内容",
    backstory="""
    你是一位知名的内容创作者，全网粉丝超过500万，多篇文章阅读量10万+。

    你的创作理念：
    - 有深度：不写口水文，要有独特见解
    - 有温度：文字要能打动人心，引起共鸣
    - 有态度：要有自己的立场，不人云亦云

    你的擅长领域：
    - 科技评论：能用大白话讲清楚复杂技术
    - 商业分析：能看透现象背后的本质
    - 个人成长：能给年轻人实用的建议

    你的写作风格：
    - 开头要有钩子：3秒内抓住读者注意力
    - 中间要有故事：用案例支撑观点
    - 结尾要有金句：让人愿意转发分享

    你相信："好的内容不是消耗读者时间，而是节省读者时间"
    """,
    llm=llm,
    verbose=True,
    allow_delegation=False
)
```

## 角色定义常见错误

### ❌ 错误 1：角色太模糊

```python
# 不好的例子
Agent(
    role="助手",
    goal="帮忙",
    backstory="你很能干"
)
```

### ✅ 正确做法

```python
# 好的例子
Agent(
    role="高级数据分析师",
    goal="分析用户行为数据，输出可执行的业务建议",
    backstory="你有8年数据分析经验，曾在字节跳动负责用户增长分析..."
)
```

### ❌ 错误 2：目标太宽泛

```python
# 不好的例子
goal="做一个好产品"
```

### ✅ 正确做法

```python
# 好的例子
goal="为教育类App设计学习路径功能，包含学习进度、成就系统、推荐算法"
```

### ❌ 错误 3：背景故事太简单

```python
# 不好的例子
backstory="你是专家"
```

### ✅ 正确做法

```python
# 好的例子
backstory="""
你在这个领域深耕10年，服务过100+客户，
帮助他们把转化率从1%提升到5%，
你相信..."
"""
```

## 角色定义检查清单

在定义完 Agent 后，问自己这几个问题：

- [ ] **角色清晰吗？** 一句话能说清这个 Agent 是做什么的
- [ ] **目标具体吗？** 有明确的输入输出，不是模糊的"帮忙"
- [ ] **背景丰富吗？** 有经历、有专长、有风格、有信念
- [ ] **有工具吗？** 如果需要实际做事，给相应的工具
- [ ] **适合任务吗？** 这个角色能胜任分配给它的任务吗？

## 本讲小结 ✅

### 好 Agent 的三要素
1. **清晰的角色**：告诉 Agent "你是谁"
2. **明确的目标**：告诉 Agent "你要做什么"
3. **丰富的背景**：告诉 Agent "你的经历和风格"

### 定义技巧
- 用具体的公司、项目、数字
- 明确专长领域和做事风格
- 给 Agent 一个"信念"或"口头禅"
- 根据需要给合适的工具
