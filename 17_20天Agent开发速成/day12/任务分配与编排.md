# 任务分配与编排 📋

&gt; 📖 **本讲学习目标**：能定义清晰的 Task，能设计任务之间的依赖关系，能选择合适的编排策略  
&gt; ⏰ **预计学习时间**：1 小时

---

## 什么是好的 Task 定义？

### 好 Task 的三要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **清晰的描述** | 告诉 Agent 具体做什么 | "分析用户反馈，总结前3个最常见问题" |
| **明确的输出要求** | 告诉 Agent 输出什么格式 | "1000字分析报告，包含问题列表和建议" |
| **合适的执行者** | 分配给正确的 Agent | 把数据分析任务分配给数据分析师 Agent |

## Task 定义实战

### 基础模板

```python
Task(
    description="""
    【任务描述，详细说明要做什么】
    1. 第一步：...
    2. 第二步：...
    3. 第三步：...
    """,
    agent=agent_name,
    expected_output="""
    【期望输出格式】
    - 格式1：JSON
    - 格式2：Markdown列表
    - 格式3：1000字报告
    """
)
```

### 实战案例 1：产品调研 Task

```python
task_research = Task(
    description="""
    对智能教育市场进行调研，具体要求：
    1. 分析市场规模和增长趋势
    2. 调研主要竞争对手及其优缺点
    3. 总结目标用户的核心需求
    4. 给出进入这个市场的机会点建议

    请用数据支撑你的分析，不要空泛。
    """,
    agent=market_researcher,
    expected_output="""
    一份完整的市场调研报告，包含：
    - 市场概况（200字）
    - 竞品分析（3个主要竞品，每个150字）
    - 用户需求（前5个核心需求）
    - 机会点建议（3个方向）
    总字数：1500字左右
    """
)
```

## 任务依赖关系设计

### 依赖类型 1：顺序依赖

```mermaid
graph LR
    A[Task A] --&gt; B[Task B]
    B --&gt; C[Task C]
```

```python
task_a = Task(description="第一步...", agent=agent1)
task_b = Task(description="基于Task A做...", agent=agent2, context=[task_a])
task_c = Task(description="基于Task B做...", agent=agent3, context=[task_b])
```

### 依赖类型 2：并行 + 汇总

```mermaid
graph TB
    A[主任务] --&gt; B[Task A]
    A --&gt; C[Task B]
    A --&gt; D[Task C]
    B --&gt; E[汇总 Task]
    C --&gt; E
    D --&gt; E
```

```python
# 并行任务
task_a = Task(description="调研竞品A", agent=agent1)
task_b = Task(description="调研竞品B", agent=agent2)
task_c = Task(description="调研竞品C", agent=agent3)

# 汇总任务（依赖前面3个）
task_summary = Task(
    description="汇总3个竞品的调研结果",
    agent=agent4,
    context=[task_a, task_b, task_c]
)
```

### 依赖类型 3：混合依赖

```mermaid
graph TB
    A[Task A] --&gt; B[Task B]
    A --&gt; C[Task C]
    B --&gt; D[Task D]
    C --&gt; D
```

```python
task_a = Task(description="产品规划", agent=pm)
task_b = Task(description="前端设计", agent=designer, context=[task_a])
task_c = Task(description="后端设计", agent=architect, context=[task_a])
task_d = Task(description="集成开发", agent=engineer, context=[task_b, task_c])
```

## 编排策略选择

### 策略 1：顺序编排（Sequential）

```python
Crew(
    agents=[agent1, agent2, agent3],
    tasks=[task1, task2, task3],
    process=Process.sequential  # 顺序执行
)
```

**适用场景**：
- 有明确前后依赖的任务
- 需要前一个输出作为后一个输入
- 流程相对简单

---

### 策略 2：分层编排（Hierarchical）

```python
Crew(
    agents=[manager_agent, worker1, worker2, worker3],
    tasks=[task1, task2, task3],
    process=Process.hierarchical,  # 分层管理
    manager_llm=manager_llm  # 管理者用的 LLM
)
```

**适用场景**：
- 复杂任务需要协调
- 需要有人做决策和分配
- 任务之间关系复杂

---

### 策略 3：异步编排（Async）

```python
task1 = Task(description="...", async_execution=True)
task2 = Task(description="...", async_execution=True)
task3 = Task(description="...", context=[task1, task2])

Crew(
    agents=[agent1, agent2, agent3],
    tasks=[task1, task2, task3]
)
```

**适用场景**：
- 任务之间独立，可以并行
- 追求效率，想缩短总时间
- 有明确的汇总步骤

## Task 定义检查清单

在定义完 Task 后，问自己这几个问题：

- [ ] **描述清晰吗？** Agent 看了知道具体要做什么
- [ ] **输出明确吗？** 知道输出什么格式、多少字数
- [ ] **分配正确吗？** 给了合适的 Agent 来执行
- [ ] **依赖合理吗？** 需要的前置任务都在 context 里了
- [ ] **可以衡量吗？** 完成与否有明确的判断标准

## 常见问题 FAQ

### Q1: Task 描述写多长合适？

| 任务类型 | 建议长度 |
|---------|---------|
| **简单任务** | 1-2 句话 |
| **中等任务** | 1 段话（3-5 句） |
| **复杂任务** | 分点列出（3-5 个点） |

### Q2: 如何让 Task 输出更稳定？

**技巧 1：给示例**
```python
expected_output="""
例如：
{
    "name": "产品名称",
    "features": ["功能1", "功能2"]
}
"""
```

**技巧 2：指定格式**
```python
expected_output="请输出严格的 JSON 格式，不要有其他文字"
```

**技巧 3：约束条件**
```python
expected_output="字数控制在500-800字，分3个段落"
```

## 本讲小结 ✅

### 好 Task 的三要素
1. **清晰的描述**：告诉 Agent 做什么
2. **明确的输出**：告诉 Agent 输出什么
3. **合适的执行者**：分配给正确的 Agent

### 依赖关系
- **顺序依赖**：A → B → C
- **并行汇总**：A/B/C 并行 → 汇总
- **混合依赖**：复杂的依赖关系

### 编排策略
- **sequential**：顺序执行
- **hierarchical**：分层管理
- **async**：异步并行
