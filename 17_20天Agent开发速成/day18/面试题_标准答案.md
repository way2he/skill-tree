## 问题 1：FastAPI 和 Flask 的区别是什么？为什么 Agent 开发常用 FastAPI？

### 回答要点：

**对比：**

| 特性 | Flask | FastAPI |
|------|-------|---------|
| **异步** | 需额外扩展 | 原生支持 async/await |
| **类型提示** | 无 | 原生支持 Pydantic |
| **自动文档** | 需手动 | 自动生成 Swagger/OpenAPI |
| **性能** | 一般 | 更高（基于 Starlette + Pydantic） |
| **学习曲线** | 简单 | 稍陡 |

**为什么 Agent 常用 FastAPI：**
1. **异步支持**：Agent 常需并发调用多个工具
2. **类型安全**：Pydantic 验证输入输出
3. **自动文档**：方便前后端协作
4. **高性能**：适合生产环境

## 问题 2：什么是 SSE（Server-Sent Events）？和 WebSocket 的区别是什么？

### 回答要点：

**SSE：**
- 服务器 → 客户端的单向推送
- 基于 HTTP，简单易用
- 适合：实时日志、进度更新、通知

**WebSocket：**
- 服务器 ↔ 客户端双向通信
- 独立协议，更复杂
- 适合：聊天、实时协作、游戏

**对比：**

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| **方向** | 服务器 → 客户端 | 双向 |
| **协议** | HTTP | WebSocket |
| **复杂度** | 简单 | 复杂 |
| **适用场景** | 推送通知 | 实时交互 |

## 问题 3：Docker 容器化的优势是什么？如何优化 Docker 镜像大小？

### 回答要点：

**Docker 优势：**
1. **环境一致性**：开发/测试/生产环境一致
2. **快速部署**：启动快，秒级
3. **隔离性**：应用之间互不影响
4. **可移植**：一次构建，到处运行

**优化镜像大小：**
1. **多阶段构建**：编译和运行分离
2. **用小基础镜像**：alpine、slim
3. **合并层**：减少 RUN 命令
4. **.dockerignore**：排除不必要的文件
