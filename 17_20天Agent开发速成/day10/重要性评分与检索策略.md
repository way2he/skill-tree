# 重要性评分与检索策略 🎯

&gt; 🎯 **学习目标**：设计重要性评分和高效检索策略

---

## 一、重要性评分机制

### 1.1 为什么需要重要性评分？

```
记忆 = 珍贵的珠宝 + 无用的垃圾
重要性评分 = 区分珠宝和垃圾
```

---

### 1.2 评分维度

| 维度 | 说明 | 示例 |
|------|------|------|
| **显式反馈** | 用户明确标记 | 收藏、点赞 |
| **隐式反馈** | 用户行为 | 多次查看、引用 |
| **内容特征** | 内容本身 | 包含重要信息、长文本 |
| **访问频率** | 访问次数 | 高频访问更重要 |
| **时间衰减** | 最近更重要 | 近一周 > 一个月前 |

---

### 1.3 评分公式

```
重要性 = w1×内容分数 + w2×访问分数 + w3×时间分数
```

**归一化**：将各维度分数归一化到 [0, 1]

---

### 1.4 代码实现

```python
def calculate_importance(memory):
    """计算重要性分数"""
    score = 0
    
    # 1. 内容特征
    if has_keywords(memory.content, ["重要", "关键", "记住"]):
        score += 0.3
    score += min(len(memory.content) / 1000, 0.2)  # 长度
    
    # 2. 访问频率
    score += min(memory.access_count / 10, 0.3)
    
    # 3. 时间衰减
    days_ago = (now() - memory.timestamp).days
    time_score = max(0, 1 - days_ago / 30)  # 30天衰减
    score += 0.2 * time_score
    
    return min(score, 1.0)
```

## 二、检索策略

### 2.1 常见检索策略

| 策略 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **向量检索** | 语义相似度 | 理解语义 | 可能忽略关键词 |
| **关键词检索** | BM25/TF-IDF | 精确匹配 | 不理解语义 |
| **时间排序** | 最近优先 | 简单 | 忽略相关性 |
| **混合检索** | 多种结合 | 全面 | 复杂 |

---

### 2.2 混合检索实现

```python
def hybrid_retrieve(query, memory_store, top_k=5):
    """混合检索"""
    # 1. 多路召回
    vector_results = memory_store.vector_search(query, top_k*2)
    keyword_results = memory_store.keyword_search(query, top_k*2)
    recent_results = memory_store.get_recent(top_k*2)
    
    # 2. 合并去重
    candidates = merge_and_dedupe(
        vector_results,
        keyword_results,
        recent_results
    )
    
    # 3. 重排序
    scored = []
    for candidate in candidates:
        score = 0
        score += 0.5 * candidate.vector_score  # 向量相似度
        score += 0.3 * candidate.importance   # 重要性
        score += 0.2 * time_decay(candidate)   # 时间衰减
        scored.append((candidate, score))
    
    # 4. 排序返回 Top K
    scored.sort(key=lambda x: x[1], reverse=True)
    return [item[0] for item in scored[:top_k]]
```

## 三、遗忘机制

### 3.1 为什么需要遗忘？

```
无限记忆 = 存储爆炸 + 检索变慢 + 噪声干扰
遗忘 = 清理无用信息，保留精华
```

---

### 3.2 遗忘策略

| 策略 | 说明 |
|------|------|
| **FIFO** | 最早的先删除 |
| **LRU** | 最久未用的删除 |
| **重要性优先** | 保留重要的，删除不重要的 |
| **压缩** | 旧记忆摘要/合并 |

---

### 3.3 代码实现

```python
def forget(memory_store, max_size=10000):
    """遗忘策略"""
    if len(memory_store) <= max_size:
        return
    
    # 计算遗忘分数（越低越可能被删除）
    for memory in memory_store:
        memory.forget_score = (
            0.7 * (1 - memory.importance) +
            0.3 * days_since_last_access(memory) / 365
        )
    
    # 删除得分最低的
    num_to_delete = len(memory_store) - max_size
    to_delete = sorted(memory_store, key=lambda m: m.forget_score)[:num_to_delete]
    memory_store.delete_all(to_delete)
```

## 四、评估指标

### 4.1 检索质量指标

| 指标 | 说明 |
|------|------|
| **Precision@k** | Top K 中相关的比例 |
| **Recall@k** | 相关的被找回的比例 |
| **MRR** | 第一个相关结果的位置倒数 |
| **NDCG** | 考虑排序质量的指标 |
