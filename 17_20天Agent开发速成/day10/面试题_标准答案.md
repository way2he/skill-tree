## 基础题（3 道）

### 第 1 题：三层记忆架构

**参考答案**：

#### 设计思想
参考人类记忆模型，分层设计是为了平衡容量和效率：
- 快速访问的层容量小
- 大容量的层访问相对慢

#### 各层特点

| 层级 | 持续时间 | 容量 | 作用 | 示例 |
|------|---------|------|------|------|
| **感觉记忆** | 秒级 | 所有输入 | 暂存输入，过滤 | 原始输入 |
| **短期/工作记忆** | 会话级 | 当前会话 | 保存当前状态 | 对话历史 |
| **长期记忆** | 永久 | 所有历史 | 保存知识经验 | 用户资料、历史摘要 |

#### 例子
- 用户问："我上次提到的那个项目怎么样了？"
- 短期记忆：搜索当前会话
- 长期记忆：检索历史对话中的项目信息

### 第 2 题：重要性评分

**参考答案**：

#### 评分维度
1. **内容特征**：关键词（重要、关键）、长度、完整性
2. **访问频率**：访问次数、最近访问时间
3. **用户反馈**：收藏、点赞、显式标记
4. **时间衰减**：最近的更重要

#### 评分公式
```
重要性 = w1×内容分数 + w2×访问分数 + w3×反馈分数 + w4×时间分数
```

#### 实现思路
```python
def calculate_importance(memory):
    score = 0
    
    # 内容特征
    if has_keywords(memory.content, ["重要", "关键", "记住"]):
        score += 0.3
    score += min(len(memory.content) / 1000, 0.2)
    
    # 访问频率
    score += min(memory.access_count / 10, 0.3)
    
    # 时间衰减
    days_ago = (now() - memory.timestamp).days
    time_score = max(0, 1 - days_ago / 30)
    score += 0.2 * time_score
    
    return min(score, 1.0)  # 归一化到 [0,1]
```

### 第 3 题：检索策略对比

**参考答案**：

| 策略 | 说明 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **向量检索** | 语义相似度 | 理解语义 | 可能忽略关键词 | 语义相关查询 |
| **关键词检索** | BM25/TF-IDF | 精确匹配 | 不理解语义 | 精确关键词查询 |
| **时间排序** | 最近优先 | 简单直观 | 忽略相关性 | 最新信息查询 |
| **重要性排序** | 重要优先 | 关注高价值 | 可能忽略时效 | 重要信息查询 |
| **混合检索** | 多种结合 | 全面准确 | 实现复杂 | 通用场景 |

## 进阶题（3 道）

### 第 4 题：遗忘机制

**参考答案**：

#### 为什么需要遗忘
1. **存储限制**：防止存储爆炸
2. **检索效率**：记忆越多，检索越慢
3. **质量控制**：删除无用信息，减少噪声
4. **资源优化**：释放存储空间

#### 常见遗忘策略
| 策略 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **FIFO** | 最早的先删除 | 简单 | 可能删除重要的 |
| **LRU** | 最久未用删除 | 考虑使用情况 | 实现稍复杂 |
| **重要性优先** | 保留重要的 | 考虑价值 | 需要评分机制 |
| **压缩** | 旧记忆摘要合并 | 保留信息 | 有信息损失 |

#### 推荐方案
```
遗忘分数 = 0.7×(1-重要性) + 0.3×(时间/365)
删除分数最低的
```

### 第 5 题：混合检索

**参考答案**：

#### 混合检索思路
```
查询 → 多路召回 → 合并去重 → 重排序 → Top K
```

#### 多路召回
```python
# 多路召回
vector_results = vector_store.search(query, top_k*2)
keyword_results = bm25.search(query, top_k*2)
recent_results = get_recent(top_k*2)

# 合并去重
candidates = merge_and_dedupe(
    vector_results,
    keyword_results,
    recent_results
)
```

#### 重排序
```python
# 重排序
for candidate in candidates:
    score = (
        0.5 * candidate.vector_score +  # 向量相似度
        0.3 * candidate.importance +    # 重要性
        0.2 * time_decay(candidate)     # 时间衰减
    )

# 排序返回
candidates.sort(key=lambda x: x.score, reverse=True)
return candidates[:top_k]
```

### 第 6 题：评估指标

**参考答案**：

#### 评估维度
1. **检索质量**：是否找到相关记忆
2. **检索效率**：查询响应时间
3. **存储效率**：空间占用
4. **用户满意度**：用户反馈

#### 评估指标
| 指标 | 说明 | 公式 |
|------|------|------|
| **Precision@k** | Top K 中相关的比例 | 相关数 / K |
| **Recall@k** | 相关的被找回的比例 | 找回的相关数 / 总相关数 |
| **MRR** | 第一个相关结果的位置倒数 | 1 / rank |
| **NDCG** | 考虑排序质量 | DCG / IDCG |
| **响应时间** | 查询耗时 | - |
