## 第一部分：元数据过滤

### 什么是元数据？

元数据是**关于数据的数据**，描述文档的属性。

### 常见元数据类型

| 元数据 | 说明 | 例子 |
|--------|------|------|
| **分类/标签** | 文档属于什么类别 | 技术、产品、运营 |
| **时间** | 文档创建/更新时间 | 2024-01-15 |
| **作者** | 谁写的 | 张三、李四 |
| **来源** | 文档来源 | 官网、公众号、内部文档 |
| **优先级** | 重要程度 | P0、P1 |

### 为什么要元数据过滤？

#### 问题
用户问："最近 2024 年 AI 相关的内容"
- 纯向量检索：可能把 2020 年的旧内容也检索出来
- 加元数据过滤：只检索 2024 年的文档，更准确！

#### 例子
```
查询: "2024年 AI 相关内容"

步骤:
1. 元数据过滤: 只看 year=2024 的文档
2. 在过滤后的文档里做向量检索
3. 返回结果
```

### 元数据过滤的实现

#### 在向量数据库中

| 数据库 | 元数据过滤 |
|--------|-----------|
| **Chroma** | 支持 `where` 条件 |
| **Qdrant** | 支持 `filter` |
| **Pinecone** | 支持元数据过滤 |

#### Chroma 代码示例
```python
# 按时间过滤
results = collection.query(
    query_texts=["AI 技术趋势"],
    where={"year": {"$eq": 2024}},
    n_results=3
)

# 按分类过滤
results = collection.query(
    query_texts=["产品文档"],
    where={"category": "product"},
    n_results=3
)
```

## 第二部分：混合检索（Hybrid Search）

### 为什么需要混合检索？

#### 纯向量检索的问题
- 对**关键词精确匹配**不好（比如用户搜 "iPhone 15 价格"，纯向量可能找到 "iPhone 14"）
- 对**新术语/专有名词**效果不好

#### 纯关键词检索（BM25）的问题
- 对**语义匹配**不好（比如用户搜 "苹果手机"，找不到 "iPhone"）

#### 混合检索 = BM25 + 向量
- **BM25**：关键词精确匹配
- **向量**：语义匹配
- **结合**：召回率更高！

---

### BM25 是什么？

#### 一句话说明
BM25 是传统的信息检索算法，基于 TF-IDF 改进，用于关键词匹配。

#### 核心思想
- 词频越高越重要
- 文档越短越重要
- 太常见的词（停用词）不重要

#### BM25 公式（了解即可）
```
Score = Σ IDF * [TF * (k1+1)] / [TF + k1 * (1 - b + b * (doc_len / avg_len))]
```

#### BM25 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 关键词精确匹配好 | ❌ 语义匹配差 |
| ✅ 计算快 | ❌ 对同义词效果不好 |
| ✅ 可解释性强 | ❌ 不理解语义 |

---

### 混合检索流程

```
用户查询
    |
    +-- BM25 检索 → Top 20
    |
    +-- 向量检索 → Top 20
    |
    +-- 结果合并 & 去重
    |
    +-- Rerank 重排序 → Top 5
    |
    最终结果
```

---

### 混合检索代码示例

```python
# -*- coding: utf-8 -*-
"""
混合检索示例：BM25 + 向量
"""

import numpy as np
from rank_bm25 import BM25Okapi
from sentence_transformers import SentenceTransformer, util

# 1. 准备文档
documents = [
    "iPhone 15 价格 5999 元起，A17 芯片",
    "iPhone 14 降价促销，A15 芯片",
    "苹果手机新品发布会，iPhone 15 亮相",
    "华为 Mate 60 Pro 发布，麒麟芯片回归",
    "小米 14 系列，骁龙 8 Gen 3",
    "vivo X100 拍照手机，蔡司镜头"
]

query = "iPhone 15 价格"

# 2. BM25 检索
print("=" * 60)
print("BM25 检索结果")
print("=" * 60)
tokenized_docs = [doc.split() for doc in documents]
bm25 = BM25Okapi(tokenized_docs)
bm25_scores = bm25.get_scores(query.split())
# 排序
bm25_results = sorted(zip(bm25_scores, documents), key=lambda x: -x[0])[:3]
for i, (score, doc) in enumerate(bm25_results, 1):
    print(f"{i}. (score: {score:.4f}) {doc}")

# 3. 向量检索
print("\n" + "=" * 60)
print("向量检索结果")
print("=" * 60)
model = SentenceTransformer('BAAI/bge-small-zh-v1.5')
doc_embeddings = model.encode(documents, convert_to_tensor=True)
query_embedding = model.encode(query, convert_to_tensor=True)
vec_hits = util.semantic_search(query_embedding, doc_embeddings, top_k=3)[0]
for i, hit in enumerate(vec_hits, 1):
    print(f"{i}. (score: {hit['score']:.4f}) {documents[hit['corpus_id']]}")

# 4. 混合检索结果（合并去重）
print("\n" + "=" * 60)
print("混合检索结果（合并去重）")
print("=" * 60)
bm25_set = set([doc for _, doc in bm25_results])
vec_set = set([documents[hit['corpus_id']] for hit in vec_hits])
hybrid_set = bm25_set.union(vec_set)
print(f"混合后文档数: {len(hybrid_set)}")
print("\n合并后的文档:")
for doc in hybrid_set:
    print(f"- {doc}")

print("\n" + "=" * 60)
print("混合检索优势")
print("=" * 60)
print("""
BM25: 关键词精确匹配好（如 'iPhone 15 价格'）
向量: 语义匹配好（如 '苹果手机' = 'iPhone'）
混合: 两者结合，召回率更高！
""")
```

## 混合检索最佳实践

### 建议
1. ✅ **检索数量**：各召回 20-50 个
2. ✅ **结果合并**：去重后用 Rerank 重排
3. ✅ **权重可调**：BM25 和向量分数可以加权
4. ✅ **评估对比**：看混合检索在你的数据上是否更好
