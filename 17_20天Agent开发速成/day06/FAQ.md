# Day06 常见问题 FAQ ❓

---

## RAG 优化相关

### Q1：分块大小怎么选？有什么最佳实践？

| 场景 | 推荐大小 |
|------|---------|
| 问答 | 256-512 tokens |
| 摘要 | 512-1024 tokens |
| 长文档 | 1024-2048 tokens |
| 父子分块 | 父 1024-2048 / 子 128-256 |

**最佳实践**：
- 加重叠（50-100 tokens）
- 优先语义分块
- 测试评估，看哪种在你的数据上效果最好

---

### Q2：混合检索中，BM25 和向量的权重怎么调？

**建议**：
- 先各 0.5（等权重）
- 然后做消融实验，看哪个权重效果好
- 可以学习权重（用验证集调）

---

### Q3：查询重写一定有效果吗？什么时候用？

**不一定**：
- 如果用户查询本身就很好，可能不需要重写
- 如果查询很口语化、很啰嗦，重写效果明显

**建议**：
- 先试试不重写
- 效果不好再加查询重写
- A/B 测试对比

---

## RAGAS 评估相关

### Q4：没有 ground_truth 怎么评估？

**方案**：
- 只用不需要 ground_truth 的指标：Faithfulness、Answer Relevance
- 或者人工标注少量验证集

---

### Q5：RAGAS 评估成本高吗？

**成本**：
- RAGAS 用 LLM 评估，有 API 成本
- 建议：先少量测试（50-100 条）
- 效果稳定后，定期评估（每周/每月）

---

## NLP 相关

### Q6：现在做文本分类，直接上 BERT 吗？

**建议**：
- 先看数据量：
  - 数据很少（&lt;1000）：FastText 或 TF-IDF+SVM
  - 数据中等（1000-10000）：小 BERT（distilbert）
  - 数据多（&gt;10000）：BERT-base/ERNIE
- 优先试开源中文模型：ernie-3.0、chinese-bert-wwm 等
