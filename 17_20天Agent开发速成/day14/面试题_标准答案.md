# Day14 面试题标准答案 ✅

## 第 1 题：KV Cache

**答案要点**：

KV Cache 缓存 Transformer 每一步计算的 Key 和 Value，避免重复计算：

- 无 Cache：每步都要重新计算之前所有 token 的 KV（O(n²)）
- 有 Cache：每步只用计算当前 token 的 KV（O(n)）
- 效果：推理速度快 2-5 倍

---

## 第 2 题：量化对比

**答案要点**：

| 精度 | 显存 | 相对大小 | 效果 |
|------|------|---------|------|
| FP32 | 100% | 基准 | 最好 |
| FP16 | 50% | 1/2 | 很好 |
| INT8 | 25% | 1/4 | 可接受 |
| INT4 | 12.5% | 1/8 | 有一些损失 |

---

## 第 3 题：LoRA 优点

**答案要点**：

- 显存少：只训练小部分参数（0.1-1%）
- 训练快：不用算大矩阵的梯度
- 存储小：LoRA adapter 只有几十 MB
- 灵活：可以在不同任务间切换
- 效果好：接近全量微调

---

## 第 4 题：LoRA 原理

**答案要点**：

LoRA 认为：
1. 预训练模型已经有很好的知识
2. 适应新任务只需要小的调整
3. 这个调整可以用低秩矩阵来表示

所以：
- W（大矩阵，冻结）
- A × B（小矩阵，训练）
- 最终：W + A × B

---

## 第 5 题：QLoRA 改进

**答案要点**：

QLoRA = LoRA + 量化（INT4）：
1. 把基础模型量化到 INT4
2. 用 LoRA 微调
3. 计算时用 FP16/BF16

效果：
- 7B 模型可以在 8GB 显存上微调！
- 效果接近 FP16 LoRA

---

## 第 6 题：决策建议

**答案要点**：

决策树：
- 显存够？
  - 够 → FP16 + KV Cache
  - 不够 → 试试 INT8
    - INT8 还不够 → INT4/QLoRA

- 速度要求？
  - 实时性高 → vLLM/TensorRT-LLM
  - 批量处理 → 批处理 + KV Cache
