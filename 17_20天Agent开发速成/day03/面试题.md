## 一、概念题（共40分，每题8分）

### 第1题：temperature 和 top_p 的区别（8分）
**问题**：大模型 API 里的 temperature 和 top_p 参数分别是什么作用？实际使用中怎么选？

<details>
<summary>💡 参考答案与打分细则</summary>

| 评分点 | 分值 | 说明 |
|--------|------|------|
| temperature 作用 | 3分 | 控制输出的随机性/创造性，0-1之间。0=最确定，每次输出都差不多；1=最发散，每次输出差异大 |
| top_p 作用 | 3分 | 核采样，控制候选词的范围。0.1=只从前10%概率的词里选，更确定；1=全部词都可以选 |
| 使用建议 | 2分 | 建议两个参数只改一个，不要同时改。追求准确用低温度/低top_p，追求创意用高温度/高top_p |

**评分标准**：说清楚两个参数的作用得6分，给出使用建议得8分。
</details>

### 第2题：Token 与成本计算（8分）
**问题**：什么是 Token？大模型是怎么计费的？如果一次调用输入 2000 Token，输出 1000 Token，用 GPT-3.5-turbo（输入 $0.0005/1k，输出 $0.0015/1k），成本是多少？

<details>
<summary>💡 参考答案与打分细则</summary>

| 评分点 | 分值 | 说明 |
|--------|------|------|
| Token 定义 | 2分 | 大模型的基本处理单位，1 Token ≈ 0.7个中文汉字 ≈ 0.75个英文单词 |
| 计费方式 | 3分 | 输入 Token + 输出 Token 都要计费，分开计价，通常输出比输入贵 |
| 计算结果 | 3分 | 输入成本：2k × $0.0005/k = $0.001；输出成本：1k × $0.0015/k = $0.0015；总成本 = $0.0025（约人民币 1 分 8 厘） |

**评分标准**：定义正确2分，计费方式正确3分，计算结果正确3分，共8分。
</details>

**问题**：大模型 API 的 stream 流式输出有什么优势？实现流式输出需要注意什么？

### 第5题：Function Calling 原理（8分）
**问题**：大模型的 Function Calling 是怎么工作的？大模型真的会执行函数吗？

<details>
<summary>💡 参考答案与打分细则</summary>

| 评分点 | 分值 | 说明 |
|--------|------|------|
| 原理 | 4分 | 你把函数定义（函数名、参数、描述）传给大模型，大模型根据用户问题决定：① 直接回答；② 需要调用某个函数，返回要调用的函数名和参数 |
| 执行主体 | 4分 | 大模型 **不执行** 函数！它只告诉你「应该调用什么函数，参数是什么」，真正执行函数的是你的代码，执行完结果再传回给大模型 |

**常见误区**：很多人以为大模型会执行函数，实际上它只做「决策」，不做「执行」。

**评分标准**：原理说清楚得4分，执行主体说清楚得4分，共8分。
</details>

## 二、代码题（共40分，每题20分）

### 第6题：带重试和错误处理的 API 封装（20分）
**问题**：请封装一个生产级的大模型调用函数，要求：
1. 支持流式和非流式输出
2. 429/5xx 错误自动重试（指数退避）
3. 记录每次调用的 Token 消耗和成本
4. 异常捕获和友好错误提示

<details>
<summary>💡 参考答案与打分细则</summary>

| 评分点 | 分值 | 说明 |
|--------|------|------|
| 基本调用功能 | 5分 | 能正常调用 API，支持消息输入 |
| 流式输出支持 | 5分 | 支持 stream 参数，正确处理流式返回 |
| 重试机制 | 5分 | 指数退避重试，只重试可重试的错误 |
| Token 统计 + 成本计算 | 5分 | 记录输入输出 Token，计算成本 |

**参考实现核心代码结构**：
```python
import openai
import time
from tenacity import retry, stop_after_attempt, wait_exponential

class LLMClient:
    def __init__(self, api_key):
        self.client = openai.OpenAI(api_key=api_key)
    
    @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10))
    def call(self, messages, model="gpt-3.5-turbo", stream=False, temperature=0.7):
        try:
            response = self.client.chat.completions.create(
                model=model,
                messages=messages,
                temperature=temperature,
                stream=stream
            )
            
            if stream:
                return self._stream_response(response)
            else:
                return self._handle_normal_response(response)
                
        except openai.RateLimitError:
            print("触发限流，重试中...")
            raise  # tenacity 会自动重试
        except Exception as e:
            print(f"调用失败: {e}")
            return None
    
    def _handle_normal_response(self, response):
        # 统计 Token 和成本
        input_tokens = response.usage.prompt_tokens
        output_tokens = response.usage.completion_tokens
        cost = self._calc_cost(input_tokens, output_tokens, response.model)
        print(f"Token消耗: 输入{input_tokens}, 输出{output_tokens}, 成本: ${cost:.6f}")
        return response.choices[0].message.content
```

**评分标准**：四个功能点各5分，共20分。
</details>

### 第7题：统一多模型适配层（20分）
**问题**：请设计一个统一的多模型适配层，要求：
1. 支持至少 2 家不同的大模型（如 OpenAI + 通义千问）
2. 调用接口完全一致，换模型只需要改一个参数
3. 支持自动降级：主模型失败自动切备用模型

<details>
<summary>💡 参考答案与打分细则</summary>

| 评分点 | 分值 | 说明 |
|--------|------|------|
| 统一接口设计 | 7分 | 基类抽象，不同模型实现相同接口 |
| 多模型实现 | 7分 | 至少2个模型的具体实现，参数转换 |
| 自动降级机制 | 6分 | 主模型失败自动尝试备用模型 |

**参考设计**：
```python
from abc import ABC, abstractmethod

class BaseLLM(ABC):
    @abstractmethod
    def chat(self, messages, **kwargs):
        pass

class OpenAILLM(BaseLLM):
    def chat(self, messages, **kwargs):
        # OpenAI 实现
        pass

class QwenLLM(BaseLLM):
    def chat(self, messages, **kwargs):
        # 通义千问实现，参数转换
        pass

class MultiModelLLM:
    def __init__(self, primary: BaseLLM, fallback: BaseLLM):
        self.primary = primary
        self.fallback = fallback
    
    def chat(self, messages, **kwargs):
        try:
            return self.primary.chat(messages, **kwargs)
        except Exception as e:
            print(f"主模型失败: {e}, 切换备用模型")
            return self.fallback.chat(messages, **kwargs)

# 使用
llm = MultiModelLLM(
    primary=OpenAILLM(),
    fallback=QwenLLM()
)
llm.chat(...)  # 自动降级
```

**评分标准**：统一接口设计7分，多模型实现7分，降级机制6分，共20分。
</details>
