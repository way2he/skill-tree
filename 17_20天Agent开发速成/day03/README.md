## 📚 上午：OpenAI API 完整指南

### 1. API 核心参数详解（45分钟）

#### 核心参数清单

| 参数 | 作用 | 常用值 | 注意事项 |
|------|------|--------|---------|
| `model` | 用哪个模型 | gpt-3.5-turbo, gpt-4o | 价格差10倍以上！ |
| `messages` | 对话历史 | 数组格式 | system + user + assistant |
| `temperature` | 温度 | 0-1，默认0.7 | 0=最确定，1=最发散 |
| `top_p` | 核采样 | 0-1，默认1 | 建议和温度只改一个 |
| `max_tokens` | 最大输出长度 | 数字 | 控制成本，防止无限输出 |
| `stream` | 流式输出 | true/false | 用户体验提升神器 |
| `functions` | 函数定义 | 数组 | Function Calling 用 |
| `function_call` | 函数调用控制 | "auto"/"none"/指定 | 强制或禁止调用工具 |

#### temperature vs top_p 怎么选？

| 场景 | 建议 |
|------|------|
| 代码生成、事实问答 | temperature=0-0.3，追求准确 |
| 创意写作、头脑风暴 | temperature=0.7-1.0，追求发散 |
| 翻译、摘要 | temperature=0.3-0.5，平衡 |

---

### 2. 流式输出 + Function Calling（45分钟）

#### 流式输出为什么重要？
- 用户体验提升 10 倍：不用等全部输出完，边生成边显示
- 感知速度更快：实际上总时间一样，但用户觉得快很多
- 长文本输出不会超时

#### Function Calling 核心原理
- 让大模型决定：「我现在需要调用工具吗？调用哪个？参数是什么？」
- 大模型不真的执行工具，只告诉你应该调用什么，参数是什么
- 你代码里真正执行，执行结果再传回给大模型

### 3. 错误处理 + 限流重试（30分钟）

#### 常见错误类型

| 错误码 | 原因 | 处理方式 |
|--------|------|---------|
| 429 | 限流了，请求太频繁 | 指数退避重试 |
| 401 | API Key 错了 | 检查 Key |
| 403 | 没有权限 | 检查账号权限 |
| 400 | 参数错误 | 检查传参 |
| 5xx | 服务器端错误 | 重试，或者降级用其他模型 |

### 5. Token 计费 + 成本估算（1小时）

#### 什么是 Token？
- Token 是大模型的「计费单位」
- 1 Token ≈ 0.7 个中文汉字 ≈ 0.75 个英文单词
- 输入 + 输出 都要计费！

#### 成本估算公式
```
总成本 = (输入 Token × 输入单价) + (输出 Token × 输出单价)
```

**真实案例**：
- GPT-3.5-turbo：输入 $0.0005/1k，输出 $0.0015/1k
- GPT-4o：输入 $5/1M，输出 $15/1M
- 一次对话 1k 输入 + 1k 输出：3.5 花 0.14 分，4o 花 1.4 分
- 差 10 倍！所以：简单任务用便宜模型，复杂任务才用贵的

#### 成本优化技巧
1. 模型分级：简单任务用 3.5，复杂才用 4
2. 控制历史：对话不要带太多历史，自动截断
3. 限制输出：max_tokens 不要设太大
4. 缓存回答：相同问题，直接返回缓存，不重复调用

## 🧠 晚上：MCP 协议 + 微调入门

### 6. MCP 模型控制协议入门（1小时）

#### 什么是 MCP？
**Model Control Protocol** - 模型控制协议
- 统一的大模型调用标准
- 不是某个公司的，是行业正在形成的标准
- 目标：写一次代码，能调用所有大模型

#### MCP 核心概念
1. **统一接口**：所有模型都是同一个调用方式
2. **能力发现**：自动检测这个模型支持什么能力（流式？函数调用？）
3. **中间件**：统一的日志、监控、限流、缓存，不用每个模型写一遍
4. **可观测性**：统一的指标，成本、延迟、成功率一目了然

### 7. 模型微调：什么时候需要？怎么做？（1小时）

#### 灵魂拷问：我需要微调吗？

| 场景 | 需要微调吗？ | 更好的方案 |
|------|-------------|-----------|
| 让模型模仿某种说话风格 | ❌ 不需要 | 系统提示词 + Few-shot |
| 让模型遵循特定格式 | ❌ 不需要 | 结构化输出 + 校验 |
| 学习大量私有知识 | ❌ 不需要 | RAG 检索增强生成 |
| 降低推理成本（用小模型达到大模型效果） | ✅ 需要 | 微调 |
| 超低延迟（<100ms） | ✅ 需要 | 微调小模型 |
| 极高准确率要求（99%+） | ✅ 需要 | 微调 + 人工校验 |

> **黄金原则**：能靠提示词解决的，绝不微调！

#### 微调的前提条件
1. 至少 **100 条** 高质量标注数据，越多越好
2. 已经把提示词优化到极致了，还是达不到要求
3. 数据格式完全统一，质量很高

#### 微调五步法
1. **数据准备**：整理对话格式的数据（问题 → 标准答案）
2. **数据清洗**：去重、去低质量、格式统一
3. **发起微调**：用 OpenAI 或其他平台的微调接口
4. **测试评估**：微调后的模型和原模型对比测试
5. **上线部署**：替换生产环境的模型
