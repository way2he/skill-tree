# Plan 与 Spec 功能对比

## 快速概览

| 维度 | Plan | Spec |
|------|------|------|
| **适用场景** | 中小型功能开发、模块级重构 | 大型系统级任务、从零搭建、大规模重构、高质量项目、多人协作、长期维护项目 |
| **生成文档** | 单一计划文档（`plan.md`） | 完整三阶段文档组：<br>- `spec.md`（需求大纲）<br>- `tasks.md`（任务列表）<br>- `checklist.md`（验收清单） |
| **文档位置** | `.trae/documents/` | `.trae/specs/`（按任务分组） |
| **特点** | 有目标、有边界，AI 生成计划后等待确认 | 流程完整，从需求对齐到任务执行再到结果检查 |
| **优势** | 通过文档与 AI 对齐需求，提高准确率 | 规范驱动，可作为项目知识资产长期留存 |
| **Token 消耗** | 较少 | 较多（值得） |

---

## 详细对比

### Plan 模式

**特点**：
- 调用方式：`/plan`
- 生成单一计划文档，包含目标、设计修改范围、步骤等
- 让 AI 开发有目标，有边界，用户知道它打算怎么做，能干预

**优势**：
- 通过文档与 AI 对齐需求，提高准确率

**适用场景**：
- 需求相对明确，需要一定计划但不需要过于详细的场景
- 中小型功能开发
- 模块级重构

---

### Spec 模式

**特点**：
- 调用方式：`/spec`
- 生成完整的三阶段文档组：
  1. **需求大纲**（`spec.md`）：整体描述需求、变更范围、场景等
  2. **任务文档**（`tasks.md`）：拆分具体任务，明确依赖关系
  3. **验收清单**（`checklist.md`）：功能检查列表，确保任务完成

**优势**：
- 流程完整，从需求对齐到任务执行再到结果检查，全流程规范驱动

**适用场景**：
- 从零搭建新系统/模块
- 大规模重构
- 多人协作项目
- 高质量/高稳定性项目的开发
- 需长期维护的项目

---

## 快速选择指南

| 场景 | 推荐模式 |
|------|----------|
| 小范围修改/简单脚本 | 普通模式 |
| 中型功能开发/模块重构 | Plan |
| 从零搭建新系统/模块 | Spec |
| 大规模重构/技术栈迁移 | Spec |
| 多人协作项目 | Spec |
| 高质量/高稳定性项目 | Spec |
| 需长期维护的项目 | Spec |

## 工作流程对比

### Plan 模式工作流程

```
1. 输入需求
   ↓
2. AI 分析并生成 plan.md
   ↓
3. 用户确认或修改计划
   ↓
4. 执行任务
```

### Spec 模式工作流程

```
1. 输入需求
   ↓
2. AI 生成 spec.md + tasks.md + checklist.md
   ↓
3. 用户确认或修改文档
   ↓
4. 执行任务（自动更新任务和清单状态）
   ↓
5. 验收检查
```

## 使用方式

### 启用 Plan

1. 在 SOLO 对话框输入 `/`，然后从列表中选择 Plan
2. 或直接输入 `/plan` 命令，然后按下回车键

### 启用 Spec

1. 在 SOLO 对话框输入 `/`，然后从列表中选择 Spec
2. 或直接输入 `/spec` 命令，然后按下回车键

## Vibe Coding vs Spec Coding

### Vibe Coding（凭感觉开发）
- 优点：上手快、反馈快、适合做小实验、很适合原型和灵感验证
- 缺点：不稳定、不可复用、难协作、难维护、容易越做越乱

### Spec Coding（规范开发）
- 思路：先把需求、约束、边界、验收标准写清楚，再让 AI 去实现
- 核心：Spec 就是给 AI 写的"开发施工图 + 验收合同"
- 比喻：Vibe Coding 像即兴弹琴，Spec Coding 像先写好总谱，再让乐手按谱演奏

---

## 一份好的 Spec 通常包含什么

1. **需求目标**：明确"要做什么"
   - 解决什么问题
   - 给谁用
   - 触发场景是什么
   - 最终希望达成什么结果

2. **行为要求**：明确"系统应该怎么做"
   - 在什么条件下
   - 系统应该执行什么动作
   - 出现异常时要怎么处理
   - 不允许发生什么情况

3. **验收标准**：明确"做到什么程度才算完成"
   - 哪些结果算通过
   - 哪些不算通过
   - 成功的判定标准是什么

---

## 总结

| 模式 | 定位 | 比喻 |
|------|------|------|
| **Plan** | 轻量级计划 | 有目标的旅行路线图 |
| **Spec** | 完整规范方案 | 建筑施工图 + 验收合同 |

**简单总结**：Plan 是轻量级计划，Spec 是完整的规范方案。如果不确定，想做高质量项目，直接用 Spec！
