# 第8章 成本模型与优化

## 概述

大语言模型 API 的成本是生产环境中不可忽视的核心指标。OpenAI 与 Anthropic 均采用 **Token 计费** 模式，但在模型分级、缓存策略、批量折扣等方面存在显著差异。本章将系统梳理两家厂商的定价体系，并提供可落地的成本优化策略与监控代码示例。

---

## OpenAI 定价模型

### 模型定价表（美元/百万 Token）

| 模型 | 输入 | 输出 |
|------|------|------|
| GPT-4o | $2.50 | $10.00 |
| GPT-4o-mini | $0.15 | $0.60 |
| GPT-5 | $10.00 | $30.00（估计） |
| GPT-5.2 | $15.00 | $60.00（估计） |
| text-embedding-3-small | $0.02 | - |
| text-embedding-3-large | $0.13 | - |
| whisper-1 | $0.006/分钟 | - |
| tts-1 | $15.00/百万字符 | - |
| dall-e-3 | $0.04-$0.12/张 | - |

> **说明**：GPT-5 及 GPT-5.2 的价格为基于公开信息的估计值，实际定价以 OpenAI 官方公告为准。

### Token 计费方式

OpenAI 的计费核心公式为：

```
总费用 = (input_tokens × 输入单价) + (output_tokens × 输出单价)
```

**关键机制：**

- **input_tokens + output_tokens**：分别计费，输出单价通常为输入单价的 4 倍左右。
- **Prompt Caching（缓存 Token 折扣）**：当请求中包含与之前请求相同的前缀时，缓存命中的 Token 享受 50% 折扣。缓存有效期为 5-10 分钟（模型不同有所差异），适用于多轮对话、系统提示复用等场景。
- **图片 Token 换算**：图片按分辨率折算为 Token 数量，例如一张 1024×1024 的图片约折算 1105 个 Token（low detail）或最高 85 个 Token 的 tile 块（high detail）。

## Anthropic 定价模型

### 模型定价表（美元/百万 Token）

| 模型 | 输入 | 输出 |
|------|------|------|
| Claude Opus 4/4.6/4.7 | $15.00 | $75.00 |
| Claude Sonnet 4/4.5/4.6 | $3.00 | $15.00 |
| Claude Haiku 3/3.5 | $0.25 | $1.25 |

> **说明**：同系列内不同小版本（如 Sonnet 4 与 Sonnet 4.5）价格一致，差异主要体现在能力与延迟上。

### Token 计费方式

Anthropic 同样采用输入 + 输出分别计费的模式，但提供了更细粒度的计费维度：

```
总费用 = (input_tokens × 输入单价) + (output_tokens × 输出单价)
       + (cache_write_tokens × 缓存写入单价)
       + (cache_read_tokens × 缓存读取单价)
       + (thinking_tokens × 输出单价)
```

**关键机制：**

- **input_tokens + output_tokens**：基础计费，与 OpenAI 一致。
- **Prompt Caching**：
  - **缓存写入**：首次缓存时收取额外费用，以 Sonnet 为例为 $1.25/百万 Token。
  - **缓存读取**：缓存命中后大幅降低成本，以 Sonnet 为例为 $0.10/百万 Token（相比正常输入价格 $3.00 降低了 96.7%）。
  - 缓存最小单位为 1024 Token，写入后有效期约 5 分钟。
- **Extended Thinking**：开启后模型会生成 `thinking` 内容块，thinking tokens 按输出 Token 价格计费。不需要深度推理时应关闭此功能以节省成本。

## 定价对比表

下表对同级别模型进行横向价格对比（单位：美元/百万 Token）：

| 定位 | OpenAI 模型 | 输入 / 输出 | Anthropic 模型 | 输入 / 输出 |
|------|-------------|-------------|----------------|-------------|
| **旗舰** | GPT-5.2 | $15.00 / $60.00 | Claude Opus 4.7 | $15.00 / $75.00 |
| **高端** | GPT-5 | $10.00 / $30.00 | Claude Opus 4 | $15.00 / $75.00 |
| **中端** | GPT-4o | $2.50 / $10.00 | Claude Sonnet 4.5 | $3.00 / $15.00 |
| **轻量** | GPT-4o-mini | $0.15 / $0.60 | Claude Haiku 3.5 | $0.25 / $1.25 |

**对比要点：**

1. **旗舰级**：GPT-5.2 的输出价格（$60）显著低于 Claude Opus 4.7（$75），输入价格持平。
2. **中端**：GPT-4o 在输入和输出价格上均低于 Claude Sonnet 4.5，性价比更高。
3. **轻量级**：GPT-4o-mini 的价格约为 Claude Haiku 3.5 的 60%，是成本敏感场景的首选。
4. **缓存优势**：Anthropic 的缓存读取价格（$0.10）远低于 OpenAI 的 50% 折扣，在大量重复前缀场景下 Anthropic 成本优势明显。

## 成本优化策略

### 通用策略

| 策略 | 说明 | 预期节省 |
|------|------|----------|
| **选择合适模型** | 简单分类、摘要用轻量模型（GPT-4o-mini / Haiku），复杂推理用高端模型 | 50%-90% |
| **控制 max_tokens** | 设置合理的 `max_tokens` / `max_output_tokens` 上限，避免生成过长响应 | 20%-40% |
| **Prompt 压缩与精简** | 移除冗余指令，使用简洁表述；对长文档先做摘要再输入 | 30%-60% |
| **系统提示复用** | 将固定系统提示放在请求前缀，利用缓存机制降低重复计费 | 20%-50% |
| **语义缓存** | 对相同或相似的用户输入做应用层缓存，直接返回历史结果 | 可达 80%+ |

### OpenAI 特有

**1. Batches API（批量处理）**

- 支持 24 小时和 168 小时（7 天）两个处理窗口。
- 批量请求享受 **50% 折扣**。
- 适用于非实时场景：数据标注、批量内容生成、日志分析等。
- 单次批量请求最多支持 50,000 个独立请求。

```python
from openai import OpenAI

client = OpenAI()

# 创建批量任务
batch_file = client.files.create(
    file=open("batch_requests.jsonl", "rb"),
    purpose="batch"
)

batch = client.batches.create(
    input_file_id=batch_file.id,
    endpoint="/v1/chat/completions",
    completion_window="24h"  # 或 "168h"
)
```

**2. Prompt Caching**

- 缓存命中 Token 享受 50% 折扣。
- 系统提示保持不变时自动触发缓存。
- 适用于多轮对话、工具定义复用等场景。

### Anthropic 特有

**1. Prompt Caching（提示缓存）**

- 缓存读取成本仅为正常输入的 **3.3%**（以 Sonnet 为例：$0.10 vs $3.00）。
- 需要在 API 请求中显式标记缓存断点（`cache_control` 参数）。
- 特别适合长系统提示 + 多轮对话场景，成本可降低 90% 以上。

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "你是一个专业的技术文档助手...(很长的系统提示)",
            "cache_control": {"type": "ephemeral"}  # 标记缓存断点
        }
    ],
    messages=[{"role": "user", "content": "请解释什么是 Prompt Caching"}]
)
```

**2. Extended Thinking 按需开启**

- 开启 Extended Thinking 后，thinking tokens 按输出价格计费。
- 对于简单任务（如格式转换、信息提取），应关闭此功能。
- 仅在需要复杂推理（如数学证明、多步逻辑链）时开启。

```python
# 不需要深度推理时，不要设置 thinking 参数
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "将以下文本翻译为英文"}]
)

# 需要深度推理时，显式开启
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000  # 限制 thinking token 数量
    },
    messages=[{"role": "user", "content": "请证明勾股定理"}]
)
```

## 成本监控代码示例

以下代码展示如何从 API 响应中提取 `usage` 信息并计算实际成本。

### OpenAI 成本计算

```python
"""
OpenAI API 成本计算工具模块
从 API 响应中提取 token 用量并计算费用
"""

from dataclasses import dataclass


@dataclass
class OpenAIModelPricing:
    """OpenAI 模型定价配置"""
    input_price_per_million: float   # 输入价格（美元/百万Token）
    output_price_per_million: float  # 输出价格（美元/百万Token）
    cache_read_discount: float = 0.5  # 缓存读取折扣比例


# OpenAI 模型定价表
OPENAI_PRICING: dict[str, OpenAIModelPricing] = {
    "gpt-4o": OpenAIModelPricing(2.50, 10.00),
    "gpt-4o-mini": OpenAIModelPricing(0.15, 0.60),
    "gpt-5": OpenAIModelPricing(10.00, 30.00),
    "gpt-5.2": OpenAIModelPricing(15.00, 60.00),
}


def calculate_openai_cost(
    model: str,
    prompt_tokens: int,
    completion_tokens: int,
    cached_tokens: int = 0
) -> dict[str, float]:
    """
    计算 OpenAI API 调用成本

    Args:
        model: 模型名称，如 "gpt-4o"
        prompt_tokens: 输入 prompt 的 token 数量
        completion_tokens: 模型生成的 token 数量
        cached_tokens: 缓存命中的 token 数量

    Returns:
        包含各项费用明细的字典，键包括 input_cost、output_cost、
        cache_savings、total_cost

    Raises:
        ValueError: 当模型名称不在定价表中时抛出
        TypeError: 当 token 数量参数类型不正确时抛出
    """
    # 参数类型校验
    if not isinstance(prompt_tokens, int) or not isinstance(completion_tokens, int):
        raise TypeError("prompt_tokens 和 completion_tokens 必须为整数")

    if not isinstance(cached_tokens, int):
        raise TypeError("cached_tokens 必须为整数")

    if prompt_tokens < 0 or completion_tokens < 0 or cached_tokens < 0:
        raise ValueError("token 数量不能为负数")

    if cached_tokens > prompt_tokens:
        raise ValueError("缓存命中 token 数不能超过 prompt token 数")

    if model not in OPENAI_PRICING:
        raise ValueError(f"未知模型: {model}，可用模型: {list(OPENAI_PRICING.keys())}")

    pricing = OPENAI_PRICING[model]

    # 计算输入成本（缓存部分享受折扣）
    uncached_input = prompt_tokens - cached_tokens
    input_cost = (
        (uncached_input * pricing.input_price_per_million / 1_000_000)
        + (cached_tokens * pricing.input_price_per_million * pricing.cache_read_discount / 1_000_000)
    )

    # 计算输出成本
    output_cost = completion_tokens * pricing.output_price_per_million / 1_000_000

    # 计算缓存节省金额
    cache_savings = cached_tokens * pricing.input_price_per_million * (1 - pricing.cache_read_discount) / 1_000_000

    return {
        "input_cost": round(input_cost, 6),
        "output_cost": round(output_cost, 6),
        "cache_savings": round(cache_savings, 6),
        "total_cost": round(input_cost + output_cost, 6),
    }


def extract_openai_usage(response: object) -> dict[str, int]:
    """
    从 OpenAI API 响应对象中提取 token 用量信息

    Args:
        response: OpenAI API 返回的 ChatCompletion 对象

    Returns:
        包含 prompt_tokens、completion_tokens、total_tokens 的字典
    """
    usage = response.usage
    return {
        "prompt_tokens": usage.prompt_tokens,
        "completion_tokens": usage.completion_tokens,
        "total_tokens": usage.total_tokens,
    }
```

### Anthropic 成本计算

```python
"""
Anthropic API 成本计算工具模块
从 API 响应中提取 token 用量并计算费用，支持缓存和 Extended Thinking
"""

from dataclasses import dataclass


@dataclass
class AnthropicModelPricing:
    """Anthropic 模型定价配置"""
    input_price_per_million: float    # 输入价格
    output_price_per_million: float   # 输出价格
    cache_write_price_per_million: float  # 缓存写入价格
    cache_read_price_per_million: float   # 缓存读取价格


# Anthropic 模型定价表
ANTHROPIC_PRICING: dict[str, AnthropicModelPricing] = {
    "claude-opus-4-20250514": AnthropicModelPricing(15.00, 75.00, 18.75, 1.50),
    "claude-sonnet-4-20250514": AnthropicModelPricing(3.00, 15.00, 3.75, 0.30),
    "claude-haiku-3-5-20241022": AnthropicModelPricing(0.25, 1.25, 0.30, 0.03),
}


def calculate_anthropic_cost(
    model: str,
    input_tokens: int,
    output_tokens: int,
    cache_creation_input_tokens: int = 0,
    cache_read_input_tokens: int = 0,
    thinking_tokens: int = 0
) -> dict[str, float]:
    """
    计算 Anthropic API 调用成本

    Args:
        model: 模型名称
        input_tokens: 输入 token 数量
        output_tokens: 输出 token 数量（不含 thinking）
        cache_creation_input_tokens: 缓存写入 token 数量
        cache_read_input_tokens: 缓存读取 token 数量
        thinking_tokens: Extended Thinking 产生的 token 数量

    Returns:
        包含各项费用明细的字典

    Raises:
        ValueError: 当模型未知或 token 数量不合法时抛出
        TypeError: 当参数类型不正确时抛出
    """
    # 参数类型校验
    int_params = {
        "input_tokens": input_tokens,
        "output_tokens": output_tokens,
        "cache_creation_input_tokens": cache_creation_input_tokens,
        "cache_read_input_tokens": cache_read_input_tokens,
        "thinking_tokens": thinking_tokens,
    }
    for name, value in int_params.items():
        if not isinstance(value, int):
            raise TypeError(f"{name} 必须为整数")
        if value < 0:
            raise ValueError(f"{name} 不能为负数")

    if model not in ANTHROPIC_PRICING:
        raise ValueError(f"未知模型: {model}，可用模型: {list(ANTHROPIC_PRICING.keys())}")

    pricing = ANTHROPIC_PRICING[model]

    # 基础输入成本（扣除缓存部分）
    base_input_tokens = input_tokens - cache_creation_input_tokens - cache_read_input_tokens
    if base_input_tokens < 0:
        raise ValueError("缓存 token 总数不能超过输入 token 总数")

    base_input_cost = base_input_tokens * pricing.input_price_per_million / 1_000_000

    # 缓存写入成本
    cache_write_cost = cache_creation_input_tokens * pricing.cache_write_price_per_million / 1_000_000

    # 缓存读取成本
    cache_read_cost = cache_read_input_tokens * pricing.cache_read_price_per_million / 1_000_000

    # 输出成本
    output_cost = output_tokens * pricing.output_price_per_million / 1_000_000

    # Thinking 成本（按输出价格计费）
    thinking_cost = thinking_tokens * pricing.output_price_per_million / 1_000_000

    # 缓存节省金额（相比不使用缓存的差额）
    cache_savings = (
        cache_read_input_tokens * (pricing.input_price_per_million - pricing.cache_read_price_per_million) / 1_000_000
    )

    total = base_input_cost + cache_write_cost + cache_read_cost + output_cost + thinking_cost

    return {
        "base_input_cost": round(base_input_cost, 6),
        "cache_write_cost": round(cache_write_cost, 6),
        "cache_read_cost": round(cache_read_cost, 6),
        "output_cost": round(output_cost, 6),
        "thinking_cost": round(thinking_cost, 6),
        "cache_savings": round(cache_savings, 6),
        "total_cost": round(total, 6),
    }


def extract_anthropic_usage(response: object) -> dict[str, int]:
    """
    从 Anthropic API 响应对象中提取 token 用量信息

    Args:
        response: Anthropic API 返回的 Message 对象

    Returns:
        包含各维度 token 用量的字典
    """
    usage = response.usage
    return {
        "input_tokens": usage.input_tokens,
        "output_tokens": usage.output_tokens,
        "cache_creation_input_tokens": getattr(usage, "cache_creation_input_tokens", 0),
        "cache_read_input_tokens": getattr(usage, "cache_read_input_tokens", 0),
    }
```

### 综合使用示例

```python
"""
成本监控综合使用示例
演示如何在生产环境中追踪 API 调用成本
"""
import json
from datetime import datetime


class CostTracker:
    """API 调用成本追踪器"""

    def __init__(self) -> None:
        self.records: list[dict] = []

    def track_openai(
        self,
        model: str,
        response: object,
        request_id: str = ""
    ) -> dict[str, float]:
        """
        追踪一次 OpenAI API 调用的成本

        Args:
            model: 使用的模型名称
            response: API 响应对象
            request_id: 可选的请求标识符

        Returns:
            本次调用的成本明细
        """
        from openai_vs_anthropic_cost import (
            extract_openai_usage,
            calculate_openai_cost,
        )

        usage = extract_openai_usage(response)
        cost = calculate_openai_cost(
            model=model,
            prompt_tokens=usage["prompt_tokens"],
            completion_tokens=usage["completion_tokens"],
        )

        # 记录调用信息
        self.records.append({
            "timestamp": datetime.now().isoformat(),
            "provider": "openai",
            "model": model,
            "request_id": request_id,
            "usage": usage,
            "cost": cost,
        })

        return cost

    def track_anthropic(
        self,
        model: str,
        response: object,
        request_id: str = ""
    ) -> dict[str, float]:
        """
        追踪一次 Anthropic API 调用的成本

        Args:
            model: 使用的模型名称
            response: API 响应对象
            request_id: 可选的请求标识符

        Returns:
            本次调用的成本明细
        """
        from openai_vs_anthropic_cost import (
            extract_anthropic_usage,
            calculate_anthropic_cost,
        )

        usage = extract_anthropic_usage(response)
        cost = calculate_anthropic_cost(
            model=model,
            input_tokens=usage["input_tokens"],
            output_tokens=usage["output_tokens"],
            cache_creation_input_tokens=usage["cache_creation_input_tokens"],
            cache_read_input_tokens=usage["cache_read_input_tokens"],
        )

        # 记录调用信息
        self.records.append({
            "timestamp": datetime.now().isoformat(),
            "provider": "anthropic",
            "model": model,
            "request_id": request_id,
            "usage": usage,
            "cost": cost,
        })

        return cost

    def get_summary(self) -> dict[str, float]:
        """
        获取累计成本汇总

        Returns:
            包含总成本、OpenAI 成本、Anthropic 成本的字典
        """
        total = sum(r["cost"]["total_cost"] for r in self.records)
        openai_total = sum(
            r["cost"]["total_cost"] for r in self.records if r["provider"] == "openai"
        )
        anthropic_total = sum(
            r["cost"]["total_cost"] for r in self.records if r["provider"] == "anthropic"
        )

        return {
            "total_calls": len(self.records),
            "total_cost": round(total, 4),
            "openai_cost": round(openai_total, 4),
            "anthropic_cost": round(anthropic_total, 4),
        }

    def export_report(self, filepath: str) -> None:
        """
        导出成本报告为 JSON 文件

        Args:
            filepath: 输出文件路径
        """
        report = {
            "generated_at": datetime.now().isoformat(),
            "summary": self.get_summary(),
            "records": self.records,
        }
        with open(filepath, "w", encoding="utf-8") as f:
            json.dump(report, f, ensure_ascii=False, indent=2)
```

## 要点总结

1. **计费模式一致**：OpenAI 和 Anthropic 均采用 input + output 分别计费的 Token 模式，但 Anthropic 提供更细粒度的缓存读写分离计费。
2. **模型选择是最大的成本杠杆**：轻量模型（GPT-4o-mini / Claude Haiku）与旗舰模型（GPT-5.2 / Claude Opus）的价格差距可达 **100 倍**，应根据任务复杂度合理选型。
3. **缓存是降低重复成本的关键**：Anthropic 的缓存读取价格仅为正常输入的 3.3%，在长系统提示 + 多轮对话场景下优势巨大；OpenAI 的缓存折扣为 50%。
4. **批量处理适合离线场景**：OpenAI Batches API 提供 50% 折扣，适合非实时的批量任务。
5. **Extended Thinking 需按需开启**：Anthropic 的 thinking tokens 按输出价格计费，简单任务关闭后可显著降低成本。
6. **持续监控是基础**：通过 `usage` 字段追踪每次调用的 Token 消耗，建立成本基线，及时发现异常用量。
