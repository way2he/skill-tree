# 第6章：多模态能力

## 概述

多模态能力是大语言模型从"纯文本"走向"全感官"的关键里程碑。OpenAI 和 Anthropic 在多模态方向上各有侧重：OpenAI 覆盖面更广，从图片理解、图片生成、语音识别、语音合成到视频理解形成了完整闭环；Anthropic 则在 PDF 理解和 Computer Use（计算机操控）等特色能力上独树一帜。

本章将从 API 调用方式、代码示例和实际应用场景三个维度，对两家厂商的多模态能力进行全面对比。

## OpenAI 多模态能力

### 图片理解（Vision）

OpenAI 提供两种方式传入图片，分别对应 Responses API 和 Chat Completions API。

#### Responses API — input_image 类型

在 Responses API 中，图片通过 `input_image` 类型直接传入：

```python
from openai import OpenAI

client = OpenAI()


def analyze_image_with_responses_api(
    image_url: str,
    prompt: str = "请描述这张图片的内容",
    model: str = "gpt-4o"
) -> str:
    """
    使用 Responses API 进行图片理解。

    Args:
        image_url: 图片的 URL 地址或 Base64 编码（data:image/png;base64,...）
        prompt: 对图片的提问
        model: 使用的模型名称

    Returns:
        模型对图片内容的描述文本
    """
    response = client.responses.create(
        model=model,
        input=[
            {"role": "user", "content": prompt},
            {
                "role": "user",
                "content": [
                    {
                        "type": "input_image",
                        "image_url": image_url  # 支持 URL 和 Base64 data URI
                    }
                ]
            }
        ]
    )
    return response.output_text


# 使用示例
result = analyze_image_with_responses_api(
    image_url="https://example.com/photo.jpg",
    prompt="这张图片中有什么？"
)
print(result)
```

#### Chat Completions API — image_url 类型

在 Chat Completions API 中，图片通过 `image_url` 类型传入 `content` 列表：

```python
from openai import OpenAI

client = OpenAI()


def analyze_image_with_chat_api(
    image_url: str,
    prompt: str = "请描述这张图片的内容",
    model: str = "gpt-4o"
) -> str:
    """
    使用 Chat Completions API 进行图片理解。

    Args:
        image_url: 图片的 URL 地址或 Base64 编码（data:image/png;base64,...）
        prompt: 对图片的提问
        model: 使用的模型名称

    Returns:
        模型对图片内容的描述文本
    """
    response = client.chat.completions.create(
        model=model,
        messages=[
            {
                "role": "user",
                "content": [
                    {"type": "text", "text": prompt},
                    {
                        "type": "image_url",
                        "image_url": {
                            # 支持 URL 和 Base64 两种方式
                            "url": image_url,
                            # 可选：指定图片细节级别
                            # "detail": "low"  # "low" | "high" | "auto"
                        }
                    }
                ]
            }
        ]
    )
    return response.choices[0].message.content


# 使用 URL 方式
result = analyze_image_with_chat_api(
    image_url="https://example.com/photo.jpg",
    prompt="请分析这张图片中的文字内容"
)
print(result)

# 使用 Base64 方式
import base64


def image_to_base64(image_path: str) -> str:
    """将本地图片文件转换为 Base64 data URI。"""
    with open(image_path, "rb") as f:
        image_data = base64.b64encode(f.read()).decode("utf-8")
    # 根据文件扩展名推断 MIME 类型
    ext = image_path.lower().split(".")[-1]
    mime_map = {"jpg": "jpeg", "jpeg": "jpeg", "png": "png", "gif": "gif", "webp": "webp"}
    mime_type = mime_map.get(ext, "png")
    return f"data:image/{mime_type};base64,{image_data}"


result = analyze_image_with_chat_api(
    image_url=image_to_base64("local_photo.png"),
    prompt="这张图片中有几个人？"
)
print(result)
```

**关键参数说明：**

| 参数 | 说明 |
|------|------|
| `detail: "high"` | 高清模式，使用更多 token 进行精细分析 |
| `detail: "low"` | 低分辨率模式，更快且更省 token |
| `detail: "auto"` | 自动判断（默认值） |

### 图片生成（DALL·E）

OpenAI 的 DALL·E 系列模型提供图片生成、编辑和变体创建能力。

```python
from openai import OpenAI
import base64
from pathlib import Path

client = OpenAI()


def generate_image(
    prompt: str,
    model: str = "dall-e-3",
    size: str = "1024x1024",
    quality: str = "standard",
    n: int = 1
) -> str:
    """
    使用 DALL·E 模型生成图片。

    Args:
        prompt: 图片描述文本
        model: 模型名称，支持 "dall-e-2" 和 "dall-e-3"
        size: 图片尺寸，dall-e-3 支持 1024x1024/1792x1024/1024x1792
        quality: 图片质量，"standard" 或 "hd"（仅 dall-e-3）
        n: 生成图片数量，dall-e-3 仅支持 n=1

    Returns:
        生成图片的 URL 地址
    """
    response = client.images.generate(
        model=model,
        prompt=prompt,
        size=size,
        quality=quality,
        n=n
    )
    return response.data[0].url


def edit_image(
    image_path: str,
    prompt: str,
    mask_path: str | None = None,
    model: str = "dall-e-2",
    size: str = "1024x1024"
) -> str:
    """
    使用 DALL·E 模型编辑图片（仅 dall-e-2 支持）。

    Args:
        image_path: 原始图片路径（PNG 格式，带透明通道）
        prompt: 编辑指令
        mask_path: 遮罩图片路径，透明区域为可编辑区域
        model: 模型名称，目前仅支持 "dall-e-2"
        size: 输出图片尺寸

    Returns:
        编辑后图片的 URL 地址
    """
    with open(image_path, "rb") as img_file, open(mask_path, "rb") as mask_file:
        response = client.images.edit(
            model=model,
            image=img_file,
            mask=mask_file,
            prompt=prompt,
            size=size
        )
    return response.data[0].url


def create_variation(
    image_path: str,
    model: str = "dall-e-2",
    n: int = 1,
    size: str = "1024x1024"
) -> str:
    """
    使用 DALL·E 模型创建图片变体（仅 dall-e-2 支持）。

    Args:
        image_path: 原始图片路径
        model: 模型名称，目前仅支持 "dall-e-2"
        n: 生成变体数量
        size: 输出图片尺寸

    Returns:
        变体图片的 URL 地址
    """
    with open(image_path, "rb") as img_file:
        response = client.images.create_variation(
            model=model,
            image=img_file,
            n=n,
            size=size
        )
    return response.data[0].url


# 使用示例
url = generate_image(
    prompt="一只戴着墨镜的猫在夏威夷海滩上冲浪，卡通风格",
    model="dall-e-3",
    size="1024x1024",
    quality="hd"
)
print(f"生成的图片 URL: {url}")
```

**模型对比：**

| 特性 | dall-e-2 | dall-e-3 |
|------|----------|----------|
| 最大分辨率 | 1024x1024 | 1792x1024 |
| 质量选项 | 仅标准 | standard / hd |
| 批量生成 | 支持 n=1-10 | 仅 n=1 |
| 图片编辑 | 支持 | 不支持 |
| 变体创建 | 支持 | 不支持 |
| 文字渲染 | 较弱 | 显著提升 |

### 音频处理

OpenAI 提供语音转文字（STT）和文字转语音（TTS）两种能力。

#### 语音转文字（Whisper）

```python
from openai import OpenAI
from pathlib import Path

client = OpenAI()


def transcribe_audio(
    audio_path: str,
    model: str = "whisper-1",
    language: str | None = None,
    response_format: str = "json"
) -> str:
    """
    使用 Whisper 模型将音频转录为文字。

    Args:
        audio_path: 音频文件路径（支持 mp3, mp4, mpeg, mpga, m4a, wav, webm）
        model: 模型名称
        language: 音频语言代码（如 "zh", "en"），None 表示自动检测
        response_format: 输出格式（json, text, srt, verbose_json, vtt）

    Returns:
        转录文本（json 格式时返回完整对象，text 格式时返回纯文本）
    """
    with open(audio_path, "rb") as audio_file:
        response = client.audio.transcriptions.create(
            model=model,
            file=audio_file,
            language=language,
            response_format=response_format
        )

    # 根据返回格式处理结果
    if response_format == "json":
        return response.text
    return str(response)


# 使用示例
text = transcribe_audio(
    audio_path="meeting_recording.mp3",
    language="zh",
    response_format="text"
)
print(f"转录结果: {text}")
```

#### 文字转语音（TTS）

```python
from openai import OpenAI
from pathlib import Path

client = OpenAI()


def text_to_speech(
    text: str,
    output_path: str,
    model: str = "tts-1",
    voice: str = "alloy",
    response_format: str = "mp3",
    speed: float = 1.0
) -> str:
    """
    使用 TTS 模型将文字转换为语音。

    Args:
        text: 要转换的文本（最大 4096 字符）
        output_path: 输出音频文件路径
        model: 模型名称，"tts-1"（快速）或 "tts-1-hd"（高质量）
        voice: 语音角色，可选值：
            - alloy: 中性、温和
            - echo: 回声、清晰
            - fable: 故事叙述风格
            - onyx: 深沉、成熟
            - nova: 年轻、活力
            - shimmer: 柔和、温暖
        response_format: 输出格式（mp3, opus, aac, flac, wav, pcm）
        speed: 语速，范围 0.25 到 4.0

    Returns:
        输出文件的绝对路径
    """
    response = client.audio.speech.create(
        model=model,
        voice=voice,
        input=text,
        response_format=response_format,
        speed=speed
    )

    # 将音频流写入文件
    output = Path(output_path)
    output.parent.mkdir(parents=True, exist_ok=True)
    with open(output, "wb") as f:
        for chunk in response.iter_bytes(chunk_size=4096):
            f.write(chunk)

    return str(output.resolve())


# 使用示例 — 6 种语音角色演示
voices = ["alloy", "echo", "fable", "onyx", "nova", "shimmer"]
for voice in voices:
    path = text_to_speech(
        text=f"你好，我是{voice}语音角色，很高兴认识你。",
        output_path=f"output_{voice}.mp3",
        voice=voice
    )
    print(f"已生成: {path}")
```

**TTS 模型对比：**

| 特性 | tts-1 | tts-1-hd |
|------|-------|----------|
| 延迟 | 更低 | 略高 |
| 音质 | 标准 | 更高 |
| 适用场景 | 实时对话 | 音频内容制作 |

## Anthropic 多模态能力

### 图片理解（Vision）

Anthropic 的 Claude 模型通过 `content` 列表中 `type="image"` 的消息块传入图片，`source` 字段包含图片的编码方式和数据。

```python
import anthropic
import base64

client = anthropic.Anthropic()


def analyze_image(
    image_path: str,
    prompt: str = "请描述这张图片的内容",
    model: str = "claude-sonnet-4-20250514",
    max_tokens: int = 1024
) -> str:
    """
    使用 Anthropic Claude 模型进行图片理解。

    Args:
        image_path: 本地图片文件路径
        prompt: 对图片的提问
        model: 模型名称
        max_tokens: 最大输出 token 数

    Returns:
        模型对图片内容的描述文本
    """
    # 读取图片并编码为 Base64
    with open(image_path, "rb") as img_file:
        image_data = base64.standard_b64encode(img_file.read()).decode("utf-8")

    # 根据文件扩展名推断 MIME 类型
    ext = image_path.lower().split(".")[-1]
    mime_map = {
        "jpg": "image/jpeg",
        "jpeg": "image/jpeg",
        "png": "image/png",
        "gif": "image/gif",
        "webp": "image/webp"
    }
    media_type = mime_map.get(ext, "image/png")

    # 调用 API
    message = client.messages.create(
        model=model,
        max_tokens=max_tokens,
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "source": {
                            "type": "base64",
                            "media_type": media_type,
                            "data": image_data
                        }
                    },
                    {
                        "type": "text",
                        "text": prompt
                    }
                ]
            }
        ]
    )
    return message.content[0].text


def analyze_image_from_url(
    image_url: str,
    prompt: str = "请描述这张图片的内容",
    model: str = "claude-sonnet-4-20250514",
    max_tokens: int = 1024
) -> str:
    """
    使用 Anthropic Claude 模型理解 URL 图片。

    Anthropic 也支持通过 URL 方式传入图片，
    此时 source.type 设为 "url"。

    Args:
        image_url: 图片的 URL 地址
        prompt: 对图片的提问
        model: 模型名称
        max_tokens: 最大输出 token 数

    Returns:
        模型对图片内容的描述文本
    """
    message = client.messages.create(
        model=model,
        max_tokens=max_tokens,
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "source": {
                            "type": "url",
                            "url": image_url
                        }
                    },
                    {
                        "type": "text",
                        "text": prompt
                    }
                ]
            }
        ]
    )
    return message.content[0].text


# 使用示例
result = analyze_image(
    image_path="chart.png",
    prompt="请分析这张图表中的数据趋势"
)
print(result)
```

**Anthropic 图片输入格式说明：**

| source.type | 说明 | 适用场景 |
|-------------|------|----------|
| `base64` | 图片以 Base64 编码传入 | 本地文件 |
| `url` | 图片以 URL 形式传入 | 网络图片 |

### PDF 理解

PDF 理解是 Anthropic 的独有能力，支持将 PDF 文件直接作为输入，模型可以阅读和理解 PDF 中的文字、表格、图片等内容。

```python
import anthropic
import base64

client = anthropic.Anthropic()


def analyze_pdf(
    pdf_path: str,
    prompt: str = "请总结这份 PDF 文档的主要内容",
    model: str = "claude-sonnet-4-20250514",
    max_tokens: int = 4096
) -> str:
    """
    使用 Anthropic Claude 模型理解 PDF 文档。

    这是 Anthropic 的独有功能，无需预先提取 PDF 文本，
    模型可以直接理解 PDF 的原始排版和内容。

    Args:
        pdf_path: 本地 PDF 文件路径
        prompt: 对文档的提问
        model: 模型名称
        max_tokens: 最大输出 token 数

    Returns:
        模型对 PDF 文档的分析文本
    """
    # 读取 PDF 并编码为 Base64
    with open(pdf_path, "rb") as pdf_file:
        pdf_data = base64.standard_b64encode(pdf_file.read()).decode("utf-8")

    # 调用 API，使用 type="document" 传入 PDF
    message = client.messages.create(
        model=model,
        max_tokens=max_tokens,
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "document",
                        "source": {
                            "type": "base64",
                            "media_type": "application/pdf",
                            "data": pdf_data
                        }
                    },
                    {
                        "type": "text",
                        "text": prompt
                    }
                ]
            }
        ]
    )
    return message.content[0].text


# 使用示例
summary = analyze_pdf(
    pdf_path="annual_report.pdf",
    prompt="请提取这份年报中的关键财务数据，并以表格形式呈现"
)
print(summary)
```

**PDF 理解的优势：**

- **保留原始排版**：模型能理解 PDF 中的表格、多栏布局等复杂排版
- **无需预处理**：不需要使用 PyPDF2、pdfplumber 等库预先提取文本
- **图文混合理解**：可以同时理解 PDF 中的文字和图片内容
- **大文档支持**：支持处理多页 PDF 文档

### Computer Use（计算机操控）

Computer Use 是 Anthropic 最具前瞻性的独有特色能力，允许 Claude 模型像人类一样操控计算机——移动鼠标、点击按钮、输入文字、截取屏幕等。

```python
import anthropic
import base64

client = anthropic.Anthropic()


def computer_use_demo(
    prompt: str,
    model: str = "claude-sonnet-4-20250514"
) -> str:
    """
    Computer Use 功能演示。

    Claude 通过 computer 工具操控计算机，包括：
    - 截取屏幕截图
    - 移动鼠标到指定坐标
    - 点击鼠标（左键/右键）
    - 键盘输入文字
    - 按下组合键（如 Ctrl+C）

    Args:
        prompt: 让模型执行的任务描述
        model: 支持 Computer Use 的模型名称

    Returns:
        模型的响应文本
    """
    # 定义 computer 工具
    tools = [
        {
            "name": "computer",
            "type": "computer_20250124",
            "display_width_px": 1920,
            "display_height_px": 1080,
        }
    ]

    # 调用 API
    response = client.messages.create(
        model=model,
        max_tokens=1024,
        tools=tools,
        messages=[
            {
                "role": "user",
                "content": prompt
            }
        ]
    )

    # 处理工具调用结果
    for content_block in response.content:
        if content_block.type == "text":
            print(f"Claude: {content_block.text}")
        elif content_block.type == "tool_use":
            print(f"工具调用: {content_block.name}")
            if content_block.name == "computer":
                action = content_block.input.get("action")
                print(f"  动作: {action}")
                if action == "screenshot":
                    print("  → 截取屏幕截图")
                elif action == "mouse_move":
                    print(f"  → 移动鼠标到 ({content_block.input.get('coordinate')})")
                elif action == "left_click":
                    print(f"  → 左键点击 ({content_block.input.get('coordinate')})")
                elif action == "type":
                    print(f"  → 输入文字: {content_block.input.get('text')}")
                elif action == "key":
                    print(f"  → 按下按键: {content_block.input.get('keys')}")
                elif action == "left_click_drag":
                    print(f"  → 拖拽 ({content_block.input.get('start_coordinate')} → {content_block.input.get('end_coordinate')})")

    return str(response)


# 使用示例
computer_use_demo(
    prompt="请打开浏览器，搜索 'Anthropic Claude'，然后截图告诉我搜索结果"
)
```

**Computer Use 支持的操作：**

| 操作 | 说明 |
|------|------|
| `screenshot` | 截取当前屏幕截图 |
| `mouse_move` | 移动鼠标到指定坐标 |
| `left_click` | 鼠标左键单击 |
| `right_click` | 鼠标右键单击 |
| `double_click` | 鼠标双击 |
| `left_click_drag` | 鼠标左键拖拽 |
| `type` | 键盘输入文字 |
| `key` | 按下按键或组合键 |
| `scroll` | 滚动鼠标滚轮 |
| `wait` | 等待指定时间 |

> **注意：** Computer Use 目前仍处于 Beta 阶段，实际使用时需要配合屏幕截图的循环处理逻辑，形成一个"观察-思考-行动"的闭环。生产环境中建议使用 Anthropic 官方提供的 `computer-use` 参考实现。

---

## 多模态能力对比表

| 对比维度 | OpenAI | Anthropic |
|----------|--------|-----------|
| **图片理解（Vision）** | 支持（GPT-4o、GPT-5 系列） | 支持（Claude 3.5+ 系列） |
| **图片生成** | 支持（DALL·E 2/3） | 不支持 |
| **音频转文字（STT）** | 支持（Whisper 模型） | 不支持 |
| **文字转语音（TTS）** | 支持（6 种语音角色） | 不支持 |
| **视频理解** | 支持（GPT-5 系列） | 不支持 |
| **PDF 理解** | 不支持（需预处理提取文本） | 支持（原生 PDF 输入） |
| **计算机操控** | 不支持 | 支持（Computer Use） |

**总结性评价：**

- **OpenAI** 在多模态的**广度**上领先，覆盖了文本、图片、音频、视频的生成与理解，形成了从输入到输出的完整多模态闭环，适合需要多种媒体处理能力的应用场景。
- **Anthropic** 在多模态的**深度**上有独特优势，PDF 原生理解和 Computer Use 是其差异化竞争力，特别适合文档处理、自动化测试、RPA（机器人流程自动化）等场景。

---

## 要点总结

1. **图片理解**是两家厂商都支持的基础多模态能力。OpenAI 通过 `image_url` / `input_image` 类型传入，Anthropic 通过 `type="image"` 的 `source` 块传入，两者都支持 URL 和 Base64 两种编码方式。

2. **OpenAI 独有**的生成类能力（DALL·E 图片生成、TTS 语音合成）和感知类能力（Whisper 语音识别、GPT-5 视频理解）构成了完整的多模态生态，适合需要丰富媒体处理的应用。

3. **Anthropic 独有**的 PDF 原生理解能力省去了繁琐的文档预处理步骤，模型可以直接理解 PDF 的原始排版，在文档分析场景中具有显著优势。

4. **Computer Use** 是 Anthropic 最具前瞻性的能力，让 AI 能够像人类一样操控计算机界面，为自动化、测试、RPA 等领域开辟了全新的可能性。

5. **选型建议**：如果应用需要"全能型"的多模态处理（图片生成、语音合成等），选择 OpenAI；如果应用侧重于文档理解或计算机自动化操控，选择 Anthropic。
