# 04 - 流式调用

## 概述

两家都基于 SSE（Server-Sent Events）实现流式输出，但在事件模型和抽象层次上有显著差异。

| 维度 | OpenAI | Anthropic |
|------|--------|-----------|
| 底层协议 | SSE（Server-Sent Events） | SSE（Server-Sent Events） |
| 抽象层次 | 提供两种 API（Responses / Chat Completions），事件模型统一 | 提供基础流式和上下文管理器流式两种写法 |
| 事件粒度 | 较粗，以"响应对象"为载体 | 较细，区分 message / content_block / delta 三层 |
| 累积文本 | 需手动拼接 | 上下文管理器模式提供 `stream.text` 自动累积 |

---

## OpenAI 流式调用

### Responses API 流式

Responses API 是 OpenAI 较新的 API，通过 `stream=True` 参数启用流式输出，返回 SSE 事件流，每个 event 有 `type` 字段标识事件类型。

**关键事件类型：**

| 事件 type | 说明 |
|-----------|------|
| `response.created` | 响应对象已创建 |
| `response.in_progress` | 响应处理中 |
| `response.output_item.added` | 输出项已添加 |
| `response.output_text.delta` | 文本增量（核心事件） |
| `response.output_text.done` | 文本输出完成 |
| `response.completed` | 整个响应完成 |

**同步示例：**

```python
from openai import OpenAI

client = OpenAI()


def stream_responses_sync() -> None:
    """使用 Responses API 进行同步流式调用。"""
    with client.responses.create(
        model="gpt-4o",
        input="请用三句话介绍流式输出的优势。",
        stream=True,
    ) as stream:
        for event in stream:
            # 只处理文本增量事件
            if event.type == "response.output_text.delta":
                print(event.delta, end="", flush=True)
    print()  # 换行


stream_responses_sync()
```

**异步示例：**

```python
import asyncio
from openai import AsyncOpenAI

client = AsyncOpenAI()


async def stream_responses_async() -> None:
    """使用 Responses API 进行异步流式调用。"""
    async with client.responses.create(
        model="gpt-4o",
        input="请用三句话介绍流式输出的优势。",
        stream=True,
    ) as stream:
        async for event in stream:
            if event.type == "response.output_text.delta":
                print(event.delta, end="", flush=True)
    print()


asyncio.run(stream_responses_async())
```

### Chat Completions API 流式

Chat Completions API 是 OpenAI 的经典接口，同样通过 `stream=True` 启用流式，但返回的是 chunk 迭代器，每个 chunk 的 `choices[0].delta.content` 是增量文本。

**同步示例：**

```python
from openai import OpenAI

client = OpenAI()


def stream_chat_completions_sync() -> None:
    """使用 Chat Completions API 进行同步流式调用。"""
    stream = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
        stream=True,
    )

    for chunk in stream:
        # 每个 chunk.choices[0].delta.content 是增量文本
        delta = chunk.choices[0].delta.content
        if delta is not None:
            print(delta, end="", flush=True)
    print()


stream_chat_completions_sync()
```

**异步示例：**

```python
import asyncio
from openai import AsyncOpenAI

client = AsyncOpenAI()


async def stream_chat_completions_async() -> None:
    """使用 Chat Completions API 进行异步流式调用。"""
    stream = await client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
        stream=True,
    )

    async for chunk in stream:
        delta = chunk.choices[0].delta.content
        if delta is not None:
            print(delta, end="", flush=True)
    print()


asyncio.run(stream_chat_completions_async())
```

## Anthropic 流式调用

### 基础流式（stream=True）

Anthropic 的基础流式通过 `stream=True` 参数启用，返回 `MessageStream` 对象。事件模型采用三层结构：`message` -> `content_block` -> `delta`，粒度比 OpenAI 更细。

**关键事件类型：**

| 事件 type | 说明 |
|-----------|------|
| `message_start` | 消息开始，包含模型、usage 等元信息 |
| `content_block_start` | 内容块开始（text / tool_use） |
| `content_block_delta` | 内容增量（`text_delta` 或 `input_json_delta`） |
| `content_block_stop` | 内容块结束 |
| `message_delta` | 消息级别增量（如 stop_reason） |
| `message_stop` | 消息结束 |

**同步示例：**

```python
import anthropic

client = anthropic.Anthropic()


def stream_basic_sync() -> None:
    """使用基础流式进行同步调用。"""
    with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
    ) as stream:
        for event in stream:
            # 只处理文本增量事件
            if event.type == "content_block_delta":
                # event.delta 是 TextDelta 或 InputJsonDelta
                if hasattr(event.delta, "text"):
                    print(event.delta.text, end="", flush=True)
    print()


stream_basic_sync()
```

**异步示例：**

```python
import asyncio
import anthropic

client = anthropic.AsyncAnthropic()


async def stream_basic_async() -> None:
    """使用基础流式进行异步调用。"""
    async with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
    ) as stream:
        async for event in stream:
            if event.type == "content_block_delta":
                if hasattr(event.delta, "text"):
                    print(event.delta.text, end="", flush=True)
    print()


asyncio.run(stream_basic_async())
```

### 上下文管理器流式（推荐）

Anthropic 推荐使用 `client.messages.stream()` 上下文管理器模式，它提供了更高级的便捷方法，无需手动处理底层事件。

**核心便捷属性/方法：**

| 属性/方法 | 说明 |
|-----------|------|
| `stream.text` | 自动累积的完整文本（实时更新） |
| `stream.get_final_message()` | 获取最终完整的 Message 对象 |
| `stream.get_final_text()` | 获取最终完整文本字符串 |
| `stream.current_message` | 当前消息对象（实时更新） |

**同步示例：**

```python
import anthropic

client = anthropic.Anthropic()


def stream_context_manager_sync() -> None:
    """使用上下文管理器流式进行同步调用（推荐方式）。"""
    with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
    ) as stream:
        # 方式一：使用 stream.text 自动累积文本
        for text in stream.text_stream:
            print(text, end="", flush=True)
        print()

        # 方式二：直接获取最终完整消息
        final_message = stream.get_final_message()
        print(f"\n[模型]: {final_message.model}")
        print(f"[输入 tokens]: {final_message.usage.input_tokens}")
        print(f"[输出 tokens]: {final_message.usage.output_tokens}")


stream_context_manager_sync()
```

**异步示例：**

```python
import asyncio
import anthropic

client = anthropic.AsyncAnthropic()


async def stream_context_manager_async() -> None:
    """使用上下文管理器流式进行异步调用（推荐方式）。"""
    async with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{"role": "user", "content": "请用三句话介绍流式输出的优势。"}],
    ) as stream:
        async for text in stream.text_stream:
            print(text, end="", flush=True)
        print()

        # 获取最终消息
        final_message = await stream.get_final_message()
        print(f"\n[停止原因]: {final_message.stop_reason}")


asyncio.run(stream_context_manager_async())
```

## 流式事件模型对比表

| 对比维度 | OpenAI | Anthropic |
|----------|--------|-----------|
| **触发方式** | `stream=True` 参数 | `stream=True` 参数 |
| **事件粒度** | 两层：response 级 + delta 级 | 三层：message -> content_block -> delta |
| **文本增量** | `event.delta`（Responses API）或 `chunk.choices[0].delta.content`（Chat API） | `event.delta.text` |
| **累积文本** | 需手动拼接所有 delta | `stream.text` 自动累积（上下文管理器模式） |
| **工具调用流式** | `response.function_call_arguments.delta` 事件 | `input_json_delta` 事件 |
| **异步支持** | `AsyncOpenAI` + `async for` | `AsyncAnthropic` + `async for` |
| **上下文管理器** | Responses API 支持 `with` 语句 | 基础流式和上下文管理器流式均支持 |
| **最终消息获取** | 需等待 `response.completed` 事件 | `stream.get_final_message()` |
| **事件类型数量** | ~10 种 | ~6 种（更精简） |

## 工具调用的流式处理

### OpenAI：流式工具调用

OpenAI 在 Responses API 中通过 `response.function_call_arguments.delta` 事件逐步传递工具调用的参数 JSON。

```python
from openai import OpenAI

client = OpenAI()


def stream_tool_call_openai() -> None:
    """OpenAI 流式工具调用示例。"""
    tools = [
        {
            "type": "function",
            "name": "get_weather",
            "description": "获取指定城市的天气信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称",
                    }
                },
                "required": ["city"],
            },
        }
    ]

    with client.responses.create(
        model="gpt-4o",
        input="北京今天天气怎么样？",
        tools=tools,
        stream=True,
    ) as stream:
        for event in stream:
            if event.type == "response.output_text.delta":
                print(event.delta, end="", flush=True)
            elif event.type == "response.function_call_arguments.delta":
                # 逐步接收工具调用参数
                print(f"\n[工具参数增量]: {event.delta}", end="", flush=True)
    print()


stream_tool_call_openai()
```

### Anthropic：流式工具调用

Anthropic 通过 `input_json_delta` 事件逐步传递工具调用的 JSON 参数，与文本增量使用相同的事件类型 `content_block_delta`，通过 delta 的类型区分。

```python
import anthropic
import json

client = anthropic.Anthropic()


def stream_tool_call_anthropic() -> None:
    """Anthropic 流式工具调用示例。"""
    tools = [
        {
            "name": "get_weather",
            "description": "获取指定城市的天气信息",
            "input_schema": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称",
                    }
                },
                "required": ["city"],
            },
        }
    ]

    tool_args = ""

    with client.messages.stream(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        tools=tools,
        messages=[{"role": "user", "content": "北京今天天气怎么样？"}],
    ) as stream:
        for event in stream:
            if event.type == "content_block_delta":
                if hasattr(event.delta, "text"):
                    # 文本增量
                    print(event.delta.text, end="", flush=True)
                elif hasattr(event.delta, "partial_json"):
                    # 工具调用参数增量
                    tool_args += event.delta.partial_json
                    print(f"\n[工具参数增量]: {event.delta.partial_json}", end="", flush=True)

    # 拼接完成后解析完整 JSON
    if tool_args:
        parsed_args = json.loads(tool_args)
        print(f"\n[完整工具参数]: {json.dumps(parsed_args, ensure_ascii=False)}")


stream_tool_call_anthropic()
```

### 工具调用流式对比

| 对比维度 | OpenAI | Anthropic |
|----------|--------|-----------|
| **事件类型** | 独立的 `response.function_call_arguments.delta` 事件 | 复用 `content_block_delta`，通过 `partial_json` 区分 |
| **参数格式** | 原始字符串增量 | JSON 片段增量（`partial_json`） |
| **工具名称获取** | `response.function_call_arguments.done` 或 `output_item` 事件 | `content_block_start` 事件中的 `name` 字段 |
| **参数拼接** | 需手动拼接字符串 | 需手动拼接 JSON 片段 |
| **解析时机** | 拼接完成后 `json.loads()` | 拼接完成后 `json.loads()` |

---

## 要点总结

1. **协议一致，模型不同**：两家都基于 SSE 协议，但 OpenAI 的事件粒度较粗（两层），Anthropic 更细（三层：message -> content_block -> delta）。

2. **Anthropic 的上下文管理器模式更友好**：`stream.text` 自动累积文本、`stream.get_final_message()` 直接获取最终消息，减少了手动拼接的样板代码。

3. **OpenAI 有两套流式 API**：Responses API（新）和 Chat Completions API（经典），事件模型略有不同，需注意区分。

4. **工具调用流式**：两家都支持逐步传递工具参数，但事件类型设计不同 -- OpenAI 使用独立事件类型，Anthropic 复用 `content_block_delta` 并通过 delta 子类型区分。

5. **异步支持**：两家都提供完整的异步 SDK（`AsyncOpenAI` / `AsyncAnthropic`），用法与同步版本高度一致，只需将 `for` 替换为 `async for`。

6. **选择建议**：
   - 追求简洁和高级抽象 -> Anthropic 的上下文管理器流式
   - 需要兼容旧代码 -> OpenAI 的 Chat Completions 流式
   - 使用最新特性 -> OpenAI 的 Responses API 流式
