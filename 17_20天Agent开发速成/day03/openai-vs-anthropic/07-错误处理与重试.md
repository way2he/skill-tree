# 第7章 错误处理与重试

## 概述

在生产环境中调用 LLM API，网络抖动、速率限制、服务端过载等异常是常态。一个健壮的应用必须具备完善的错误处理与重试机制。OpenAI 和 Anthropic 的 Python SDK 在错误处理方面采用了**几乎相同的异常层次结构**，均继承自统一的基类设计，并内置了自动重试功能。本章将从异常体系、自动重试、手动处理三个维度逐一对比，最后给出兼容两家的通用错误处理封装。

---

## OpenAI 错误处理

### 异常层次结构

OpenAI SDK 提供了一套层次分明的异常类，所有异常均位于 `openai` 命名空间下：

```
openai.APIError（基类）
├── openai.APIConnectionError
├── openai.RateLimitError（429）
├── openai.APIStatusError
│   ├── openai.BadRequestError（400）
│   ├── openai.AuthenticationError（401）
│   ├── openai.PermissionDeniedError（403）
│   ├── openai.NotFoundError（404）
│   ├── openai.ConflictError（409）
│   └── openai.UnprocessableEntityError（422）
└── openai.APIResponseValidationError
```

| 异常类 | 对应 HTTP 状态码 | 典型场景 |
|---|---|---|
| `APIConnectionError` | — | 网络不通、DNS 解析失败 |
| `RateLimitError` | 429 | 请求频率超过限额 |
| `BadRequestError` | 400 | 请求参数格式错误 |
| `AuthenticationError` | 401 | API Key 无效或过期 |
| `PermissionDeniedError` | 403 | 无权访问该资源 |
| `NotFoundError` | 404 | 模型或端点不存在 |
| `ConflictError` | 409 | 资源冲突 |
| `UnprocessableEntityError` | 422 | 请求语义正确但无法处理 |
| `APIResponseValidationError` | — | 响应体不符合预期 schema |

### 自动重试机制

OpenAI SDK 内置了自动重试功能，默认会对**连接错误**和**速率限制错误（429）**进行重试。

**核心参数：`max_retries`**

```python
from openai import OpenAI

# 默认 max_retries=2，即最多重试 2 次（共 3 次请求）
client = OpenAI(max_retries=2)

# 关闭自动重试
client_no_retry = OpenAI(max_retries=0)

# 增加重试次数（适用于高并发场景）
client_aggressive = OpenAI(max_retries=5)
```

**自定义重试逻辑**

可以通过 `default_headers` 或继承重写的方式自定义重试行为，但更推荐使用 `tenacity` 等第三方库实现精细控制：

```python
import openai
from tenacity import (
    retry,
    stop_after_attempt,
    wait_exponential,
    retry_if_exception_type,
)

client = openai.OpenAI(max_retries=0)  # 关闭内置重试，使用自定义逻辑


@retry(
    stop=stop_after_attempt(5),                          # 最多尝试 5 次
    wait=wait_exponential(multiplier=1, min=1, max=60),  # 指数退避：1s, 2s, 4s, 8s, 16s
    retry=retry_if_exception_type(openai.RateLimitError), # 仅对速率限制重试
)
def call_with_retry(prompt: str) -> str:
    """带自定义重试策略的 OpenAI 调用"""
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
    )
    return response.choices[0].message.content
```

### 手动错误处理最佳实践

**基础 try/except 模式**

```python
import openai

client = openai.OpenAI()


def safe_completion(prompt: str) -> str | None:
    """安全的补全调用，捕获各类异常并返回 None"""
    try:
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=[{"role": "user", "content": prompt}],
            timeout=30.0,  # 单次请求超时 30 秒
        )
        return response.choices[0].message.content

    except openai.AuthenticationError as e:
        # API Key 无效 —— 通常需要检查配置
        print(f"认证失败: {e.message}")
        return None

    except openai.RateLimitError as e:
        # 速率限制 —— 可读取 retry-after 头部决定等待时间
        retry_after = e.response.headers.get("retry-after")
        wait_seconds = int(retry_after) if retry_after else 5
        print(f"触发速率限制，建议等待 {wait_seconds} 秒后重试")
        return None

    except openai.BadRequestError as e:
        # 请求参数有误 —— 需要检查输入
        print(f"请求参数错误: {e.message}")
        return None

    except openai.APIConnectionError as e:
        # 网络连接问题
        print(f"网络连接失败: {e.message}")
        return None

    except openai.APIStatusError as e:
        # 其他 HTTP 状态码错误（兜底）
        print(f"API 错误 [{e.status_code}]: {e.message}")
        return None

    except openai.APIError as e:
        # SDK 层面的未知错误（最终兜底）
        print(f"未知错误: {e.message}")
        return None
```

**超时设置**

```python
import openai

client = OpenAI(
    timeout=60.0,       # 全局默认超时（秒）
    max_retries=2,
)

# 也可以在单次调用中覆盖超时
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}],
    timeout=10.0,  # 本次调用超时 10 秒
)
```

## Anthropic 错误处理

### 异常层次结构

Anthropic SDK 的异常体系与 OpenAI **结构完全一致**，仅命名空间不同（`anthropic` 替代 `openai`）：

```
anthropic.APIError（基类）
├── anthropic.APIConnectionError
├── anthropic.RateLimitError（429）
├── anthropic.APIStatusError
│   ├── anthropic.BadRequestError（400）
│   ├── anthropic.AuthenticationError（401）
│   ├── anthropic.PermissionDeniedError（403）
│   ├── anthropic.NotFoundError（404）
│   ├── anthropic.ConflictError（409）
│   └── anthropic.UnprocessableEntityError（422）
└── anthropic.APIResponseValidationError
```

> **设计渊源**：Anthropic 的 Python SDK 早期基于 OpenAI SDK 的代码库 fork 而来，因此两者在异常体系、重试机制、客户端配置等方面保持了高度一致。这对需要同时对接两家的开发者来说是一个利好。

### 自动重试机制

Anthropic SDK 同样内置了自动重试，默认行为与 OpenAI 一致。

**核心参数：`max_retries`**

```python
import anthropic

# 默认 max_retries=2
client = anthropic.Anthropic(max_retries=2)

# 关闭自动重试
client_no_retry = anthropic.Anthropic(max_retries=0)

# 增加重试次数
client_aggressive = anthropic.Anthropic(max_retries=5)
```

**使用 tenacity 自定义重试**

```python
import anthropic
from tenacity import (
    retry,
    stop_after_attempt,
    wait_exponential,
    retry_if_exception_type,
)

client = anthropic.Anthropic(max_retries=0)


@retry(
    stop=stop_after_attempt(5),
    wait=wait_exponential(multiplier=1, min=1, max=60),
    retry=retry_if_exception_type(anthropic.RateLimitError),
)
def call_with_retry(prompt: str) -> str:
    """带自定义重试策略的 Anthropic 调用"""
    message = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}],
    )
    return message.content[0].text
```

## 错误处理对比表

| 对比维度 | OpenAI | Anthropic |
|---|---|---|
| **异常类命名空间** | `openai.*` | `anthropic.*` |
| **异常层次结构** | 6 层（基类 → 连接/速率限制/状态码/验证） | 6 层（与 OpenAI 完全一致） |
| **默认重试次数** | `max_retries=2` | `max_retries=2` |
| **重试触发条件** | 连接错误 + 429 | 连接错误 + 429 |
| **超时配置方式** | `OpenAI(timeout=60.0)` 全局 + 单次覆盖 | `Anthropic(timeout=60.0)` 全局 + 单次覆盖 |
| **RateLimit 处理** | `retry-after` header | `retry-after` header |
| **错误响应体格式** | `error.message` + `error.code` + `error.param` | `error.message` + `error.type` |
| **流式错误处理** | 在迭代 stream 时捕获异常 | 在迭代 stream 时捕获异常 |
| **设计渊源** | 原创设计 | 基于 OpenAI SDK fork，后独立演进 |

> **核心结论**：两家 SDK 的错误处理 API 几乎 1:1 对应，唯一的实际差异在于命名空间和错误响应体的字段命名（`code` vs `type`）。这意味着编写兼容两家的错误处理代码非常容易。

## 通用错误处理模式

以下封装同时兼容 OpenAI 和 Anthropic，通过统一接口屏蔽底层差异：

```python
"""
通用 LLM 错误处理封装
同时兼容 OpenAI 和 Anthropic SDK
"""

from typing import Protocol, runtime_checkable
from dataclasses import dataclass
import logging

import openai
import anthropic

logger = logging.getLogger(__name__)


@dataclass
class LLMResult:
    """统一的 LLM 调用结果"""
    content: str
    model: str
    provider: str
    usage_input_tokens: int
    usage_output_tokens: int


@runtime_checkable
class LLMClient(Protocol):
    """LLM 客户端协议，OpenAI 和 Anthropic 客户端均满足"""

    def chat_completions_create(self, *args, **kwargs): ...
    def messages_create(self, *args, **kwargs): ...


def _extract_error_info(exc: Exception) -> str:
    """
    从异常中提取可读的错误信息

    Args:
        exc: 捕获的异常对象

    Returns:
        格式化的错误描述字符串
    """
    # 尝试获取 HTTP 状态码（仅 APIStatusError 及其子类有）
    status_code = getattr(exc, "status_code", None)
    message = getattr(exc, "message", str(exc))

    if status_code:
        return f"[HTTP {status_code}] {message}"
    return message


def _handle_rate_limit(exc: openai.RateLimitError | anthropic.RateLimitError) -> float:
    """
    处理速率限制错误，提取 retry-after 时间

    Args:
        exc: 速率限制异常

    Returns:
        建议等待的秒数
    """
    response = getattr(exc, "response", None)
    if response:
        retry_after = response.headers.get("retry-after")
        if retry_after:
            try:
                return float(retry_after)
            except ValueError:
                pass
    return 5.0  # 默认等待 5 秒


def safe_llm_call(
    prompt: str,
    client: LLMClient,
    *,
    model: str,
    max_tokens: int = 1024,
    timeout: float = 60.0,
    max_retries: int = 3,
) -> LLMResult | None:
    """
    通用的安全 LLM 调用函数，兼容 OpenAI 和 Anthropic

    根据客户端类型自动选择调用方式，统一处理异常和重试逻辑。

    Args:
        prompt: 用户输入的提示文本
        client: OpenAI 或 Anthropic 客户端实例
        model: 模型名称（如 "gpt-4o" 或 "claude-sonnet-4-20250514"）
        max_tokens: 最大生成 token 数
        timeout: 单次请求超时时间（秒）
        max_retries: 最大重试次数（不含首次请求）

    Returns:
        LLMResult 对象，调用失败时返回 None
    """
    # 判断客户端类型
    is_openai = isinstance(client, openai.OpenAI)
    is_anthropic = isinstance(client, anthropic.Anthropic)

    if not is_openai and not is_anthropic:
        raise TypeError("client 必须是 OpenAI 或 Anthropic 客户端实例")

    provider = "openai" if is_openai else "anthropic"

    for attempt in range(max_retries + 1):
        try:
            if is_openai:
                response = client.chat.completions.create(
                    model=model,
                    messages=[{"role": "user", "content": prompt}],
                    max_tokens=max_tokens,
                    timeout=timeout,
                )
                return LLMResult(
                    content=response.choices[0].message.content or "",
                    model=response.model,
                    provider=provider,
                    usage_input_tokens=response.usage.prompt_tokens if response.usage else 0,
                    usage_output_tokens=response.usage.completion_tokens if response.usage else 0,
                )
            else:
                message = client.messages.create(
                    model=model,
                    max_tokens=max_tokens,
                    messages=[{"role": "user", "content": prompt}],
                    timeout=timeout,
                )
                # 提取文本内容（ Anthropic 可能返回多个 content block）
                text = "".join(
                    block.text for block in message.content if block.type == "text"
                )
                return LLMResult(
                    content=text,
                    model=message.model,
                    provider=provider,
                    usage_input_tokens=message.usage.input_tokens if message.usage else 0,
                    usage_output_tokens=message.usage.output_tokens if message.usage else 0,
                )

        except (openai.RateLimitError, anthropic.RateLimitError) as e:
            if attempt < max_retries:
                wait = _handle_rate_limit(e)
                logger.warning(
                    "速率限制 (%s)，第 %d/%d 次重试，等待 %.1f 秒",
                    provider, attempt + 1, max_retries, wait,
                )
                import time
                time.sleep(wait)
                continue
            logger.error("速率限制，已达最大重试次数: %s", _extract_error_info(e))
            return None

        except (openai.AuthenticationError, anthropic.AuthenticationError) as e:
            logger.error("认证失败 [%s]: %s", provider, _extract_error_info(e))
            return None  # 认证错误不应重试

        except (openai.BadRequestError, anthropic.BadRequestError) as e:
            logger.error("请求参数错误 [%s]: %s", provider, _extract_error_info(e))
            return None  # 参数错误不应重试

        except (
            openai.APIConnectionError,
            anthropic.APIConnectionError,
        ) as e:
            if attempt < max_retries:
                wait = 2 ** attempt  # 指数退避：1s, 2s, 4s
                logger.warning(
                    "连接失败 (%s)，第 %d/%d 次重试，等待 %d 秒",
                    provider, attempt + 1, max_retries, wait,
                )
                import time
                time.sleep(wait)
                continue
            logger.error("连接失败，已达最大重试次数: %s", _extract_error_info(e))
            return None

        except (openai.APIStatusError, anthropic.APIStatusError) as e:
            logger.error("API 错误 [%s]: %s", provider, _extract_error_info(e))
            return None

        except (openai.APIError, anthropic.APIError) as e:
            logger.error("SDK 错误 [%s]: %s", provider, _extract_error_info(e))
            return None

    return None


# ---- 使用示例 ----

if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO)

    openai_client = openai.OpenAI(max_retries=0)
    anthropic_client = anthropic.Anthropic(max_retries=0)

    # 调用 OpenAI
    result = safe_llm_call(
        prompt="用一句话介绍 Python",
        client=openai_client,
        model="gpt-4o",
        max_retries=3,
    )
    if result:
        print(f"[{result.provider}] {result.content}")

    # 调用 Anthropic（接口完全一致，仅换 client 和 model）
    result = safe_llm_call(
        prompt="用一句话介绍 Python",
        client=anthropic_client,
        model="claude-sonnet-4-20250514",
        max_retries=3,
    )
    if result:
        print(f"[{result.provider}] {result.content}")
```

## 要点总结

1. **异常体系高度一致**：OpenAI 和 Anthropic 的异常类层次结构完全对应，仅命名空间不同（`openai.*` vs `anthropic.*`），这得益于 Anthropic SDK 早期基于 OpenAI SDK fork 的历史渊源。

2. **内置重试行为相同**：两者默认 `max_retries=2`，均对连接错误和 429 速率限制自动重试，且均支持通过 `timeout` 参数进行全局和单次超时配置。

3. **速率限制处理**：两家均通过 `retry-after` 响应头告知客户端应等待的时间，手动处理时应优先读取该头部而非使用固定等待时间。

4. **不应重试的错误**：认证错误（401）、参数错误（400）等属于"确定性失败"，重试不会改变结果，应直接向上层报告。

5. **通用封装可行**：由于 API 高度对称，编写同时兼容两家的错误处理封装成本极低，核心思路是利用 `isinstance` 判断客户端类型，按类型分发调用逻辑，统一捕获异常。

6. **推荐实践**：关闭 SDK 内置重试（`max_retries=0`），使用 `tenacity` 等库实现更精细的重试策略（如指数退避、按异常类型筛选），便于统一日志和监控。
