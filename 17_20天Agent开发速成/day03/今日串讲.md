## 今天学到了什么？

### 1. 流式输出与指数退避
- 流式输出不是什么高大上的东西，就是 SSE 技术，逐字返回提升用户体验
- 指数退避加 jitter 是分布式系统的经典思想，避免惊群效应
- 面试时能举个实际例子，比说 10 句概念都管用

### 2. 模型选型与成本
- 国产模型比 OpenAI 便宜 5-10 倍，大部分场景够用了
- Token 计费有坑，中文比英文贵 1.5 倍，输出比输入贵 2-3 倍
- 成本优化 5 大招：换模型、删废话、压缩上下文、限长度、缓存

### 3. 统一接口层设计
- 策略模式 + 工厂模式 + 适配器模式，完美解决多模型兼容问题
- 开闭原则：对扩展开放，对修改关闭
- 换模型应该是配置问题，不是代码问题

### 4. 评估指标
- 准确率是最坑人的指标，样本不均衡时完全没用
- 精确率 = 别误报，宁放过不杀错
- 召回率 = 别漏报，宁杀错不放过
- 没有绝对的好坏，看 FP 和 FN 哪个代价大

### 5. Agent 评估方案
- Agent 评估比普通分类器复杂多了
- 四个维度：正确性、工具使用、效率、用户体验
- 测试集 + 自动化评估 + Bad Case 复盘 + 持续迭代
- 没有量化评估，就没有优化方向！

---

## 印象最深的 3 个点

### 1. 指数退避的 jitter
以前只知道指数退避，不知道还要加随机抖动。现在明白了，不加 jitter 就是制造请求风暴。面试时这个点一问一个准，能答上来的肯定是真懂分布式的。

### 2. 准确率的陷阱
99.9% 准确率的模型可能完全没用！这个例子太经典了，面试官几乎必问。以后再也不敢只看准确率了。

### 3. Agent 评估的完整流程
以前觉得评估就是跑几个例子看看效果，现在知道了是一套完整的工程体系：测试集设计 → 指标体系 → 自动化评估 → Bad Case 复盘 → 持续迭代。

---

## 面试时我会怎么答？

### 问：怎么评估 Agent 效果？

我会这么答：
1. 先设计测试集，覆盖所有场景，每个场景 20-30 个用例
2. 从四个维度评估：正确性（精确率召回率）、工具使用（成功率）、效率（耗时Token）、用户体验（人工评分）
3. 写自动化评估脚本，每次发版都跑
4. Bad Case 分类复盘，针对性优化
5. 持续迭代，每个版本都要有评估报告，和上一个版本对比
