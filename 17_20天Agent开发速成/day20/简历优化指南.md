# 简历优化指南 📝

## 简历结构

1. **基本信息**：姓名、电话、邮箱、GitHub/博客/LinkedIn
2. **个人简介**：1-2句话，突出核心优势
3. **技能**：分点列出，不要太杂
4. **项目经历**：STAR法则，量化结果
5. **工作经历**（如果有）：同上
6. **教育经历**：学校、专业、时间
7. **其他**：证书、竞赛、开源贡献

## 项目经历怎么写？

**STAR法则**：
- Situation：背景
- Task：任务
- Action：你的行动（重点）
- Result：结果（量化！）

**不好的例子**：
```
做了一个RAG系统
```

**好的例子**：
```
项目：企业知识库RAG系统
- 背景：解决大模型幻觉问题，让回答基于企业内部文档
- 任务：从0到1搭建RAG系统，支持多格式文档
- 行动：
  * 实现文档切分（Token-based + Semantic）
  * 用Chroma做向量库，BGE Embedding
  * 实现重排序（Reranker）提升准确率
  * 用FastAPI做API，Docker部署
- 结果：
  * 答案准确率从65%提升到88%
  * 响应速度从3s降到0.8s
  * 上线后被3个部门使用
```

## 技巧

1. **量化结果**：用数字说话（百分比、速度、用户数...）
2. **动词开头**：实现、优化、搭建、设计...
3. **技术细节**：写具体用了什么（不是"用了Python"，是"用Python + FastAPI + Chroma"）
4. **匹配JD**：看职位要求，调整简历关键词
5. **1页原则**：应届生1页，工作经验丰富最多2页
6. **不要造假**：可以包装，但不能瞎编
