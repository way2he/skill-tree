# 模块划分 📦

## 模块清单

### 1. orchestrator.py（协调器）

**职责**：
- 接收用户输入
- 协调各个模块
- 管理对话状态
- 控制执行流程

**核心类**：
```python
class CodeInterpreterOrchestrator:
    def __init__(self, llm):
        self.llm = llm
        self.code_generator = CodeGenerator(llm)
        self.code_executor = CodeExecutor()
        self.result_interpreter = ResultInterpreter(llm)

    def run(self, user_query: str) -&gt; str:
        # 1. 生成代码
        code = self.code_generator.generate(user_query)
        # 2. 执行代码
        result = self.code_executor.execute(code)
        # 3. 解释结果
        interpretation = self.result_interpreter.interpret(result)
        return interpretation
```

---

### 2. code_generator.py（代码生成器）

**职责**：
- 理解用户需求
- 生成合适的代码
- 代码检查和优化

**核心类**：
```python
class CodeGenerator:
    def generate(self, user_query: str) -&gt; str:
        # 写个 prompt，让 LLM 生成代码
        prompt = self._build_prompt(user_query)
        code = self.llm.generate(prompt)
        return code
```

---

### 3. code_executor.py（代码执行器）

**职责**：
- 安全地执行代码
- 管理执行环境
- 捕获输出和错误

**核心类**：
```python
class CodeExecutor:
    def execute(self, code: str) -&gt; dict:
        # 在沙箱里执行代码
        # 捕获 stdout, stderr, 结果
        result = self._execute_in_sandbox(code)
        return result
```

---

### 4. result_interpreter.py（结果解释器）

**职责**：
- 理解执行结果
- 自然语言解释
- 可视化处理

**核心类**：
```python
class ResultInterpreter:
    def interpret(self, execution_result: dict) -&gt; str:
        # 用 LLM 解释执行结果
        interpretation = self.llm.interpret(execution_result)
        return interpretation
```
