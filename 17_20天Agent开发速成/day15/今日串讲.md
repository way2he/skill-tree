## 核心要点回顾

### 多 Agent 设计
```
专业分工：每个 Agent 有明确的角色和目标
任务编排：定义任务之间的依赖关系
冲突解决：投票、协商、仲裁
信息共享：共享记忆、共享状态
```

### 推理优化
- KV Cache：O(n²) → O(n)
- 量化：FP16 → INT8 → INT4
- vLLM：高吞吐推理

### 项目开发
- 先做需求分析
- 再做架构设计
- 模块划分清晰
- 接口定义明确
- 迭代开发
