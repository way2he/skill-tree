## 基础题（3 道）

### 第 1 题：Plan-and-Execute vs ReAct

**参考答案**：

#### P&E 核心思想
```
先规划，再执行
理解任务 → 制定计划 → 按步执行 → 检查调整 → 完成
```

#### P&E vs ReAct 对比

| 维度 | ReAct | Plan-and-Execute |
|------|-------|------------------|
| **方式** | 边想边做 | 先想后做 |
| **流程** | Thought→Action→Observation | Plan→Execute→Review→Adjust |
| **优势** | 灵活，适合探索 | 结构清晰，避免弯路 |
| **劣势** | 可能重复或走偏 | 计划可能过时 |
| **适用** | 探索性任务 | 复杂结构化任务 |

#### 适用场景
- ReAct：需要探索、信息不完整的任务
- P&E：结构清晰、步骤明确的任务

### 第 2 题：任务拆解

**参考答案**：

#### 拆解方法
1. **按步骤**：时间/逻辑顺序
   - 例：注册→登录→填写→提交
2. **按组件**：系统模块
   - 例：前端+后端+数据库+部署
3. **按抽象层次**：从粗到细
   - 例：设计→实现→测试→发布

#### 拆解原则
- **原子性**：每个子任务独立可执行
- **可验证**：有明确的成功标准
- **明确依赖**：标注前置依赖
- **适当粒度**：不要太粗或太细

#### 拆解器实现思路
```python
prompt = """
将任务拆解为子任务：
任务：{task}

输出格式：
- id: 序号
- description: 描述
- tool: 需要的工具
- depends_on: 依赖的前置任务
"""
```

### 第 3 题：多模态 Agent 架构

**参考答案**：

#### 多模态挑战
- **表示统一**：不同模态如何统一表示
- **信息融合**：如何整合多种模态的信息
- **对齐**：不同模态的时序/语义对齐
- **计算开销**：多模态通常计算量更大

#### 架构设计
```
用户输入 (文本/图像/语音)
    ↓
各模态单独编码 (Text/Image/Audio Encoder)
    ↓
多模态融合模块 (Fusion)
    ↓
统一理解与推理
    ↓
多模态生成模块 (Text/Image/Audio Generator)
    ↓
输出 (文本/图像/语音)
```

#### 示例
- 输入：一张猫的图片 + "这是什么动物？"
- 过程：图片编码 + 文本编码 → 融合 → 推理 → "这是一只猫"
- 输出：文本 + 可选的更多猫的图片

## 进阶题（3 道）

### 第 4 题：动态调整策略

**参考答案**：

#### 为什么需要调整
- 子任务执行失败
- 获得新信息，计划过时
- 发现更优路径
- 用户需求变更

#### 调整策略
| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **重试** | 重试失败步骤 | 临时/随机问题 |
| **替换** | 换一种方法 | 当前方法不合适 |
| **重排** | 调整步骤顺序 | 依赖关系变化 |
| **重规划** | 重新制定计划 | 重大变化 |

#### 实现思路
```
失败 → 分析原因 → 选择策略 → 应用调整 → 继续执行
```

### 第 5 题：异常处理

**参考答案**：

#### 常见失败类型
1. **工具调用失败**：API 错误、超时
2. **结果不符合预期**：成功但结果不对
3. **计划过时**：新信息使计划无效
4. **资源不足**：Token 超限、内存不足

#### 处理流程
```
失败
  ↓
记录错误
  ↓
分析原因
  ↓
是临时问题？→ 是 → 重试 (有上限)
  ↓ 否
可以换方法？→ 是 → 替换方法
  ↓ 否
需要重规划？→ 是 → 重规划
  ↓ 否
询问用户 / 优雅退出
```

### 第 6 题：评估指标

**参考答案**：

#### 评估维度
1. **任务完成度**：是否完成目标
2. **执行效率**：时间/步骤数/成本
3. **计划质量**：计划是否合理
4. **调整能力**：处理异常的能力

#### 评估指标
| 指标 | 说明 | 计算 |
|------|------|------|
| **成功率** | 任务成功的比例 | 成功数 / 总数 |
| **平均步骤数** | 完成任务的平均步数 | 总步数 / 成功数 |
| **调整次数** | 计划调整的频率 | 调整数 / 总数 |
| **完成时间** | 耗时 | - |
| **用户满意度** | 用户评分 | 人工评分 |
