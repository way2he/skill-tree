# 动态调整与多模态 🎯

&gt; 🎯 **学习目标**：实现动态调整和多模态能力

---

## 一、动态调整

### 1.1 为什么需要动态调整？

```
计划永远赶不上变化
```

常见情况：
- 子任务执行失败
- 获得新信息，需要修改计划
- 发现更优的执行路径
- 用户中途变更需求

---

### 1.2 调整策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **重试** | 重试失败的步骤 | 临时问题 |
| **替换** | 换一种方法执行 | 方法不合适 |
| **重排** | 调整步骤顺序 | 依赖变化 |
| **重规划** | 重新制定计划 | 重大变化 |

---

### 1.3 实现思路

```python
class AdjustmentEngine:
    def analyze_failure(self, state: PlanState, error: Exception) -> Adjustment:
        """分析失败原因"""
        prompt = f"""
计划执行失败：
当前状态：{state}
错误信息：{error}

请分析失败原因并给出调整建议：
1. 重试
2. 换方法
3. 调整顺序
4. 重规划
"""
        return self.llm(prompt)
    
    def apply_adjustment(self, plan: Plan, adjustment: Adjustment) -> Plan:
        """应用调整"""
        if adjustment.type == "replan":
            return self.replan(plan, state)
        elif adjustment.type == "reorder":
            return self.reorder(plan, adjustment)
        # ...
```

## 二、多模态能力

### 2.1 多模态是什么？

```
多模态 = 文本 + 图像 + 音频 + 视频 + ...
```

---

### 2.2 多模态 Agent 架构

```
用户输入 (文本/图像/语音)
    ↓
输入统一表示 (Embedding)
    ↓
多模态理解模块
    ↓
推理/决策
    ↓
多模态生成模块
    ↓
输出 (文本/图像/语音)
```

---

### 2.3 常见多模态任务

| 任务 | 说明 | 示例 |
|------|------|------|
| **图像理解** | 分析图像内容 | 这张图里有什么？ |
| **文本→图像** | 生成图像 | 画一只可爱的猫 |
| **视觉问答** | 图像+文本回答 | 图中的人在做什么？ |
| **语音交互** | ASR + TTS | 语音对话 |

---

### 2.4 实现要点

```python
class MultimodalAgent:
    def process(self, input_data: MultimodalInput) -> MultimodalOutput:
        """处理多模态输入"""
        
        # 1. 输入编码
        if input_data.text:
            text_emb = self.text_encoder(input_data.text)
        if input_data.image:
            image_emb = self.image_encoder(input_data.image)
        
        # 2. 多模态融合
        combined = self.fusion(text_emb, image_emb)
        
        # 3. 推理
        reasoning = self.llm(combined)
        
        # 4. 输出生成
        output = MultimodalOutput()
        if reasoning.needs_text:
            output.text = self.text_generator(reasoning)
        if reasoning.needs_image:
            output.image = self.image_generator(reasoning)
        
        return output
```

## 三、设计模式

### 3.1 失败处理流程

```
执行失败
    ↓
分析失败原因
    ↓
可以重试？→ 是 → 重试
    ↓ 否
换方法？→ 是 → 替换方法
    ↓ 否
需要重规划？→ 是 → 重规划
    ↓ 否
放弃/询问用户
```

---

### 3.2 多模态协调

```
多模态输入
    ↓
每个模态单独编码
    ↓
融合表示
    ↓
统一理解
    ↓
生成相应输出
```

---

## 四、面试关键点

1. **动态调整策略**：重试/替换/重排/重规划

3. **失败处理**：如何优雅处理执行失败
4. **评估**：如何评估 P&E 系统的好坏
