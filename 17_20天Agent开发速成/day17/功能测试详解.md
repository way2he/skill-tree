## 一、测试金字塔

### 金字塔结构

```
        /\
       /E2E\       端到端测试（少量）
      /------\
     / 集成测试 \    集成测试（适量）
    /------------\
   /  单元测试    \   单元测试（大量）
  ----------------
```

### 各层特点

| 层级 | 特点 | 比例 | 速度 | 成本 |
|------|------|------|------|------|
| **单元测试** | 测单个函数/类 | 70% | ⚡ 快 | 低 |
| **集成测试** | 测模块间协作 | 20% | 🐢 中 | 中 |
| **E2E测试** | 测完整用户流程 | 10% | 🐌 慢 | 高 |

## 二、pytest单元测试

### 1. 基础示例

```python
# calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为0")
    return a / b
```

```python
# test_calculator.py
import pytest
from calculator import add, subtract, multiply, divide

def test_add():
    assert add(1, 2) == 3
    assert add(0, 0) == 0
    assert add(-1, 1) == 0

def test_subtract():
    assert subtract(5, 3) == 2
    assert subtract(0, 5) == -5

def test_multiply():
    assert multiply(2, 3) == 6
    assert multiply(0, 5) == 0

def test_divide():
    assert divide(6, 2) == 3
    assert divide(5, 2) == 2.5
    
    with pytest.raises(ValueError):
        divide(1, 0)
```

**运行测试：**
```bash
pytest test_calculator.py -v
```

### 2. fixture使用

```python
import pytest

@pytest.fixture
def sample_list():
    return [1, 2, 3, 4, 5]

@pytest.fixture
def sample_dict():
    return {"a": 1, "b": 2, "c": 3}

def test_sum(sample_list):
    assert sum(sample_list) == 15

def test_len(sample_list, sample_dict):
    assert len(sample_list) == 5
    assert len(sample_dict) == 3
```

### 3. 参数化测试

```python
import pytest

@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
    (5, 5, 10),
])
def test_add_param(a, b, expected):
    assert add(a, b) == expected
```

## 三、面试要点

### 高频面试题

1. **测试金字塔是什么？**
   - 单元测试（底层，70%）：测单个函数，快，低成本
   - 集成测试（中层，20%）：测模块间交互
   - E2E测试（顶层，10%）：测完整流程，慢，高成本

2. **pytest的fixture是什么？**
   - 测试前置条件设置
   - 复用测试数据
   - 依赖注入

3. **如何测试异常？**
   - `with pytest.raises(ErrorType):`
