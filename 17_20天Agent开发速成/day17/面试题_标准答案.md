## 第 1 题答案

### 问题
请解释测试金字塔，以及每层的特点和比例？

### 标准答案

**测试金字塔：**

```
        /\
       /E2E\       端到端测试（10%）
      /------\
     / 集成测试 \    集成测试（20%）
    /------------\
   /  单元测试    \   单元测试（70%）
  ----------------
```

**各层对比：**

| 层级 | 特点 | 比例 | 速度 | 成本 |
|------|------|------|------|------|
| **单元测试** | 测单个函数/类，隔离 | 70% | ⚡ 快 | 低 |
| **集成测试** | 测模块间交互 | 20% | 🐢 中 | 中 |
| **E2E测试** | 测完整用户流程 | 10% | 🐌 慢 | 高 |

## 第 2 题答案

### 问题
异常处理有哪些策略？重试机制如何设计？

### 标准答案

**异常处理策略：**

| 策略 | 适用场景 |
|------|---------|
| **捕获并记录** | 非关键错误 |
| **重试 (Retry)** | 临时性错误（网络抖动、超时） |
| **熔断 (Circuit Breaker)** | 频繁错误，防止雪崩 |
| **降级 (Fallback)** | 不可用时提供备选方案 |

**重试机制设计要点：**
- 最大重试次数（如3次）
- 延迟策略（固定延迟 / 指数退避）
- 只对特定异常重试
- 重试后最终失败处理

**指数退避示例：**
- 第1次失败：等待 1 秒
- 第2次失败：等待 2 秒
- 第3次失败：等待 4 秒

## 第 3 题答案

### 问题
RAGAS的核心指标有哪些？分别衡量什么？

### 标准答案

**RAGAS的4大核心指标：**

| 指标 | 说明 | 分数范围 |
|------|------|---------|
| **Answer Relevancy** | 答案是否切题 | 0-1 |
| **Context Relevancy** | 检索的上下文是否相关 | 0-1 |
| **Faithfulness** | 回答是否基于上下文，不幻觉 | 0-1 |
| **Context Recall** | 关键信息是否都在上下文中 | 0-1 |

## 第 4 题答案

### 问题
性能优化有哪些常用方法？如何选择优化方向？

### 标准答案

**常用性能优化方法：**

1. **缓存**：减少重复计算/查询
2. **异步**：提高并发能力，不阻塞
3. **数据库优化**：索引、查询优化、分库分表
4. **算法优化**：降低时间复杂度（O(n²) → O(n)）
5. **CDN/静态资源**：加速静态内容

**如何选择优化方向：**
- 先测量，找到瓶颈（不要过早优化）
- 优先优化高影响、低成本的
- 用数据说话，优化后验证效果

## 第 5 题答案

### 问题
如何设计一个A/B测试？需要注意什么？

### 标准答案

**A/B测试步骤：**

1. 确定目标指标（点击率、转化率...）
2. 设计实验（样本量、流量分配、时长）
3. 开发上线（两个版本）
4. 收集数据（埋点）
5. 统计分析（显著性检验）
6. 决策（是否上线新方案）

**注意事项：**
- 样本量足够
- 随机分流
- 只改一个变量
- 实验时间足够长（避免新奇效应）
- 不偷看中期结果

## 第 6 题答案

### 问题
如何用pytest写单元测试？fixture是什么？

### 标准答案

**简单pytest示例：**

```python
# calculator.py
def add(a, b):
    return a + b
```

```python
# test_calculator.py
import pytest
from calculator import add

def test_add():
    assert add(1, 2) == 3
    assert add(0, 0) == 0
```

**fixture：**
- 测试前置条件设置
- 复用测试数据
- 依赖注入

**fixture示例：**

```python
@pytest.fixture
def sample_list():
    return [1, 2, 3, 4, 5]

def test_sum(sample_list):
    assert sum(sample_list) == 15
```
