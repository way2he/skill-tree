# 项目收尾清单 ✅

&gt; 🎯 **本章目标**：项目交付前的检查清单  
&gt; ⏰ **预计时间**：30 分钟

---

## 一、代码检查

- [ ] 所有代码已提交Git
- [ ] 代码已Review
- [ ] 没有遗留的TODO/FIXME
- [ ] 代码格式化完成
- [ ] 无敏感信息（密钥、密码）

---

## 二、测试检查

- [ ] 单元测试全部通过
- [ ] 集成测试全部通过
- [ ] 测试覆盖率达标（≥80%）
- [ ] 边界情况已测试
- [ ] 异常情况已测试

---

## 三、文档检查

- [ ] README.md完整
- [ ] 部署文档已写
- [ ] API文档已写
- [ ] 架构图已更新
- [ ] CHANGELOG已更新

---

## 四、部署检查

- [ ] 配置文件正确（区分环境）
- [ ] 依赖已锁定（requirements.txt / package.json）
- [ ] Docker镜像已构建
- [ ] 数据库迁移脚本已准备
- [ ] 回滚方案已准备

---

## 五、监控检查

- [ ] 日志已配置（级别、格式、收集）
- [ ] 监控指标已配置（CPU、内存、QPS、错误率）
- [ ] 告警已配置（阈值、通知渠道）
- [ ] 健康检查端点已配置
- [ ] 仪表盘已更新

---

## 六、安全检查

- [ ] 依赖漏洞已扫描
- [ ] 认证授权已测试
- [ ] 敏感数据已加密
- [ ] CORS配置正确
- [ ] SQL注入/XSS已防护

---

## 七、面试要点

### 高频面试题

1. **项目上线前需要检查什么？**
   - 代码：提交、Review、格式化
   - 测试：全部通过、覆盖够
   - 文档：README、部署文档
   - 部署：配置、依赖、回滚方案
   - 监控：日志、指标、告警
