# Day05 面试题标准答案 ✅

---

## 第 1 题答案

### 问题
请详细描述 RAG 的完整流程，以及每个环节的作用是什么？

### 标准答案

**RAG 完整流程（7 步）：**

```
1. 原始文档准备 → 2. 文档分块 → 3. 文本向量化 → 4. 向量存储
                                                    ↓
用户查询 → 5. 查询向量化 → 6. 相似度检索 → 7. 重排序 → 8. 增强 → 9. 生成回答
```

**每个环节的作用：**

| 步骤 | 作用 | 为什么需要 |
|------|------|-----------|
| **1. 原始文档准备** | 收集文档，统一格式 | 输入数据 |
| **2. 文档分块** | 把长文档切成小块 | 大模型上下文窗口有限 |
| **3. 文本向量化** | 文本 → 高维向量 | 计算机能理解语义 |
| **4. 向量存储** | 存到向量数据库 | 高效检索 |
| **5. 查询向量化** | 用户问题 → 向量 | 同上 |
| **6. 相似度检索** | 找最相似的 K 个文档 | 召回相关知识 |
| **7. 重排序** | 精排，提升准确率 | 向量检索是粗筛，不够准 |
| **8. 上下文增强** | 把文档填到提示词里 | 让大模型基于知识回答 |
| **9. 生成回答** | 大模型生成最终答案 | 输出 |

## 第 2 题答案

### 问题
向量数据库和传统关系型数据库（如 MySQL、PostgreSQL）有什么区别？

### 标准答案

| 对比项 | 传统数据库（MySQL） | 向量数据库（Chroma/Qdrant） |
|--------|---------------------|---------------------------|
| **存储内容** | 表格、JSON、文档 | 高维向量 + 元数据 |
| **查询方式** | SQL 精确匹配（`WHERE name="张三"`） | 相似度检索（`similar(vector, ...)`） |
| **索引类型** | B-Tree、Hash | HNSW、IVF、FAISS |
| **查询目标** | 精确匹配 | 语义相似 |
| **适用场景** | 用户数据、订单、业务数据 | RAG、语义搜索、推荐系统 |
| **例子** | "查 ID=123 的用户" | "查和'我喜欢苹果'最相似的文档" |

**一句话总结：**  
传统数据库是**精确匹配**，向量数据库是**语义相似度检索**。

## 第 3 题答案

### 问题
余弦相似度和欧氏距离的区别是什么？什么时候用哪个？

### 标准答案

### 核心区别

| 维度 | 余弦相似度 | 欧氏距离 |
|------|-----------|---------|
| **衡量什么** | 向量**夹角** | 向量**直线距离** |
| **范围** | [-1, 1] | [0, +∞) |
| **长度敏感吗** | ❌ 不敏感（只看方向） | ✅ 敏感 |
| **公式** | cos(θ) = (A·B)/(|A||B|) | √(Σ(xi-yi)²) |

### 什么时候用哪个？

- **余弦相似度**：**文本/语义相似度首选**
  - 理由：文本的语义和长度无关（"我喜欢苹果"和"我真的很喜欢吃苹果"语义相似但长度不同）
  
- **欧氏距离**：低维空间、位置相关（如坐标点）

**一句话总结：RAG 场景无脑选余弦相似度！**

## 第 4 题答案

### 问题
为什么 RAG 需要重排序（Rerank）？直接用向量检索不行吗？

### 标准答案

### 两阶段检索思想

| 阶段 | 方法 | 目标 | 速度 | 准确率 |
|------|------|------|------|--------|
| **第一阶段（召回）** | 向量检索 | 召回尽可能多的相关文档 | ⚡ 极快 | 70% |
| **第二阶段（精排）** | Rerank | 把最相关的排前面 | 🐢 慢 | 85% |

### 为什么需要两阶段？

- **向量检索**：快，但不够精准（可能混进去不相关的）
- **Rerank**：准，但慢（模型更大，计算更复杂）

**类比：**  
向量检索 = 海选（快速选出 100 人）  
Rerank = 面试（从 100 人里选出 3 个最好的）

## 第 5 题答案

### 问题
Word2Vec 的两种训练方式是什么？各自有什么优缺点？

### 标准答案

### 两种方式对比

| 方式 | 全称 | 做什么 | 优点 | 缺点 |
|------|------|--------|------|------|
| **CBOW** | Continuous Bag of Words | 用**上下文**预测**中心词** | 训练快，常见词好 | 稀有词效果一般 |
| **Skip-gram** | - | 用**中心词**预测**上下文** | 稀有词效果好，总体更好 | 训练稍慢 |

### 直观例子

句子：`我 喜欢 吃 苹果`

- **CBOW**：`[我, 吃]` → 预测 `喜欢`
- **Skip-gram**：`喜欢` → 预测 `[我, 吃, 苹果]`

### 选型建议
- 一般选 Skip-gram（效果好）
- 追求速度选 CBOW

## 第 6 题答案

### 问题
BPE（Byte-Pair Encoding）是怎么工作的？为什么大模型都用 BPE 做 Tokenization？

### 标准答案

### BPE 工作原理（4 步）

1. **初始词表**：都是单个字符（如 `a, b, c, ...`）
2. **统计频率**：统计所有相邻字符对的出现频率
3. **合并最高频**：把出现最多的字符对合并成一个 Token
4. **重复**：继续第 2-3 步，直到词表达到目标大小

### 例子
```
初始: l, o, w, n, e, r
统计: "low" 出现多，合并 "lo"
继续: "low" 出现多，合并 "low"
最终词表包含: "low", "new", "year", ...
```

### 为什么大模型都用 BPE？

| 原因 | 说明 |
|------|------|
| **词表大小可控** | 不用存所有可能的词 |
| **处理未登录词** | 可以把生词拆成子词 |
| **平衡语义和粒度** | 整词语义好但词表大，字符词表小但没语义 |

**对比：**
- 整词分词：词表太大（百万级），OOV 严重
- 字符分词：词表小（几百），但没语义
- **BPE（子词）：** 折中方案，最佳！
