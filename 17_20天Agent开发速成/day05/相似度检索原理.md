## 什么是相似度？

### 一句话定义
**衡量两个向量有多像**

```
向量A: [0.1, 0.2, 0.3]
向量B: [0.15, 0.25, 0.35]
    ↓ 相似度计算
0.95 (很像！)
```

---

## 三大相似度算法

### 1. 余弦相似度（Cosine Similarity）⭐

#### 定义
看两个向量的**夹角大小**

```
cos(θ) = (A · B) / (|A| × |B|)
```

#### 直观理解
- **θ = 0°** → cos(θ) = 1 → 完全一样
- **θ = 90°** → cos(θ) = 0 → 垂直（不相关）
- **θ = 180°** → cos(θ) = -1 → 完全相反

#### 范围
[-1, 1]

#### 优点
- ✅ 不关心向量长度，只关心方向
- ✅ 适合文本相似度（语义和长度无关）
- ✅ 最常用！

#### 缺点
- ❌ 忽略了向量的大小

#### 代码实现
```python
# -*- coding: utf-8 -*-
import numpy as np

def cosine_similarity(a, b):
    """余弦相似度"""
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

# 测试
vec1 = np.array([1, 2, 3])
vec2 = np.array([1.1, 2.1, 3.1])
vec3 = np.array([3, 2, 1])
vec4 = np.array([-1, -2, -3])

print(f"vec1 &amp; vec2: {cosine_similarity(vec1, vec2):.4f}")  # 很像
print(f"vec1 &amp; vec3: {cosine_similarity(vec1, vec3):.4f}")  # 一般
print(f"vec1 &amp; vec4: {cosine_similarity(vec1, vec4):.4f}")  # 相反
```

### 2. 欧氏距离（Euclidean Distance）

#### 定义
两个向量在空间中的**直线距离**

```
distance = √[(x1-x2)² + (y1-y2)² + ...]
```

#### 直观理解
- **距离 = 0** → 完全一样
- **距离小** → 像
- **距离大** → 不像

#### 范围
[0, +∞)

#### 优点
- ✅ 直观易懂
- ✅ 适合低维空间

#### 缺点
- ❌ 对向量长度敏感
- ❌ 高维空间效果一般

#### 代码实现
```python
def euclidean_distance(a, b):
    """欧氏距离"""
    return np.linalg.norm(a - b)

# 测试
vec1 = np.array([1, 2, 3])
vec2 = np.array([1.1, 2.1, 3.1])
vec3 = np.array([10, 20, 30])  # 同方向，但长很多

print(f"vec1 &amp; vec2 距离: {euclidean_distance(vec1, vec2):.4f}")
print(f"vec1 &amp; vec3 距离: {euclidean_distance(vec1, vec3):.4f}")
```

### 3. 内积（Dot Product / Inner Product）

#### 定义
```
A · B = x1×x2 + y1×y2 + ...
```

#### 与余弦相似度的关系
如果向量**归一化**（长度=1），内积 = 余弦相似度

#### 优点
- ✅ 计算最快
- ✅ 归一化后等价于余弦

#### 缺点
- ❌ 对向量长度敏感

#### 代码实现
```python
def dot_product(a, b):
    """内积"""
    return np.dot(a, b)

def normalize(v):
    """归一化"""
    return v / np.linalg.norm(v)

# 测试
vec1 = np.array([1, 2, 3])
vec2 = np.array([1.1, 2.1, 3.1])

print(f"内积: {dot_product(vec1, vec2):.4f}")
print(f"归一化后内积: {dot_product(normalize(vec1), normalize(vec2)):.4f}")
print(f"余弦相似度: {cosine_similarity(vec1, vec2):.4f}")
```

## 三种算法对比表

| 算法 | 衡量什么 | 范围 | 文本相似度 | 速度 | 推荐场景 |
|------|---------|------|-----------|------|---------|
| **余弦相似度** | 向量夹角 | [-1, 1] | ⭐⭐⭐⭐⭐ | ⚡⚡⚡ | **RAG 首选** |
| **欧氏距离** | 直线距离 | [0, ∞) | ⭐⭐⭐ | ⚡⚡⚡ | 低维空间 |
| **内积** | 投影长度 | (-∞, ∞) | ⭐⭐ | ⚡⚡⚡⚡⚡ | 归一化后用 |

## 怎么选择？

### 决策树

```
你的场景
    |
    +-- 是文本/语义相似度？
    |   |
    |   +-- 是 → 余弦相似度 ✅
    |   |
    |   +-- 否 → 低维空间？
    |       |
    |       +-- 是 → 欧氏距离
    |       |
    |       +-- 否 → 归一化后内积
    |
    +-- 向量已经归一化？
        |
        +-- 是 → 内积 = 余弦，直接用内积（更快）
        |
        +-- 否 → 余弦相似度
```

### 经验法则
- **RAG/文本检索**：余弦相似度（默认选择）
- **推荐系统**：内积（归一化后）
- **低维数据（如坐标）**：欧氏距离

## 完整对比代码示例

```python
# -*- coding: utf-8 -*-
import numpy as np
from sentence_transformers import SentenceTransformer, util

# 1. 准备数据
sentences = [
    "我喜欢吃苹果",
    "我爱吃苹果",
    "我喜欢吃香蕉",
    "今天天气真好",
    "Python 是编程语言"
]

# 2. 加载模型
model = SentenceTransformer('BAAI/bge-small-zh-v1.5')

# 3. 向量化
embeddings = model.encode(sentences, convert_to_tensor=True)

# 4. 计算相似度矩阵
print("=" * 70)
print("余弦相似度矩阵")
print("=" * 70)
cosine_scores = util.cos_sim(embeddings, embeddings)
for i in range(len(sentences)):
    for j in range(len(sentences)):
        print(f"{cosine_scores[i][j]:.3f}", end="  ")
    print()

# 5. 查询示例
query = "我喜欢吃水果"
query_embedding = model.encode(query, convert_to_tensor=True)

print("\n" + "=" * 70)
print(f"查询: {query}")
print("=" * 70)

# 余弦相似度
cosine_hits = util.semantic_search(query_embedding, embeddings, top_k=3)[0]
print("\n余弦相似度结果:")
for hit in cosine_hits:
    print(f"  {sentences[hit['corpus_id']]} (score: {hit['score']:.3f})")

# 欧氏距离（用 sklearn）
from sklearn.metrics.pairwise import euclidean_distances
euclidean_matrix = euclidean_distances(embeddings, [query_embedding])
print("\n欧氏距离结果:")
for idx in np.argsort(euclidean_matrix.flatten())[:3]:
    print(f"  {sentences[idx]} (distance: {euclidean_matrix[idx][0]:.3f})")
```

## 常见问题 QA

### Q1: 什么时候用余弦，什么时候用欧氏？
- **文本/语义**：余弦（长度不重要）
- **位置/坐标**：欧氏（长度重要）

### Q2: 余弦相似度是 0.8，算高吗？
- 0.9+：非常相似
- 0.7-0.9：比较相似
- 0.5-0.7：一般相似
- 0.5 以下：不太相似

### Q3: 为什么向量数据库一般默认用余弦？
文本语义相似度对向量长度不敏感，夹角更能代表语义。
