## 问题 1：ReAct 模式的核心思想是什么？相比 CoT 有什么区别？

### 回答要点：

**ReAct 核心思想：**
```
思考 (Thought) → 行动 (Action) → 观察 (Observation) → 思考...
```

**ReAct vs CoT 对比：**

| 特性 | CoT (思维链) | ReAct |
|------|-------------|-------|
| **模式** | 纯思考，无外部交互 | 思考 + 行动 + 观察 |
| **反馈** | 无环境反馈 | 有真实环境反馈 |
| **可靠性** | 可能产生幻觉 | 可以验证和修正 |
| **适用** | 简单推理任务 | 复杂需要交互的任务 |
| **代价** | 低（Token 少） | 高（多次调用） |

## 问题 2：自注意力的计算过程是什么？请写出公式和步骤。

### 回答要点：

**公式：**
```
Attention(Q, K, V) = Softmax(Q × K^T / √d_k) × V
```

**步骤：**

1. **Q × K^T**：计算查询和键的相似度矩阵
2. **/ √d_k**：缩放，防止 Softmax 梯度消失
3. **Softmax**：归一化为概率分布
4. **× V**：用注意力权重加权求和值矩阵

## 问题 3：为什么需要多头注意力？单头和多头的区别是什么？

### 回答要点：

**多头的作用：**
1. **多语义子空间**：不同头关注不同方面
2. **并行计算**：多头可以并行计算
3. **表达能力**：多头比单头表达能力更强

**单头 vs 多头对比：**

| 特性 | 单头 | 多头 |
|------|------|------|
| 语义 | 单一 | 多个子空间 |
| 计算 | 简单 | 可并行 |
| 表达 | 较弱 | 较强 |
| 参数 | 少 | 多 |

## 问题 4：ReAct 模式的提示词设计有什么技巧？如何提高 ReAct 的可靠性？

### 回答要点：

**提示词设计技巧：**
1. **格式严格**：明确规定 Thought/Action/Observation 格式
2. **示例完整**：给 1-3 个完整的示例
3. **工具描述清晰**：准确描述每个工具的用途和参数
4. **错误处理**：告诉模型如何处理失败情况

**提高可靠性的方法：**
- 最大步数限制，防止死循环
- 超时控制
- 错误重试机制
- 输出格式验证

## 问题 5：注意力机制中的 Mask 有什么作用？Padding Mask 和 Look-ahead Mask 的区别？

### 回答要点：

**Mask 的作用：**
- 让模型忽略某些位置的注意力

**两种 Mask 对比：**

| Mask 类型 | 作用 | 使用场景 |
|----------|------|---------|
| **Padding Mask** | 忽略 padding 位置 | Encoder + Decoder |
| **Look-ahead Mask** | 防止看到未来 token | Decoder 自回归 |

## 问题 6：如何可视化注意力权重？从中可以分析出什么？

### 回答要点：

**可视化方法：**
- 热力图（Heatmap）
- 注意力分布图
- 头之间的对比图

**可分析的内容：**
- 模型关注了哪些词
- 不同头关注的不同方面
- 注意力是否合理
- 诊断模型问题
