# Ubuntu 用户、用户组详解

## 1. 引言

用户和用户组管理是Ubuntu系统管理的核心组成部分，涉及系统安全、资源分配和权限控制等关键方面。本章将深入探讨Ubuntu用户和用户组的概念、工作原理、管理方法以及最佳实践。

## 2. 用户与用户组基础概念

### 2.1 用户（User）

用户是操作系统识别操作者的身份标识。每个用户拥有唯一的用户ID（UID），系统通过UID来区分不同用户并控制其对资源的访问权限。

### 2.2 用户组（Group）

用户组是用户的集合，用于简化权限管理。每个用户组拥有唯一的组ID（GID），用户可以属于多个组，其中有一个主组（Primary Group）和多个附属组（Supplementary Group）。

## 3. 用户与用户组相关文件

### 3.1 /etc/passwd

`/etc/passwd` 文件存储系统用户的基本信息，每行代表一个用户，格式如下：

```
username:password:uid:gid:gecos:home_directory:shell
```

各字段说明：
- **username**: 登录用户名
- **password**: 密码占位符（通常为'x'，实际密码存储在/etc/shadow）
- **uid**: 用户ID
- **gid**: 主组ID
- **gecos**: 用户全名和其他信息
- **home_directory**: 用户主目录
- **shell**: 用户默认shell

### 3.2 /etc/shadow

`/etc/shadow` 文件存储用户密码的哈希值和密码策略，只有root用户可读取，格式如下：

```
username:encrypted_password:last_change:min_age:max_age:warn:inactive:expire:reserved
```

### 3.3 /etc/group

`/etc/group` 文件存储用户组信息，每行代表一个组，格式如下：

```
groupname:password:gid:members
```

### 3.4 /etc/gshadow

`/etc/gshadow` 文件存储用户组的密码信息，格式与/etc/shadow类似。

## 4. 用户管理常用命令

### 4.1 用户创建命令

#### 4.1.1 命令功能概述
创建新用户是系统管理的基础操作，涉及用户身份信息、权限配置和环境初始化等多个方面。

#### 4.1.2 命令语法解析
```bash
useradd [选项] 用户名
```

#### 4.1.3 常用参数说明
- `-d, --home-dir`：指定用户主目录路径
- `-e, --expiredate`：设置账户过期日期（格式：YYYY-MM-DD）
- `-g, --gid`：指定用户主组ID或组名
- `-G, --groups`：指定用户附属组，多个组用逗号分隔
- `-m, --create-home`：自动创建用户主目录
- `-s, --shell`：指定用户默认shell
- `-u, --uid`：指定用户UID

#### 4.1.4 操作示例
创建一个名为`developer`的用户，指定主目录为`/home/developer`，默认shell为`/bin/bash`：
```bash
useradd -d /home/developer -m -s /bin/bash developer
```

> **注意**：根据系统安全策略，部分用户管理命令可能需要管理员权限。

### 4.2 用户信息修改命令

#### 4.2.1 命令功能概述
修改现有用户的属性信息，包括个人资料、权限设置和环境配置等。

#### 4.2.2 命令语法解析
```bash
usermod [选项] 用户名
```

#### 4.2.3 常用参数说明
- `-c, --comment`：修改用户GECOS信息（注释字段）
- `-d, --home`：修改用户主目录
- `-g, --gid`：修改用户主组
- `-G, --groups`：修改用户附属组
- `-l, --login`：修改用户名
- `-s, --shell`：修改默认shell
- `-u, --uid`：修改用户UID

#### 4.2.4 操作示例
修改用户`developer`的默认shell为`/bin/zsh`：
```bash
usermod -s /bin/zsh developer
```

### 4.3 用户删除命令

#### 4.3.1 命令功能概述
从系统中移除用户账户及其相关资源。

#### 4.3.2 命令语法解析
```bash
userdel [选项] 用户名
```

#### 4.3.3 常用参数说明
- `-r, --remove`：删除用户主目录及邮件池
- `-f, --force`：强制删除用户，即使用户当前已登录

#### 4.3.4 操作示例
彻底删除用户`developer`及其主目录：
```bash
userdel -r developer
```

### 4.4 密码管理命令

#### 4.4.1 命令功能概述
管理用户密码及密码策略，确保账户安全性。

#### 4.4.2 命令语法解析
```bash
passwd [选项] [用户名]
```

#### 4.4.3 常用参数说明
- `-d, --delete`：删除用户密码
- `-e, --expire`：强制用户下次登录时修改密码
- `-i, --inactive`：设置密码过期后账户停用天数
- `-l, --lock`：锁定用户账户
- `-u, --unlock`：解锁用户账户

#### 4.4.4 操作示例
强制用户`developer`下次登录修改密码：
```bash
passwd -e developer
```

## 5. 用户组管理常用命令

### 5.1 用户组创建命令

#### 5.1.1 命令功能概述
创建新的用户组，用于集中管理用户权限和资源访问控制。

#### 5.1.2 命令语法解析
```bash
groupadd [选项] 组名
```

#### 5.1.3 常用参数说明
- `-g, --gid`：指定组ID
- `-r, --system`：创建系统组
- `-K, --key`：覆盖/etc/login.defs中的默认值
- `-o, --non-unique`：允许使用非唯一的GID

#### 5.1.4 操作示例
创建一个名为`dev_team`的用户组：
```bash
groupadd dev_team
```

### 5.2 用户组修改命令

#### 5.2.1 命令功能概述
修改现有用户组的属性信息，包括组ID、组名等。

#### 5.2.2 命令语法解析
```bash
groupmod [选项] 组名
```

#### 5.2.3 常用参数说明
- `-g, --gid`：修改组ID
- `-n, --new-name`：修改组名
- `-o, --non-unique`：允许使用非唯一的GID

#### 5.2.4 操作示例
将用户组`dev_team`重命名为`development_team`：
```bash
groupmod -n development_team dev_team
```

### 5.3 用户组删除命令

#### 5.3.1 命令功能概述
从系统中移除用户组。

#### 5.3.2 命令语法解析
```bash
groupdel [选项] 组名
```

#### 5.3.3 操作示例
删除用户组`development_team`：
```bash
groupdel development_team
```

> **注意**：不能删除用户的主组，必须先修改用户的主组或删除用户。

### 5.4 用户组密码管理命令

#### 5.4.1 命令功能概述
管理用户组密码及组成员。

#### 5.4.2 命令语法解析
```bash
gpasswd [选项] 组名
```

#### 5.4.3 常用参数说明
- `-a, --add`：添加用户到组
- `-d, --delete`：从组中删除用户
- `-A, --administrators`：设置组管理员
- `-M, --members`：设置组成员列表

#### 5.4.4 操作示例
将用户`developer`添加到`development_team`组：
```bash
gpasswd -a developer development_team
```


## 7. 权限管理基础

### 7.1 文件权限表示方法

#### 7.1.1 符号表示法
Linux文件权限使用9个字符表示，分为3组，每组3个字符，分别对应所有者（User）、所属组（Group）和其他用户（Other）的权限。

权限字符说明：
- `r`（Read）：读取权限，值为4
- `w`（Write）：写入权限，值为2
- `x`（Execute）：执行权限，值为1
- `-`：无对应权限

示例：`-rwxr-xr--`表示：
- 所有者：可读、可写、可执行（rwx）
- 所属组：可读、可执行（r-x）
- 其他用户：只读（r--）

#### 7.1.2 数字表示法
数字表示法将权限转换为三位八进制数，每位对应一组权限（所有者、所属组、其他用户），计算方式为各权限值之和：

| 权限组合 | 数值 | 含义 |
|----------|------|------|
| rwx      | 7    | 可读、可写、可执行 |
| rw-      | 6    | 可读、可写 |
| r-x      | 5    | 可读、可执行 |
| r--      | 4    | 只读 |
| -wx      | 3    | 可写、可执行 |
| -w-      | 2    | 可写 |
| --x      | 1    | 可执行 |
| ---      | 0    | 无权限 |

示例：`754`表示：
- 所有者：7（rwx）
- 所属组：5（r-x）
- 其他用户：4（r--）

### 7.2 修改文件权限

#### 7.2.1 chmod命令
`chmod`命令用于修改文件或目录的权限，支持符号表示法和数字表示法。

**符号表示法语法**：
```bash
chmod [ugoa][+-=][rwx] 文件/目录
```
- `u`：所有者（user）
- `g`：所属组（group）
- `o`：其他用户（other）
- `a`：所有用户（all）
- `+`：添加权限
- `-`：移除权限
- `=`：设置权限

**数字表示法语法**：
```bash
chmod [权限数值] 文件/目录
```

**操作示例**：
1. 为文件`script.sh`添加所有者执行权限：
```bash
chmod u+x script.sh
```

2. 为目录`docs`设置权限`rwxr-xr-x`：
```bash
chmod 755 docs
```

3. 移除其他用户对文件`data.txt`的写入权限：
```bash
chmod o-w data.txt
```

#### 7.2.2 chown命令
`chown`命令用于修改文件或目录的所有者和所属组。

**语法**：
```bash
chown [选项] [所有者][:所属组] 文件/目录
```

**常用参数**：
- `-R, --recursive`：递归修改目录及其内容的所有者

**操作示例**：
1. 将文件`report.pdf`的所有者改为`developer`，所属组改为`dev_team`：
```bash
chown developer:dev_team report.pdf
```

2. 递归修改目录`project`及其内容的所有者为`admin`：
```bash
chown -R admin project/
```

### 7.3 特殊权限

#### 7.3.1 SUID（Set User ID）
- 作用：执行文件时，临时获得文件所有者的权限
- 表示：所有者权限位的`x`变为`s`（如`-rwsr-xr-x`）
- 设置命令：`chmod u+s 文件`
- 典型应用：`/bin/passwd`（普通用户可修改自己密码）

#### 7.3.2 SGID（Set Group ID）
- 作用：执行文件时，临时获得文件所属组的权限；对目录而言，在该目录下创建的新文件继承目录的所属组
- 表示：所属组权限位的`x`变为`s`（如`-rwxr-sr-x`）
- 设置命令：`chmod g+s 文件/目录`
- 典型应用：共享目录协作开发

#### 7.3.3 Sticky Bit（粘滞位）
- 作用：仅对目录有效，目录中的文件只能由所有者或root删除
- 表示：其他用户权限位的`x`变为`t`（如`drwxrwxrwt`）
- 设置命令：`chmod o+t 目录`
- 典型应用：`/tmp`目录（所有用户可写，但只能删除自己的文件）

### 7.4 ACL（访问控制列表）

ACL提供更精细的权限控制，允许为特定用户或组设置权限。

#### 7.4.1 查看ACL
```bash
getfacl 文件/目录
```

#### 7.4.2 设置ACL
```bash
setfacl [选项] 规则 文件/目录
```

**常用选项**：
- `-m, --modify`：修改ACL规则
- `-x, --remove`：删除ACL规则
- `-b, --remove-all`：删除所有ACL规则
- `-R, --recursive`：递归应用ACL规则

**规则格式**：`u:用户名:权限` 或 `g:组名:权限`

**操作示例**：
1. 为用户`testuser`授予文件`data.csv`的读写权限：
```bash
setfacl -m u:testuser:rw data.csv
```

2. 为组`testers`授予目录`testdata`的读执行权限：
```bash
setfacl -m g:testers:rx testdata/
```

## 8. 系统用户与特权管理

### 8.1 系统用户与普通用户

#### 8.1.1 用户类型区分
Ubuntu系统将用户分为两类，通过UID范围区分：
- **系统用户**：UID范围通常为1-999，用于运行系统服务和进程，无登录权限
- **普通用户**：UID范围通常为1000及以上，供实际用户登录使用

#### 8.1.2 系统用户特点
- 无交互式登录shell（通常为`/usr/sbin/nologin`或`/bin/false`）
- 无主目录或使用特殊目录（如`/var/run/`下的服务目录）
- 用于特定系统服务（如`www-data`用于Web服务，`mysql`用于数据库服务）
- 权限严格限制，仅满足服务运行所需最小权限

#### 8.1.3 查看系统用户列表
```bash
awk -F: '($3 < 1000) {print $1, $3, $7}' /etc/passwd
```

### 8.2 sudo权限管理

#### 8.2.1 sudo概述
`sudo`（Superuser Do）允许普通用户以其他用户（通常是root）身份执行命令，是Ubuntu权限管理的核心机制。

#### 8.2.2 sudo配置文件
- **主配置文件**：`/etc/sudoers`（推荐使用`visudo`命令编辑，提供语法检查）
- **配置目录**：`/etc/sudoers.d/`（存放额外配置文件，优先级高于主配置文件）

#### 8.2.3 sudoers配置语法
基本语法格式：
```
用户/组 主机=（运行身份） 命令列表
```

常用配置项：
- `ALL`：通配符，表示所有（用户、主机、命令等）
- `NOPASSWD:`：执行命令无需输入密码
- `PASSWD:`：执行命令需要输入密码（默认）
- `Cmnd_Alias`：命令别名定义
- `User_Alias`：用户别名定义
- `Host_Alias`：主机别名定义

#### 8.2.4 配置示例
1. 允许用户`developer`在所有主机上以root身份执行所有命令：
```
developer ALL=(ALL) ALL
```

2. 允许用户组`admin`无需密码执行所有命令：
```
%admin ALL=(ALL) NOPASSWD: ALL
```

3. 仅允许用户`operator`执行特定系统管理命令：
```
operator ALL=(ALL) /bin/systemctl restart apache2, /bin/systemctl status apache2
```

#### 8.2.5 sudo命令常用选项
- `-l`：列出当前用户可执行的sudo命令
- `-u`：指定以其他用户身份执行命令
- `-k`：清除sudo缓存的密码
- `-s`：启动一个shell
- `-i`：模拟初始登录（加载目标用户环境变量）

**操作示例**：
以`www-data`用户身份执行命令：
```bash
sudo -u www-data ls /var/www/
```

### 8.3 权限提升安全策略

#### 8.3.1 最小权限原则
- 仅授予用户完成工作所需的最小权限
- 避免将`ALL=(ALL) ALL`权限授予普通用户
- 为不同用户/组创建精细化的命令授权规则

#### 8.3.2 sudo日志审计
Ubuntu默认将sudo操作记录到`/var/log/auth.log`：
```bash
grep sudo /var/log/auth.log
```

#### 8.3.3 密码安全策略
- 设置合理的密码复杂度要求（通过`/etc/pam.d/common-password`配置）
- 定期更换密码
- 禁止使用空密码

#### 8.3.4 禁用root直接登录
Ubuntu默认禁用root直接登录，通过`sudo`提升权限更安全

## 9. 经验总结与常见陷阱

### 9.1 最佳实践

#### 9.1.1 用户管理规范
- 使用有意义的用户名和组名（如反映角色或部门）
- 为每个用户分配唯一账号，避免共享账户
- 定期清理不再使用的用户和组
- 为特殊项目创建专用用户和组

#### 9.1.2 权限管理建议
- 目录通常设置为`755`权限，文件通常设置为`644`权限
- 谨慎使用`chmod -R 777`（会导致严重安全隐患）
- 对可执行文件和脚本设置最小必要权限
- 优先使用ACL而非修改文件所有者实现复杂权限控制

### 9.2 常见陷阱与规避方法

#### 9.2.1 用户删除陷阱
**问题**：直接删除用户而未清理其文件，导致遗留文件归属不明
**解决**：使用`userdel -r`彻底删除用户及其主目录

#### 9.2.2 权限继承问题
**问题**：复制文件后权限未按预期继承
**解决**：使用`cp -a`保留文件属性，或明确设置目标权限

#### 9.2.3 sudo配置错误
**问题**：错误配置sudoers导致无法使用sudo
**解决**：使用`visudo`编辑（提供语法检查），保留root会话备用

#### 9.2.4 SUID滥用风险
**问题**：为普通程序设置SUID权限导致权限提升漏洞
**解决**：仅为必要系统程序设置SUID，定期审计系统SUID文件：
```bash
find / -perm -4000 2>/dev/null
```

### 9.3 故障排查流程

1. **权限问题排查**：
   ```bash
   ls -l 文件路径          # 检查文件权限
   id 用户名              # 检查用户ID和所属组
   groups 用户名          # 检查用户所属组
   getfacl 文件路径       # 检查ACL权限
   ```

2. **sudo问题排查**：
   ```bash
   sudo -l                # 检查当前用户sudo权限
   grep sudo /var/log/auth.log  # 查看sudo操作日志
   ```

## 10. 底层原理剖析

### 10.1 用户认证流程

1. 用户输入用户名和密码
2. PAM（Pluggable Authentication Modules）验证密码
3. 验证成功后，系统读取`/etc/passwd`和`/etc/group`获取用户信息
4. 初始化用户环境（读取`.bashrc`等配置文件）
5. 启动用户shell

### 10.2 权限检查机制

当用户访问文件时，系统按以下顺序检查权限：
1. 如果用户是文件所有者，应用所有者权限
2. 如果用户属于文件所属组，应用组权限
3. 应用其他用户权限

### 10.3 /etc/login.defs配置文件

该文件定义了用户和组创建的默认参数：
- UID和GID的范围
- 密码过期策略
- 主目录创建默认权限
- 邮件目录位置

示例配置：
```
UID_MIN                  1000
UID_MAX                 60000
GID_MIN                  1000
GID_MAX                 60000
PASS_MAX_DAYS            99999
PASS_MIN_DAYS            0
PASS_WARN_AGE            7
HOME_MODE                0700
```

### 11.1 高级用户管理工具

- **useradd/usermod/userdel**：命令行基础工具
- **adduser/deluser**：Debian/Ubuntu专用，提供交互式界面
- **gpasswd**：组密码和成员管理

  ```bash

  ```

- **acl-tools**：ACL权限管理工具集
- **sudoedit**：安全编辑特权文件

### 11.3 集中用户管理
