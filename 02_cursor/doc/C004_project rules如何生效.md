# Cursor Rules 使用指南

## 规则生效方式

要让自定义的 Cursor Rules 生效，您需要：

1. 在项目根目录下创建 `.cursor/rules` 目录（如果还没有的话）
2. 在该目录中创建 `.mdc` 文件（Markdown Cursor Rules 文件）
3. 在文件中编写规则内容

规则生效的条件：
- 确保规则文件使用 `.mdc` 扩展名
- 规则文件应该放在 `.cursor/rules` 目录下
- 规则文件的内容应该遵循正确的格式，包括：
  - 规则名称（使用 `#` 标记）
  - 规则描述
  - 具体的规则内容（使用 Markdown 格式）

## 规则的使用方式

在 Cursor 中，规则的使用方式主要有以下几种：

1. **自动触发**
   - 当您打开与规则相关的文件或进行相关操作时，AI 助手会自动识别并使用相应的规则
   - 例如，当您处理 WSL 相关的问题时，`wsl-assistant.mdc` 中的规则会自动生效

2. **手动引用**
   - 您可以在对话中明确提到需要使用某个规则
   - 例如，您可以说："请使用 WSL 助手规则来帮助我解决这个问题"

3. **规则组合**
   - 多个规则可以同时生效
   - 例如，当处理 Windows 和 WSL 相关的问题时，`windows-assistant.mdc` 和 `wsl-assistant.mdc` 的规则可能会同时被使用

## 规则的具体应用

以 WSL 助手规则为例，说明如何使用：

1. **WSL 相关问题的使用方式**
   - 当您需要处理 WSL 相关的问题时，AI 助手会自动遵循 `wsl-assistant.mdc` 中定义的规则
   - 例如，如果您问："如何安装 WSL2？"，AI 会：
     - 使用简体中文回答
     - 提供详细的步骤说明
     - 包含必要的命令示例
     - 解释每个步骤的目的
     - 提供安全建议

2. **规则的具体应用场景**
   - 安装和配置：当您需要安装或配置 WSL 时
   - 故障排除：当您遇到 WSL 相关的问题时
   - 性能优化：当您需要优化 WSL 性能时
   - 安全配置：当您需要加强 WSL 安全性时
   - 开发环境：当您需要配置 WSL 开发环境时

## 最佳实践

1. **使用建议**
   - 在提问时，尽量具体说明您的需求
   - 如果问题涉及多个方面，可以明确要求使用特定的规则
   - 如果需要详细的步骤说明，可以要求 AI 按照规则中的"响应方式"来回答

2. **规则的优势**
   - 确保回答的专业性和一致性
   - 提供结构化的解决方案
   - 包含必要的安全考虑
   - 提供持续的支持和跟进

3. **使用技巧**
   - 对于特定领域的问题，直接提问即可，规则会自动生效
   - 如果需要特定类型的回答（如详细的步骤说明），可以在问题中说明
   - 如果遇到复杂问题，可以要求 AI 按照规则中的特定部分来回答

## 注意事项

- 规则文件必须使用 `.mdc` 扩展名
- 规则内容应该使用 Markdown 格式
- 每个规则文件应该有一个清晰的标题和描述
- 规则文件应该放在正确的目录结构中
- 规则可以随时更新和修改，修改后会自动生效
