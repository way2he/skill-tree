# WSL入门指南

## 1. WSL简介

### 1.1 什么是WSL
WSL（Windows Subsystem for Linux）是Windows 10/11的一个功能，它允许用户在Windows系统上直接运行Linux环境，无需使用虚拟机或双系统。

### 1.2 WSL的优势
- **无缝集成**：在Windows中直接使用Linux命令和工具
- **资源占用少**：相比虚拟机，WSL占用更少的系统资源
- **文件系统互通**：Windows和Linux文件系统可以互相访问
- **开发便利**：为开发者提供类Unix环境，便于开发工作

## 2. 安装WSL

### 2.1 系统要求
- Windows 10 版本 2004 及更高版本（内部版本 19041 及更高版本）
- Windows 11
- 64位处理器
- 支持虚拟化

### 2.2 安装步骤
1. 以管理员身份打开PowerShell，运行以下命令启用WSL功能：
`它会在Windows系统中启用"Windows Subsystem for Linux"这个功能组件`
```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

2. 启用虚拟机平台功能：
```powershell
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

3. 重启计算机

4. 下载并安装[WSL2 Linux内核更新包](https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi)

5. 设置WSL 2为默认版本：
```powershell
wsl --set-default-version 2
```

6. 从Microsoft Store安装Linux发行版（推荐Ubuntu）

## 3. 基本使用

### 3.1 常用命令
```bash
# 查看已安装的Linux发行版
wsl --list

# 查看已安装的Linux发行版详细信息（包括版本和状态）
wsl --list --verbose
# 或简写为
wsl -l -v

# 启动特定发行版
wsl -d Ubuntu

# 关闭WSL
wsl --shutdown

# 更新WSL
wsl --update

# 查看WSL版本
wsl --version

# 设置默认WSL版本
wsl --set-default-version 2

# 设置特定发行版的WSL版本
wsl --set-version <发行版名称> 2

# 终止特定发行版
wsl --terminate <发行版名称>

# 卸载特定发行版
wsl --unregister <发行版名称>

# 导出发行版到文件
wsl --export <发行版名称> <文件名.tar>

# 导入发行版
wsl --import <发行版名称> <安装位置> <文件名.tar>

# 查看WSL状态
wsl --status

# 查看WSL帮助信息
wsl --help
```
### 如何查看正在运行中的linux发行版

要查看当前正在运行的WSL Linux发行版，可以使用以下命令：

```powershell
# 查看所有已安装的发行版及其状态
wsl --list --verbose
# 或简写为
wsl -l -v
```

输出示例：
```
  NAME      STATE           VERSION
* Ubuntu    Running         2
  Debian    Stopped         2
```

说明：
- `*` 表示当前默认的发行版
- `STATE` 列显示发行版的运行状态（Running/Stopped）
- `VERSION` 列显示WSL版本（1或2）

也可以使用以下命令查看更详细的状态信息：
```powershell
wsl --status
```

### 3.2 文件系统访问
- Windows文件系统在WSL中的路径：`/mnt/c/`
- WSL文件系统在Windows中的路径：`\\wsl$\<发行版名称>`

## 4. 实用技巧

### 4.1 配置开发环境
1. 更新系统包：
```bash
sudo apt update && sudo apt upgrade
```

2. 安装常用开发工具：
```bash
sudo apt install build-essential git curl
```

### 4.2 网络配置
- WSL使用Windows的网络连接
- 可以通过`ifconfig`或`ip addr`查看网络配置
- 端口转发需要在Windows防火墙中配置

## 5. 常见问题解决

### 5.1 性能优化
- 在`%UserProfile%\.wslconfig`中配置WSL资源限制
- 定期清理未使用的包和缓存
- 使用WSL 2获得更好的性能

### 5.2 故障排除
1. WSL无法启动：
   - 检查Windows功能是否启用
   - 确认BIOS中虚拟化已开启
   - 运行`sfc /scannow`修复系统文件

2. 网络连接问题：
   - 检查Windows防火墙设置
   - 确认网络适配器配置
   - 重启WSL服务

### 8.1 安全建议
- 定期更新系统和软件包
- 使用强密码
- 谨慎使用root权限

### 8.2 使用限制
- 某些Windows特定的功能可能无法在WSL中使用
- 图形界面应用需要额外配置
- 部分硬件访问可能受限
