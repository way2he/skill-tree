# .cursorrules 和 .cursor/rules 的区别

## 1. .cursorrules 文件
- 这是一个全局配置文件，位于项目根目录
- 用于定义整个项目的编辑器设置和规则
- 包含编辑器行为、文件关联、格式化规则等通用设置
- 影响所有使用该项目的开发者

## 2. .cursor/rules 目录
- 这是一个本地配置目录，位于 `.cursor` 文件夹下
- 用于存储项目特定的规则和配置
- 可以包含多个规则文件，每个文件针对不同的功能或模块
- 通常用于存储：
  - 代码风格规则
  - 项目特定的 lint 规则
  - 自定义的代码片段
  - 项目特定的编辑器配置

## 主要区别

### 1. 作用范围
- `.cursorrules` 是全局性的，影响整个项目
- `.cursor/rules` 是模块化的，可以针对不同部分设置不同规则

### 2. 配置方式
- `.cursorrules` 是单个 JSON 文件
- `.cursor/rules` 是目录，可以包含多个配置文件

### 3. 使用场景
- `.cursorrules` 适合设置基础编辑器配置
- `.cursor/rules` 适合设置项目特定的规则和自定义配置

### 4. 优先级
- `.cursor/rules` 中的规则通常会覆盖 `.cursorrules` 中的全局设置
- 这样可以实现更细粒度的控制

## 使用建议

1. 将通用的编辑器设置放在 `.cursorrules` 中
2. 将项目特定的规则和自定义配置放在 `.cursor/rules` 中
3. 使用 `.cursor/rules` 来覆盖或扩展全局设置
4. 保持配置的清晰和模块化，便于维护和管理
