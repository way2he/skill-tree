# Claude Code Skills 完全指南

## 一、什么是 Skills？

### 定义
Skills（技能）是 Claude Code 的能力扩展机制，是 Claude Code 可直接调用的最小能力单元。

通过安装不同的 Skills，你可以让 Claude Code 获得特定领域的专业能力。

### 能做什么
- 代码审查与测试驱动开发（TDD）
- 项目上下文记忆与跨会话保持
- 特定语言/框架的最佳实践（Spring Boot、Django、Go、Swift 等）
- 部署、数据库迁移、安全扫描等工程化能力
- 文档处理、内容创作、市场研究等通用能力

### 为什么需要
Claude Code 默认具备通用编程能力，但在特定场景下可能缺乏深度。
Skills 让 Claude Code：

- 获得专业领域的深度知识（如 Spring Boot 安全最佳实践）
- 遵循特定规范和模式（如 Go 语言惯用法）
- 执行复杂的工作流（如 TDD 开发流程）
- 记忆项目上下文（跨会话保持一致性）

### Skills vs 传统提示词
很多人会问：为什么不直接写提示词，而要用 Skills？

| 对比项 | 传统提示词 | Skills |
|--------|------------|--------|
| 复用性 | 每次都要复制粘贴或重新输入 | 安装一次，永久可用 |
| 管理方式 | 分散在各个对话中，难以管理 | 统一存放在本地目录，清晰有序 |
| 跨会话 | 新会话需要重新提供提示词 | 自动加载，跨会话保持一致 |
| 触发方式 | 需要手动输入完整提示词 | 自动感知上下文，智能触发 |
| 共享协作 | 难以分享给团队成员 | 可打包成插件，一键安装分享 |
| 版本管理 | 无法追踪变更 | 支持 Git 版本控制 |
| 依赖管理 | 手动处理 | 插件市场自动处理 |

**总结**：传统提示词适合临时性、一次性的需求；Skills 适合需要长期使用、团队协作、版本管理的场景。

## 二、核心概念区分

在使用 Claude Code 扩展能力之前，必须先搞清楚三个核心概念。

### Plugin（插件）
插件是 Claude Code 官方包管理体系的最小安装单位。

- 一个完整插件必须包含根目录的 plugin.json 元数据配置文件
- 可封装 1 个或多个 Skill、自定义命令（Command）、生命周期钩子（Hook）、依赖配置等内容
- 安装时以插件为单位，不能只安装插件内的某一个 Skill

### Skill（技能）
技能是 Claude Code 可直接调用的最小能力单元。

- 必须包含 SKILL.md 元数据文件
- 可选包含 skill.json 额外配置
- 可独立存在（手动安装），也可被封装在 Plugin 插件内（插件市场安装）

### Marketplace（市场）
市场是插件集合仓库。

- 官方市场：anthropics/claude-plugins-official，包含多个独立插件，每个可单独安装
- 社区市场：社区维护的仓库，可能是一个大插件（安装获得全部内容），也可能是多个独立插件的集合

### 三者关系
```
Marketplace（市场）
└── Plugin（插件）← 安装的最小单位
    ├── plugin.json（必需）
    ├── Skill A（技能）
    │   └── SKILL.md（必需）
    ├── Skill B（技能）
    ├── Command（命令）
    └── Hook（钩子）
```

## 三、Skills 的标准文件结构

### 简单 Skill 结构
最简单的 Skill 只需要一个文件夹和一个文件：

```
~/.claude/skills/          # Unix/macOS 路径，Windows 为 C:\Users\<用户名>\.claude\skills\
└── my-skill/
    └── SKILL.md
```

### 核心规则
- 文件夹名称：Skill 的唯一标识
- SKILL.md：唯一必需的文件（必须全大写 + .md 小写）

### 复杂 Skill 结构
复杂的 Skill 可以包含更多资源：

```
~/.claude/skills/          # Unix/macOS 路径，Windows 为 C:\Users\<用户名>\.claude\skills\
└── my-skill/
    ├── SKILL.md              # 必须：元数据和核心指令
    ├── skill.json            # 可选：额外的元数据配置
    ├── scripts/              # 可选：可执行脚本
    ├── references/           # 可选：参考文档
    └── assets/               # 可选：资源文件
```

### 核心文件说明
| 文件 | 必须/可选 | 说明 |
|------|-----------|------|
| SKILL.md | 必须 | 包含元数据（name、description）和核心指令，Claude 依靠 description 决定何时激活该 Skill |
| skill.json | 可选 | 额外的元数据配置，如版本、作者、分类等 |
| scripts/ | 可选 | 可执行代码（Python、Shell 等） |
| references/ | 可选 | 参考文档和示例 |
| assets/ | 可选 | 图片、模板等资源文件 |

### SKILL.md 格式示例
```markdown
---
name: explain-code
description: Explains code with visual diagrams and analogies
---

When explaining code, always include:
1. **Start with an analogy**
2. **Draw a diagram** using ASCII art
3. **Walk through the code** step-by-step
4. **Highlight a gotcha** or common mistake
```

### 关键点
- 元数据块中 name 必须与文件夹名称一致
- description 是核心，Claude 依靠这段描述来决定何时激活该 Skill

## 四、Skills 的自动触发机制

### 自动触发原理
大部分 Skills 安装后会自动根据上下文激活，你不需要手动输入命令。

Claude Code 会实时分析：
- 当前对话内容
- 打开的文件类型
- 项目结构和技术栈

然后智能判断需要调用哪些 Skills。

### 典型触发场景
| 场景 | Skill 自动激活 |
|------|---------------|
| 打开一个 Go 项目并询问代码问题 | go-review、golang-patterns 自动提供 Go 最佳实践 |
| 粘贴一段异常栈问「这是什么问题」 | 调试相关 Skills 自动分析根因 |
| 修改代码后说「帮我提交」 | Git 集成 Skills 自动生成 commit message |
| 在 Spring Boot 项目中添加新功能 | springboot-patterns、springboot-tdd 自动提供架构指导 |
| 上传一个 PDF 并问「帮我提取关键信息」 | 文档处理 Skills 自动解析内容 |

### 需要手动触发的情况
只有少数 Skills 需要手动输入 / 命令触发，主要是：
- 自定义命令型 Skills：你封装的专属工作流命令（如 /build-java）
- 明确指令型 Skills：如 /skill-create（创建新技能）、/plan（规划任务）

**核心原则**：安装 Skills 后，大部分情况下你只需要用自然语言描述需求，Claude Code 会自动调用合适的技能。

## 五、两种安装方式详解

### 方式一：插件市场安装（推荐）

#### 原理
通过 Claude Code 官方 CLI 的包管理体系，从合规的插件市场（官方/社区维护的、具备 marketplace.json 索引的插件集合仓库），以 Plugin 插件为最小单位完成安装。

安装过程由工具自动处理：路径配置、依赖安装、版本注册、更新管理。

#### 市场类型详解：官方 vs 社区 vs 独立
Claude Code 的插件市场分为三种类型，理解它们的区别非常重要：

##### 1. 官方插件市场
- **维护方**：Anthropic 官方
- **仓库地址**：anthropics/claude-plugins-official

**结构特点**：
```
anthropics/claude-plugins-official/    ← 一个仓库 = 一个市场
├── plugins/
│   ├── code-simplifier/               ← 独立插件 A
│   ├── skill-creator/                 ← 独立插件 B
│   └── code-review/                   ← 独立插件 C
└── marketplace.json                   ← 市场索引文件
```

**核心特点**：
- 仓库包含多个独立插件，每个可单独安装
- 官方审核，质量有保障
- 可以只安装你需要的某一个插件

**安装示例**：
```bash
# 添加官方市场
/plugin marketplace add https://github.com/anthropics/claude-plugins-official

# 只安装 code-simplifier 这一个插件
/plugin install code-simplifier@claude-plugins-official
```

##### 2. 社区插件市场
- **维护方**：社区开发者
- **典型仓库**：affaan-m/everything-claude-code、thedotmack/claude-mem

**结构特点**：
```
affaan-m/everything-claude-code/       ← 一个仓库 = 一个大插件
├── .claude-plugin/
│   └── plugin.json                    ← 整个仓库是一个插件
├── skills/                            ← 50+ Skills 都在这里
│   ├── python-patterns/
│   ├── springboot-patterns/
│   └── ...
└── marketplace.json
```

**核心特点**：
- 整个仓库通常是一个大插件
- 安装后获得全部 Skills，无法只安装其中一个
- 社区维护，质量参差不齐，需自行判断

**安装示例**：
```bash
# 添加社区市场
/plugin marketplace add https://github.com/affaan-m/everything-claude-code

# 安装整个大插件（获得全部 50+ Skills）
/plugin install everything-claude-code@everything-claude-code
```

##### 3. 独立市场
- **维护方**：社区开发者（但使用独立的市场仓库）
- **典型项目**：obra/superpowers

**结构特点**：
```
obra/superpowers                      ← 主仓库（Skills 源码）
obra/superpowers-marketplace          ← 单独的市场仓库（索引）
```

**为什么需要两个仓库？**
- 主仓库：存放 Skills、文档、源码
- 市场仓库：只包含 marketplace.json + plugin.json，指向主仓库

**核心特点**：
- 项目作者自己创建了一个专属的 market
- 本质上还是「社区插件」，安装后获得全部 Skills
- 需要先添加它的独立市场，再安装插件

**安装示例**：
```bash
# 先添加它的独立市场（注意：是 -marketplace 后缀的仓库）
/plugin marketplace add obra/superpowers-marketplace

# 再安装插件
/plugin install superpowers@superpowers-marketplace
```

#### 三种市场对比总结
| 对比项 | 官方市场 | 社区市场 | 独立市场 |
|--------|----------|----------|----------|
| 维护方 | Anthropic 官方 | 社区开发者 | 社区开发者 |
| 仓库数量 | 1 个 | 1 个 | 2 个（主仓库 + 市场仓库） |
| Market 名称 | claude-plugins-official | 通常与仓库名相同 | 自定义（如 superpowers-marketplace） |
| 插件结构 | 多个独立插件 | 一个大插件 | 一个大插件 |
| 能否选装单个插件 | ✅ 可以 | ❌ 一口全装 | ❌ 一口全装 |
| 质量保障 | 官方审核 | 需自行判断 | 需自行判断 |
| 推荐程度 | ⭐⭐⭐ 优先安装 | ⭐⭐ 按需选择 | ⭐⭐ 按需选择 |

**简单理解**：
- 官方市场：像应用商店，可以单独下载某个 App
- 社区市场：像软件套装（如 Office），安装就获得全部功能
- 独立市场：社区市场的变种，只是用独立的市场仓库来管理

#### 国内 Windows 用户安装方法
添加插件市场时必须使用 HTTPS 地址（SSH 格式在国内网络环境下可能受限）。

```bash
# 1. 添加插件市场（必须用 HTTPS）
/plugin marketplace add https://github.com/<marketplace-url>

# 2. 安装插件
/plugin install <plugin-name>@<marketplace-name>

# 启用/禁用插件
/plugin enable <plugin-name>@<marketplace-name>
/plugin disable <plugin-name>@<marketplace-name>

# 卸载插件
/plugin uninstall <plugin-name>@<marketplace-name>
```

**如果遇到网络问题**：
- 配置 Git 代理：`git config --global http.proxy http://127.0.0.1:端口`
- 使用国内镜像源（如 Gitee、GitCode）
- 选择本地手动安装方式

#### 安装粒度说明
安装的是插件，不是单个 Skill。

- 官方市场：可从仓库中选择安装某一个插件（每个插件大多是单 Skill）
- 社区一体化大插件：安装后获得插件内全部 Skills，无法只安装其中一个 Skill

**如果你只需要某个大插件里的某一个 Skill，有两个选择**：
- 安装整个插件（插件市场安装）—— 获得完整功能 + 自动更新
- 从插件仓库中单独复制该 Skill 文件夹到本地（本地手动安装）—— 只获得该 Skill，但无自动更新

### 方式二：本地手动安装

#### 原理
不经过 Claude Code 官方包管理体系，从任意来源获取单个独立的 Skill 文件目录，手动复制到 Claude Code 指定的本地 skills 系统目录，完成能力注册。

无论该 Skill 原本是否属于某个插件，只要是手动放置 Skill 文件的部署方式，均归为此类。

#### 步骤
```bash
# 1. 获取 Skill 文件（从 GitHub 克隆或下载 ZIP）
git clone https://github.com/xxx/xxx.git

# 2. 复制到 Skills 目录
# Windows PowerShell（用户范围，所有项目可用）
Copy-Item -Recurse ".\skill-folder" "$env:USERPROFILE\.claude\skills\"

# Windows PowerShell（项目范围，仅当前项目可用）
Copy-Item -Recurse ".\skill-folder" "$(Get-Location)\.claude\skills\"

# Git Bash / WSL（用户范围）
cp -r <skill-folder> ~/.claude/skills/

# Git Bash / WSL（项目范围）
cp -r <skill-folder> ./.claude/skills/

# 3. 重启 Claude Code
```

#### 安装路径
| 范围 | Windows 路径 | 适用场景 |
|------|-------------|----------|
| 用户范围 | C:\Users\<用户名>\.claude\skills\<skill-name>\ | 所有项目可用 |
| 项目范围 | <项目目录>\.claude\skills\<skill-name>\ | 仅当前项目，团队共享 |

#### 适用场景
- 未上架插件市场的 Skills
- 涉及敏感密钥、需要完全本地运行的安全类 Skills
- 需要深度修改 Skill 源码的场景
- 完全离线的开发环境

#### 注意事项
该方式不受插件包管理体系的限制，但需自行处理：依赖兼容、版本更新、安全校验。

#### 两种方式对比
| 对比项 | 插件市场安装 | 本地手动安装 |
|--------|--------------|--------------|
| 安装单位 | Plugin（插件） | Skill（技能） |
| 自动处理依赖 | ✅ | ❌ |
| 自动更新 | ✅ | ❌ |
| 路径配置 | ✅ 自动 | ❌ 手动 |
| 选择粒度 | 整个插件 | 单个 Skill |
| 适用场景 | 通用开发 | 离线/定制/安全敏感 |

**选择建议**：
- 90% 场景优先选择插件市场安装
- 仅以下场景选择手动安装：
  - 未上架插件市场的 Skills
  - 涉及敏感密钥的安全类 Skills
  - 需要深度修改源码的场景
  - 完全离线的开发环境

### 二、社区插件市场（精选推荐）
以下项目均为 GitHub 高星标项目，经过社区广泛使用和验证，用户评价良好。

#### ⭐⭐⭐⭐⭐ everything-claude-code（必装推荐）
- **GitHub**: affaan-m/everything-claude-code | 星标: 72k+
- **背景**：Anthropic 黑客松冠军项目，奖金 $15,000，10个月实战打磨

**用户评价**：
- "把 Claude Code 从聊天机器人变成资深工程师"
- "Claude Code 的外挂级插件"
- 中英文技术媒体广泛报道（掘金、CSDN、头条等）

**核心能力**：
- 50+ Skills 覆盖全开发流程
- 热门命令：/tdd、/plan、/go-review、/python-review、/security-review
- 框架覆盖：Spring Boot、Django、Go、Python、Swift、React/Next.js
- 工程化：部署、数据库迁移、安全扫描、Docker、E2E 测试

**安装步骤**：

1. **步骤一：安装市场和插件**
   ```bash
   /plugin marketplace add https://github.com/affaan-m/everything-claude-code
   /plugin install everything-claude-code@everything-claude-code
   ```

2. **步骤二：安装 Rules，手动复制 Rules**
   ```bash
   #1. 切换到D盘根目录，自动创建并进入 ClaudeSkillsLocal 文件夹（存在则直接进入）
   D:
   cd \ClaudeSkillsLocal 2>nul || md \ClaudeSkillsLocal && cd \ClaudeSkillsLocal
   
   #2. 克隆 everything-claude-code 仓库到当前文件夹（临时下载）
   git clone https://github.com/affaan-m/everything-claude-code.git
   
   #3. 进入刚下载完成的仓库文件夹
   cd everything-claude-code
   
   #4. 创建 Claude Code 所需的规则配置目录（已存在则忽略）
   md %USERPROFILE%\.claude\rules
   
   #5. 【全量安装】复制 rules 下所有文件夹（通用规则+所有语言规则）
   #官方允许：完整复制所有子文件夹，无覆盖、无冲突，适配所有开发语言
   #解释：/E 复制子文件夹 /I 假定目标是目录 /Y 覆盖文件不询问
   xcopy /E /I /Y rules\* %USERPROFILE%\.claude\rules\
   
   #6. 验证配置文件：列出所有规则文件夹，显示全部语言即成功
   dir %USERPROFILE%\.claude\rules
   
   #7. 退回上级目录（退出占用的文件夹，解决删除失败问题）
   cd ..
   
   #8. 【绝对安全】仅删除临时克隆的文件夹，不影响任何其他文件
   rd /s /q everything-claude-code
   ```

3. **步骤三：验证是否成功安装**
   - 重启一下 Claude Code 软件
   - 输入 `/everything-claude-code` 查看所属技能

## 七、常见问题

### Q: Skills 安装后不生效怎么办？
- 重启 Claude Code：新增 Skills 后需要重启才能生效
- 检查文件路径：确保路径正确，如 C:\Users\<用户名>\.claude\skills\<skill-name>\SKILL.md
- 检查文件名：必须是 SKILL.md（全大写 + .md 小写）
- 检查插件是否启用：使用 /plugin 查看插件状态

### Q: 如何查看已安装的 Skills？
```bash
# 插件市场安装的插件
/plugin

# 手动安装的 Skills（Windows）
dir "$env:USERPROFILE\.claude\skills\"

# 手动安装的 Skills（项目范围）
dir ".\.claude\skills\"
```

### Q: 如何卸载 Skills？
```bash
# 插件市场安装的
/plugin uninstall <plugin-name>@<marketplace-name>

# 手动安装的（Windows）
Remove-Item -Recurse -Force "$env:USERPROFILE\.claude\skills\<skill-name>"
```

### Q: 能否只安装插件里的某一个 Skill？
- **通过插件市场安装**：不能。插件是安装的最小单位，安装插件后获得其全部内容。
- **通过本地手动安装**：可以。你可以从插件仓库中单独复制该 Skill 文件夹到 ~/.claude/skills/。

**选择建议**：
- 如果该 Skill 有插件化安装渠道，优先选择插件市场安装（获得完整功能 + 自动更新）
- 如果只需要特定 Skill 且不需要更新，可选择手动安装

### Q: Windows 添加插件市场失败怎么办？
- 确保使用 HTTPS 地址，不要用 SSH 格式（如 git@github.com:xxx）
- 配置 Git 代理：`git config --global http.proxy http://127.0.0.1:端口`
- 使用国内镜像源（如 Gitee、GitCode）
- 选择本地手动安装方式

## 八、总结

### 核心要点
| 概念 | 核心文件 | 说明 |
|------|----------|------|
| Skill | SKILL.md | 能力单元，可独立存在或封装在插件内 |
| Plugin | plugin.json | 安装单位，可包含多个 Skills、Commands、Hooks |
| Marketplace | marketplace.json | 插件集合仓库 |

### 安装原则
- 90% 场景：优先选择插件市场安装（自动处理依赖、支持更新）
- 10% 场景：选择手动安装（未上架插件市场、敏感密钥、深度定制、离线环境）

### 使用原则
- 大部分 Skills 自动触发，你只需用自然语言描述需求
- 只有自定义命令或明确指令型 Skills 需要手动触发
