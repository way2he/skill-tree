# 用 Hermes Agent 两周的真实感受：它让我重新定义了"AI助手"

我之前用过不少 AI 编程助手——Claude Code、Cursor、Copilot，还有各种终端里的 ChatGPT wrapper。说实话，一开始对 Hermes Agent 没抱太大期望，以为又是换皮的"又一个 Agent 框架"。

用了两周之后，我的看法完全变了。

不是因为它能做多少事，而是因为它**记住** 了多少事。

* * *

## 先说一个让我震惊的场景

第二周的某天，我随口跟 Hermes 说了一句："帮我把昨天那个项目的测试跑一下。"

它直接 cd 到了正确目录，执行了正确的命令，甚至知道要先激活虚拟环境。没有问我"哪个项目"，没有问我"什么测试"。

我愣了一下才反应过来——它**记住了** 。不是那种 session 内的上下文记忆，而是跨 session 的持久记忆。上一次我操作这个项目的时候，它默默记下了目录结构、常用命令、甚至我的 Python 版本偏好。

这就是 Hermes 最核心的差异化：**它不只是执行任务，它在积累经验。**

* * *

## 五个让我离不开的功能

### 1\. 持久记忆：像一个真正了解你的助手

大多数 AI 助手是"金鱼记忆"——每次对话都是从零开始。你得反复告诉它你的偏好、环境、项目结构。

Hermes 有两层记忆：

•**用户记忆** ：你是谁、偏好什么、习惯什么沟通方式•**技能记忆** ：解决过的问题、发现的工作流、踩过的坑

这些记忆在 `~/.hermes/` 下持久保存，每次启动自动加载。用得越久，它越懂你。

举个例子，它知道我习惯用 Python 3.10、偏好用 uv 管理依赖、写代码不用分号。这些细节我从来没有显式告诉过它，是它从我的操作中"学"到的。

### 2\. 技能系统：让经验可以复用

这是我最喜欢的功能，没有之一。

当 Hermes 完成一个复杂任务（5 次以上工具调用），或者解决了一个棘手的 bug，它会主动建议："要不我把这个方案保存成技能？"

技能就是一个 Markdown 文档，记录了：  
\- 什么情况下用这个技能  
\- 具体步骤是什么  
\- 有哪些坑要注意

下次遇到类似问题，它自动加载技能，直接按最佳实践来。**相当于你在不断地给 AI 写 SOP。**

我现在有十几个自定义技能：部署特定项目的流程、处理某种数据格式的步骤、调试某个服务的方法……这些都是从实际操作中沉淀出来的，比任何文档都靠谱。

### 3\. 多平台网关：同一大脑，多个身体

Hermes 不只是一个终端工具。它有一个 Gateway 模式，可以把同一个 Agent 接入：

•Telegram•Discord•Slack•WhatsApp•Signal•微信（我目前在用的）•邮件•甚至 Matrix、Mattermost

关键是：**不管从哪个平台发消息，都是同一个 Agent，共享同一套记忆和技能。**

我在电脑上用终端跟它聊，出门用微信发语音，回到家看到它已经把事情办好了。这种体验真的很"贾维斯"。

### 4\. 定时任务：让 Agent 主动干活

Hermes 内置了 cron 系统，可以设置定时任务。不是那种简单的 cron job，而是：

> "每天早上 8 点，帮我抓取 AI 热点新闻，整理成简报发到微信。"

它会自动执行、格式化、投递到指定平台。如果执行出错，会在日志里记录。你可以随时查看、暂停、修改。

我现在用它做：  
\- 每日 AI 新闻简报  
\- 每周 GitHub trending 汇总  
\- 定期清理服务器日志

**让 AI 当你的值班助手，这种感觉很爽。**

### 5\. 子 Agent 调度：一个人干一个团队的活

Hermes 可以 spawn 独立的子 Agent 进程，每个子 Agent 有独立的会话、工具和环境。

实际场景：  
\- 主 Agent 负责协调和决策  
\- 子 Agent A 去写后端 API  
\- 子 Agent B 去写前端页面  
\- 子 Agent C 去跑测试

三个并行推进，主 Agent 负责汇总。这已经不是"AI 助手"了，这是**AI 团队** 。

## 它不适合谁？

说了这么多优点，也说说局限：

**不适合完全的新手。** Hermes 需要你在终端里操作，需要配置 API key，需要理解基本的 Linux 命令。如果你连 `cd` 和 `ls` 都不熟，建议先从 Claude Code 或 Cursor 开始。

**不适合需要精美 UI 的人。** Hermes 的交互是终端 + 聊天，没有图形界面。它的美感在于效率，不在于颜值。

**不适合只想要"一次性问答"的场景。** 如果你只是偶尔问 ChatGPT 一个问题，用不着 Hermes。它的价值在于**长期使用** ——用得越久，它越好用。

* * *

## 一些实用建议

如果你决定试试 Hermes，几个建议：

**第一周别急着配太多东西。** 先用默认配置跑起来，让它观察你的操作习惯。等它开始主动记住你的偏好之后，再慢慢加技能和定时任务。

**善用技能系统。** 每次完成复杂任务后，让它保存技能。一个月后你会拥有一套专属的"操作手册"。

**开启多平台。** 至少接一个即时通讯平台。能在手机上随时跟 Agent 沟通，体验完全不一样。

**用定时任务替代重复工作。** 任何你每天/每周都会做的事情，都可以考虑交给 Hermes 的 cron 系统。

* * *

## 最后的感受

用了两周 Hermes，我最大的感受是：**它让我重新思考了人和 AI 的关系。**

以前用 AI 助手，我是"提问者"，AI 是"回答者"。一问一答，用完就忘。

现在用 Hermes，我更像是在**带一个实习生** 。刚开始它什么都要问，慢慢地它学会了你的习惯，再后来它能主动帮你做事。这个"成长"的过程，比任何功能都让人有成就感。

它不完美，偶尔会犯蠢，有时候记忆也会出错。但就像一个好的实习生——关键不是它现在多厉害，而是它成长的速度有多快。
