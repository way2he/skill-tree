# article-writing Skill 详解

## 一、Skill 概述

article-writing 是一个专业的文章写作 Skill，它为 Agent 提供了结构化的文章创作流程，能够根据用户需求生成高质量的文章内容。

## 二、核心组成部分

### 1. 触发条件
- 用户明确要求写文章、写博客、写公众号推文
- 用户要求撰写技术文档、教程、指南
- 用户需要创作新闻稿、产品介绍、营销文案
- 任何需要长篇文字创作的场景

### 2. 执行流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  需求分析   │────▶│  大纲规划   │────▶│  内容创作   │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  风格确认   │────▶│  润色优化   │────▶│  最终输出   │
└─────────────┘     └─────────────┘     └─────────────┘
```

**详细步骤：**

#### 步骤一：需求分析
- 确认文章类型（公众号、博客、技术文档等）
- 明确目标读者群体
- 确定文章主题和核心观点
- 了解文章篇幅要求

#### 步骤二：大纲规划
- 制定文章结构框架
- 规划各章节内容要点
- 确定逻辑递进关系
- 设计引人入胜的开头和有力的结尾

#### 步骤三：内容创作
- 按照大纲逐段撰写
- 保持语言风格一致
- 融入相关案例和数据
- 确保内容准确可信

#### 步骤四：润色优化
- 检查语法和拼写错误
- 优化句子结构和表达
- 增强文章的可读性
- 调整段落长度和节奏

#### 步骤五：最终输出
- 生成完整文章
- 提供标题建议
- 检查格式规范
- 确认满足所有需求

### 3. 使用工具
- 文本生成工具（大模型自身能力）
- 搜索引擎（如需补充资料）
- 图片生成工具（如需配图）
- 格式转换工具（如需导出不同格式）

### 5. 注意事项

#### 写作原则：
- **用户中心**：始终考虑目标读者的需求和理解能力
- **逻辑清晰**：文章结构合理，逻辑递进自然
- **语言生动**：避免枯燥乏味，适当使用修辞手法
- **内容准确**：确保信息真实可靠，引用权威来源
- **风格统一**：保持整篇文章的语言风格一致

#### 避坑指南：
- 避免使用过于专业的术语，必要时加以解释
- 避免段落过长，影响阅读体验
- 避免内容空洞，要有具体案例和数据支撑
- 避免标题党，标题要与内容相符
- 避免抄袭，引用他人内容要注明出处

## 三、实际应用示例

### 示例一：公众号文章写作

**用户需求：**
"帮我写一篇关于人工智能在医疗领域应用的公众号文章，目标读者是普通大众，字数 1500 字左右。"

**Skill 执行过程：**

1. **需求分析**：
   - 类型：公众号文章
   - 读者：普通大众
   - 主题：AI 在医疗领域的应用
   - 篇幅：1500 字

2. **大纲规划**：
   - 标题：《AI 医生来了：人工智能如何改变我们的医疗未来？》
   - 结构：
     - 引言：用一个生动的场景开头
     - 正文一：AI 在疾病诊断中的应用
     - 正文二：AI 在药物研发中的作用
     - 正文三：AI 在个性化治疗中的优势
     - 总结：展望未来，强调人机协作

3. **内容创作**：
   - 使用通俗易懂的语言
   - 加入真实案例
   - 适当使用数据增强说服力

4. **润色优化**：
   - 调整段落长度适合手机阅读
   - 加入 emoji 增加趣味性
   - 优化标题使其更吸引人

5. **最终输出**：
   完整的公众号文章，包括标题、正文、结尾引导。

### 示例二：技术文档写作

**用户需求：**
"写一份 Python 虚拟环境使用教程，目标读者是 Python 初学者，要包含详细的操作步骤和示例代码。"

**Skill 执行过程：**

1. **需求分析**：
   - 类型：技术教程
   - 读者：Python 初学者
   - 主题：Python 虚拟环境使用
   - 要求：详细步骤 + 示例代码

2. **大纲规划**：
   - 标题：《Python 虚拟环境完全指南》
   - 结构：
     - 概述：什么是虚拟环境，为什么需要
     - 前置条件：已安装 Python
     - 详细说明：
       - 虚拟环境的创建
       - 虚拟环境的激活
       - 虚拟环境的使用
       - 虚拟环境的退出和删除
     - 常见问题
     - 参考资源

3. **内容创作**：
   - 步骤清晰，每一步都有说明
   - 提供完整的命令示例
   - 解释每个命令的作用
   - 加入注意事项提示

4. **润色优化**：
   - 检查技术准确性
   - 确保代码示例可运行
   - 优化表述使其更易理解

5. **最终输出**：
   完整的技术教程，包含代码块、命令示例、注意事项。

## 五、最佳实践建议

### 1. 需求确认阶段
- 在开始写作前，务必与用户确认以下信息：
  - 文章的具体用途
  - 目标读者是谁
  - 有没有特定的风格偏好
  - 是否有必须包含的关键点
  - 截止时间和篇幅要求

### 2. 大纲创作阶段
- 先向用户展示大纲，获得确认后再开始写正文
- 大纲要足够详细，包含各部分的核心要点
- 可以提供 2-3 个不同的大纲方案供用户选择

### 3. 内容创作阶段
- 对于长文章，可以分阶段交付，先写一部分让用户确认风格
- 保持与用户的沟通，及时调整方向
- 注意引用权威来源，增强文章可信度

### 4. 润色优化阶段
- 主动检查错别字和语法错误
- 建议多个标题供用户选择
- 提供一些配图建议（如适用）

## 六、常见问题解答

### Q: 如何确定文章的语气和风格？
A: 根据目标读者和发布平台来确定：
- 公众号文章：亲切、生动、有趣
- 技术博客：专业、严谨、清晰
- 企业官网：正式、规范、可信
- 社交媒体：轻松、简洁、互动性强

### Q: 文章写得太长或太短怎么办？
A: 
- 太长：精简内容，保留核心要点，删除冗余部分
- 太短：扩展案例，增加细节，补充相关背景知识

### Q: 如何让文章更有吸引力？
A: 
- 用一个吸引人的标题
- 开头用故事或问题引起兴趣
- 使用具体的案例和数据
- 加入个人观点和见解
- 保持语言生动有趣

### Q: 如何处理专业术语？
A: 
- 如果读者是专业人士：可以直接使用
- 如果读者是普通大众：需要用通俗易懂的语言解释
- 可以在第一次出现时加括号解释
- 必要时可以用类比的方式说明

article-writing Skill 是一个强大的内容创作工具，它通过结构化的流程和清晰的指导，帮助 Agent 产出高质量的文章内容。掌握这个 Skill，可以大幅提升内容创作的效率和质量。
