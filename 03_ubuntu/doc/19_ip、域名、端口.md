# Ubuntu IP、域名、端口详解

## 1. 概述

在Ubuntu系统中，IP地址、域名和端口是网络通信的基础组件。理解这些概念及其配置方法对于系统管理、网络故障排查和服务部署至关重要。本文将深入探讨Ubuntu环境下IP地址配置、域名解析和端口管理的相关知识，包括基本概念、常用命令、配置文件、底层原理以及实际应用中的经验总结。

## 2. IP地址

### 2.1 IP地址基本概念

IP（Internet Protocol）地址是网络中标识设备的唯一地址，用于在网络中定位和通信。目前广泛使用的IP协议有两个版本：

- IPv4：32位地址，通常表示为四组十进制数（0-255），以点分隔，如192.168.1.1
- IPv6：128位地址，通常表示为八组十六进制数，以冒号分隔，如2001:0db8:85a3:0000:0000:8a2e:0370:7334

在Ubuntu系统中，IP地址可以通过静态配置或动态获取（如DHCP）两种方式分配。

### 2.2 IP地址配置方法

#### 2.2.1 使用netplan配置（Ubuntu 18.04及以上）

Ubuntu 18.04及以上版本采用netplan作为网络配置工具，配置文件通常位于`/etc/netplan/`目录下，文件扩展名为`.yaml`。

**示例：配置静态IP**

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
```

应用配置：
```bash
sudo netplan apply
```

**示例：配置DHCP**

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: yes
```

#### 2.2.2 传统interfaces配置（Ubuntu 16.04及以下）

早期Ubuntu版本使用`/etc/network/interfaces`文件配置网络：

**示例：配置静态IP**

```bash
auto enp0s3
iface enp0s3 inet static
address 192.168.1.100
netmask 255.255.255.0
gateway 192.168.1.1
dns-nameservers 8.8.8.8 8.8.4.4
```

**示例：配置DHCP**

```bash
auto enp0s3
iface enp0s3 inet dhcp
```

应用配置：
```bash
sudo /etc/init.d/networking restart
```

### 2.3 常用IP相关命令

#### 2.3.1 查看网络接口信息

```bash
# 查看所有网络接口及其IP地址
ip addr show

# 简化显示
ip a

# 传统命令
ifconfig
```

#### 2.3.2 测试网络连接

```bash
# 测试与目标主机的连通性
ping google.com

# 指定次数
ping -c 4 google.com

# IPv6 ping
ping6 ipv6.google.com
```

#### 2.3.3 查看路由表

```bash
ip route show
ip r

# 传统命令
route -n
```

#### 2.3.4 配置临时IP地址（重启后失效）

```bash
sudo ip addr add 192.168.1.100/24 dev enp0s3
sudo ip route add default via 192.168.1.1 dev enp0s3
```

#### 2.3.5 启用/禁用网络接口

```bash
sudo ip link set enp0s3 up
sudo ip link set enp0s3 down

# 传统命令
sudo ifup enp0s3
sudo ifdown enp0s3
```

### 2.4 重要配置文件

1. `/etc/netplan/*.yaml` - Ubuntu 18.04+ 网络配置文件
2. `/etc/network/interfaces` - 传统网络配置文件
3. `/etc/resolv.conf` - DNS解析配置（通常由系统自动管理）
4. `/etc/hosts` - 本地主机名解析文件

### 2.5 IP网络底层原理

#### 2.5.1 ARP协议

地址解析协议（ARP）用于将IP地址转换为MAC地址。当主机需要与同一局域网内的其他主机通信时，会通过ARP请求获取目标主机的MAC地址。

查看ARP缓存：
```bash
arp -a
```

#### 2.5.2 子网划分

子网掩码用于将IP地址划分为网络部分和主机部分。例如，子网掩码255.255.255.0（/24）表示前24位为网络地址，后8位为主机地址。

CIDR（无类别域间路由）表示法：IP地址后跟随斜杠和网络前缀长度，如192.168.1.100/24。

#### 2.5.3 NAT（网络地址转换）

NAT允许私有网络中的多台主机通过单个公有IP地址访问互联网，常见于家庭路由器和企业网关设备。Ubuntu系统可以通过`iptables`配置NAT功能。

### 2.6 经验总结与常见陷阱

#### 2.6.1 常见问题及解决方案

1. **网络接口未激活**
   - 检查接口状态：`ip link show enp0s3`
   - 激活接口：`sudo ip link set enp0s3 up`

2. **无法获取DHCP地址**
   - 检查DHCP服务器是否可用
   - 手动释放并获取IP：`sudo dhclient -r enp0s3 && sudo dhclient enp0s3`

3. **静态IP配置后无法上网**
   - 检查网关设置是否正确
   - 检查DNS配置
   - 验证路由表：`ip route show`

#### 2.6.2 最佳实践

1. 对于服务器，始终使用静态IP配置
2. 配置多个DNS服务器以提高可靠性
3. 使用有意义的主机名，并在`/etc/hosts`中配置常用主机
4. 定期备份网络配置文件
5. 使用`ip`命令而非过时的`ifconfig`等命令

## 3. 域名

### 3.1 域名系统（DNS）基本概念

域名是互联网上用于标识和定位计算机的字符串，是IP地址的人性化替代品。域名系统（DNS）负责将域名解析为对应的IP地址，使人们能够通过易于记忆的域名访问互联网资源，而无需记住复杂的IP地址。

域名结构采用层次化结构，从右到左依次为顶级域名、二级域名、三级域名等，各层级之间用点分隔。例如，在`www.ubuntu.com`中：
- `.com` 是顶级域名（TLD）
- `ubuntu` 是二级域名
- `www` 是三级域名（通常用作万维网服务标识）

### 3.2 DNS解析原理

DNS解析过程通常包括以下步骤：

1. **本地解析**：当用户在浏览器中输入域名时，系统首先检查本地DNS缓存和`/etc/hosts`文件是否有对应的IP地址
2. **递归查询**：如果本地没有找到，操作系统会向配置的DNS服务器发送查询请求
3. **迭代查询**：DNS服务器如果没有该域名的记录，会向根域名服务器查询，然后依次向下查询顶级域名服务器、权威域名服务器等
4. **结果返回**：最终将查询到的IP地址返回给客户端，并在本地缓存该记录

DNS查询类型主要包括：
- A记录：将域名解析为IPv4地址
- AAAA记录：将域名解析为IPv6地址
- CNAME记录：域名别名
- MX记录：邮件交换记录
- NS记录：域名服务器记录
- TXT记录：文本记录，常用于SPF验证等

### 3.3 Ubuntu DNS配置

#### 3.3.1 使用resolv.conf配置（传统方法）

`/etc/resolv.conf`是DNS客户端配置文件，用于指定DNS服务器地址。在现代Ubuntu系统中，该文件通常由系统服务（如systemd-resolved）自动管理，直接编辑可能会被覆盖。

**示例配置**：
```
nameserver 8.8.8.8  # Google DNS
nameserver 8.8.4.4  # Google DNS备用
nameserver 114.114.114.114  # 国内DNS
search example.com  # 搜索域，当输入短域名时自动补全
```

#### 3.3.2 使用systemd-resolved配置（现代方法）

Ubuntu 16.10及以上版本默认使用systemd-resolved服务管理DNS解析。可以通过`systemd-resolve`命令或修改`/etc/systemd/resolved.conf`文件进行配置。

**修改全局DNS配置**：
```bash
sudo nano /etc/systemd/resolved.conf
```

在文件中添加或修改以下内容：
```
[Resolve]
DNS=8.8.8.8 8.8.4.4
FallbackDNS=114.114.114.114
Domains=example.com
```

重启服务使配置生效：
```bash
sudo systemctl restart systemd-resolved
sudo systemctl enable systemd-resolved
```

#### 3.3.3 使用netplan配置DNS

在netplan配置文件中可以直接指定DNS服务器：

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
        search: [example.com]
```

应用配置：
```bash
sudo netplan apply
```

#### 3.3.4 本地域名解析（/etc/hosts）

`/etc/hosts`文件允许用户手动指定域名与IP地址的映射关系，优先级高于DNS服务器查询。

**示例配置**：
```
127.0.0.1       localhost
127.0.1.1       ubuntu-server
192.168.1.10    server.example.com    server
10.0.0.5        nas.local             nas
```

### 3.4 常用DNS相关命令

#### 3.4.1 测试DNS解析

```bash
# 使用nslookup查询域名
nslookup www.ubuntu.com

# 指定DNS服务器查询
nslookup www.ubuntu.com 8.8.8.8

# 使用dig命令查询（更详细）
dig www.ubuntu.com

dig www.ubuntu.com AAAA  # 查询IPv6地址
dig www.ubuntu.com MX  # 查询邮件服务器

dig @8.8.8.8 www.ubuntu.com  # 指定DNS服务器

dig +short www.ubuntu.com  # 简化输出

# 使用host命令
host www.ubuntu.com
```

#### 3.4.2 查看DNS缓存

```bash
# 对于systemd-resolved
resolvectl statistics
resolvectl flush-caches

# 对于nscd
sudo nscd -g  # 查看统计信息
sudo nscd -i hosts  # 清除hosts缓存
```

#### 3.4.3 检查DNS配置

```bash
# 查看当前DNS服务器
systemd-resolve --status | grep 'DNS Servers'

# 或
cat /etc/resolv.conf
```

### 3.5 DNS故障排查

#### 3.5.1 常见DNS问题

1. **域名无法解析**
   - 检查网络连接
   - 验证DNS服务器是否可达：`ping 8.8.8.8`
   - 尝试使用其他DNS服务器：`nslookup www.ubuntu.com 114.114.114.114`

2. **DNS解析缓慢**
   - 配置多个DNS服务器
   - 检查DNS服务器响应时间：`dig +trace www.ubuntu.com`
   - 考虑使用本地DNS缓存服务

3. **DNS缓存污染**
   - 清除本地DNS缓存：`resolvectl flush-caches`
   - 使用可信DNS服务器

#### 3.5.2 DNS故障排查工具

```bash
# 跟踪DNS解析过程
dig +trace www.ubuntu.com

# 测试DNS服务器响应时间
for dns in 8.8.8.8 8.8.4.4 114.114.114.114; do
  echo "Testing $dns:";
  dig @$dns www.ubuntu.com | grep "Query time";
done
```

### 3.6 高级DNS配置

#### 3.6.1 配置本地DNS缓存

安装并配置dnsmasq作为本地DNS缓存：
```bash
sudo apt update
sudo apt install dnsmasq
```

配置dnsmasq：
```bash
sudo nano /etc/dnsmasq.conf
```

添加以下内容：
```
server=8.8.8.8
server=8.8.4.4
cache-size=1000
```

重启服务：
```bash
sudo systemctl restart dnsmasq
sudo systemctl enable dnsmasq
```

更新resolv.conf：
```bash
sudo nano /etc/resolv.conf
```

设置本地DNS：
```
nameserver 127.0.0.1
```

#### 3.6.2 使用DNS-over-TLS增强隐私

配置systemd-resolved使用DNS-over-TLS：
```bash
sudo nano /etc/systemd/resolved.conf
```

修改配置：
```
[Resolve]
DNS=8.8.8.8#dns.google 8.8.4.4#dns.google
DNSOverTLS=yes
FallbackDNS=1.1.1.1#cloudflare-dns.com 1.0.0.1#cloudflare-dns.com
```

重启服务：
```bash
sudo systemctl restart systemd-resolved
```

### 3.7 域名相关经验总结

#### 3.7.1 最佳实践

1. **配置多个DNS服务器**：至少配置2个不同的DNS服务器以提高可靠性
2. **合理设置DNS缓存**：使用本地DNS缓存可以显著提高解析速度
3. **重视`/etc/hosts`文件**：对于内部网络或开发环境，使用`/etc/hosts`可以快速配置域名解析
4. **定期清理DNS缓存**：在网络故障排查时，清理DNS缓存是重要步骤
5. **使用DNS-over-TLS或DNS-over-HTTPS**：增强DNS查询的隐私和安全性

#### 3.7.2 常见陷阱与规避

1. **`/etc/resolv.conf`被覆盖**：现代Ubuntu系统中直接编辑`/etc/resolv.conf`可能无效，应通过systemd-resolved或netplan进行配置
2. **DNS缓存导致配置不生效**：修改DNS配置后，记得清除本地缓存
3. **域名解析优先级问题**：`/etc/hosts`优先级高于DNS服务器，可能导致预期外的解析结果
4. **DNS服务器选择不当**：选择距离近、响应快的DNS服务器可以提高网络体验
5. **忽略IPv6 DNS配置**：在支持IPv6的网络中，应同时配置IPv6 DNS服务器

## 4. 端口

### 4.1 端口基本概念

端口（Port）是TCP/IP协议栈中用于标识特定网络服务的逻辑概念，它允许一台主机上的多个网络应用程序同时进行通信。端口与IP地址结合使用，形成一个完整的网络通信端点（Socket），格式为`IP地址:端口号`，如`192.168.1.100:80`。

端口本质上是一个16位的整数，取值范围为0-65535。根据用途和分配方式，端口可分为以下几类：

- **知名端口（Well-known Ports）**：0-1023，分配给特定的服务，如HTTP（80）、HTTPS（443）、SSH（22）等
- **注册端口（Registered Ports）**：1024-49151，由IANA分配给用户进程或应用程序，如MySQL（3306）、Redis（6379）等
- **动态/私有端口（Dynamic/Private Ports）**：49152-65535，用于客户端临时连接，通常由操作系统自动分配

端口基于传输层协议可分为TCP端口和UDP端口，两者独立编号。例如，TCP 80和UDP 80是不同的端口，可以同时被不同服务使用。

### 4.2 常用端口与服务对应关系

以下是Ubuntu系统中常见的端口及其对应的服务：

| 端口号 | 协议 | 服务名称 | 用途 |
|--------|------|----------|------|
| 20     | TCP  | FTP-data | 文件传输协议（数据连接） |
| 21     | TCP  | FTP      | 文件传输协议（控制连接） |
| 22     | TCP  | SSH      | 安全外壳协议，远程登录 |
| 23     | TCP  | Telnet   | 远程终端协议（不安全，不推荐使用） |
| 25     | TCP  | SMTP     | 简单邮件传输协议 |
| 53     | TCP/UDP | DNS   | 域名系统 |
| 80     | TCP  | HTTP     | 超文本传输协议 |
| 110    | TCP  | POP3     | 邮局协议版本3 |
| 143    | TCP  | IMAP     | 互联网消息访问协议 |
| 443    | TCP  | HTTPS    | 安全超文本传输协议 |
| 3306   | TCP  | MySQL    | MySQL数据库服务 |
| 5432   | TCP  | PostgreSQL | PostgreSQL数据库服务 |
| 6379   | TCP  | Redis    | Redis键值存储 |
| 8080   | TCP  | HTTP-alt | 备用HTTP端口，常用于开发服务器 |
| 8443   | TCP  | HTTPS-alt | 备用HTTPS端口 |

### 4.3 端口管理常用命令

#### 4.3.1 查看开放端口

```bash
# 使用ss命令（推荐，替代netstat）
ss -tuln  # 显示所有监听的TCP/UDP端口
ss -tulnp # 显示监听端口及对应的进程（需要root权限）
ss -tuln | grep 80  # 过滤特定端口

# 参数说明：
# -t: TCP端口
# -u: UDP端口
# -l: 仅显示监听状态的端口
# -n: 以数字形式显示端口和IP地址
# -p: 显示进程信息

# 使用netstat命令（传统方法）
netstat -tuln
netstat -tulnp | grep 80

# 使用lsof命令（查看进程打开的文件和端口）
lsof -i :80  # 查看特定端口占用情况
lsof -i tcp:80  # 指定TCP协议
lsof -i udp:53  # 指定UDP协议
lsof -i @192.168.1.100:80  # 查看特定IP:端口
```

#### 4.3.2 测试端口连通性

```bash
# 使用telnet测试TCP端口（需安装telnet）
telnet example.com 80

telnet 192.168.1.100 22

# 使用nc（netcat）测试端口（推荐，支持TCP/UDP）
nc -zv example.com 80  # TCP测试
nc -zvu example.com 53 # UDP测试
nc -zv 192.168.1.100 3306

# 参数说明：
# -z: 零I/O模式，仅扫描端口
# -v: 详细输出
# -u: UDP模式

# 使用curl测试HTTP端口
curl -I example.com:80
curl -I https://example.com:443
```

#### 4.3.3 临时开放/关闭端口

通过调整应用程序配置或使用防火墙工具控制端口开放状态：

```bash
# 查找占用端口的进程并终止（临时关闭端口）
sudo ss -tulnp | grep 8080
# 假设进程ID为1234
sudo kill -9 1234

# 临时修改服务配置（以Nginx为例）
sudo nano /etc/nginx/sites-available/default
# 修改listen端口后重启服务
sudo systemctl restart nginx
```

### 4.4 Ubuntu防火墙与端口控制

Ubuntu系统默认使用UFW（Uncomplicated Firewall）作为前端工具管理netfilter防火墙规则，简化了iptables的复杂配置。

#### 4.4.1 UFW基础操作

```bash
# 安装UFW（通常默认已安装）
sudo apt install ufw

# 启用UFW
sudo ufw enable

# 查看UFW状态
sudo ufw status
sudo ufw status verbose  # 详细状态
sudo ufw status numbered  # 显示规则编号

# 禁用UFW
sudo ufw disable

# 重置UFW规则
sudo ufw reset
```

#### 4.4.2 配置端口规则

```bash
# 允许特定端口（TCP）
sudo ufw allow 22/tcp  # 允许SSH
sudo ufw allow 80/tcp  # 允许HTTP
sudo ufw allow 443/tcp # 允许HTTPS

# 允许特定端口（UDP）
sudo ufw allow 53/udp  # 允许DNS

# 允许端口范围
sudo ufw allow 3000:3010/tcp  # 允许TCP 3000-3010端口

# 允许特定IP访问所有端口
sudo ufw allow from 192.168.1.0/24

# 允许特定IP访问特定端口
sudo ufw allow from 192.168.1.100 to any port 3306

# 拒绝访问
sudo ufw deny 23/tcp  # 拒绝Telnet

sudo ufw deny from 10.0.0.5  # 拒绝特定IP

# 删除规则
sudo ufw status numbered  # 先查看规则编号
sudo ufw delete 3  # 删除编号为3的规则
```

#### 4.4.3 高级防火墙配置（iptables）

对于复杂场景，可以直接使用iptables配置防火墙规则：

```bash
# 查看iptables规则
sudo iptables -L
sudo iptables -L -n  # 数字形式显示

# 允许本地回环
sudo iptables -A INPUT -i lo -j ACCEPT

# 允许已建立的连接
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许SSH端口
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 允许HTTP和HTTPS
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 默认拒绝所有入站连接
sudo iptables -P INPUT DROP

# 保存iptables规则（Ubuntu需安装iptables-persistent）
sudo apt install iptables-persistent
sudo netfilter-persistent save
sudo netfilter-persistent reload
```

### 4.5 端口转发配置

端口转发允许将到达特定IP和端口的流量转发到其他IP和端口，常用于服务映射和访问控制。

#### 4.5.1 使用UFW配置端口转发

1. 启用IP转发：
```bash
sudo nano /etc/ufw/sysctl.conf
```
添加或取消注释：
```
net.ipv4.ip_forward=1
net.ipv6.conf.default.forwarding=1
net.ipv6.conf.all.forwarding=1
```

2. 配置转发规则：
```bash
sudo nano /etc/ufw/before.rules
```
在`*filter`部分之前添加：
```
*nat
:PREROUTING ACCEPT [0:0]
# 将外部80端口转发到内部192.168.1.101:8080
-A PREROUTING -p tcp --dport 80 -j DNAT --to-destination 192.168.1.101:8080
# 将外部443端口转发到内部192.168.1.102:443
-A PREROUTING -p tcp --dport 443 -j DNAT --to-destination 192.168.1.102:443
COMMIT
```

3. 重启UFW：
```bash
sudo ufw disable && sudo ufw enable
```

#### 4.5.2 使用iptables配置端口转发

```bash
# 将本地8080端口转发到192.168.1.100:80
sudo iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination 192.168.1.100:80

# 添加转发规则
sudo iptables -A FORWARD -p tcp -d 192.168.1.100 --dport 80 -j ACCEPT

# 保存规则
sudo netfilter-persistent save
```

### 4.6 端口安全与防护

#### 4.6.1 常见端口攻击类型

1. **端口扫描**：攻击者通过扫描开放端口寻找潜在漏洞
2. **端口劫持**：未经授权访问开放端口并接管服务
3. **DoS/DDoS攻击**：向特定端口发送大量请求导致服务不可用
4. **缓冲区溢出**：利用特定端口服务的漏洞执行恶意代码

#### 4.6.2 端口安全防护措施

1. **最小权限原则**：只开放必要端口，关闭不必要服务
```bash
sudo systemctl disable --now telnetd  # 禁用Telnet服务
sudo ufw deny 23/tcp  # 阻止Telnet端口
```

2. **使用非标准端口**：为敏感服务（如SSH）更改默认端口
```bash
# 修改SSH端口（/etc/ssh/sshd_config）
Port 2222
# 重启SSH服务
sudo systemctl restart sshd
sudo ufw allow 2222/tcp
sudo ufw deny 22/tcp
```

3. **限制访问来源**：仅允许特定IP访问敏感端口
```bash
sudo ufw allow from 192.168.1.0/24 to any port 2222/tcp
```

4. **使用防火墙和入侵检测系统**：配置UFW/iptables规则，部署IDS/IPS

5. **定期审计端口状态**：自动化脚本检查开放端口变化
```bash
# 创建简单的端口监控脚本
#!/bin/bash
ss -tuln > /tmp/ports_$(date +%F).txt
# 对比昨日记录
if [ -f /tmp/ports_$(date -d "yesterday" +%F).txt ]; then
  diff /tmp/ports_$(date -d "yesterday" +%F).txt /tmp/ports_$(date +%F).txt
fi
```

6. **使用安全协议**：用SSH替代Telnet，HTTPS替代HTTP，SFTP替代FTP

### 4.7 端口相关经验总结

#### 4.7.1 常见问题及解决方案

1. **端口被占用**
   - 查找占用进程：`sudo ss -tulnp | grep <端口号>`
   - 终止进程：`sudo kill -9 <进程ID>`
   - 或修改应用配置使用其他端口

2. **防火墙规则不生效**
   - 检查UFW状态：`sudo ufw status`
   - 验证规则顺序：规则从上到下匹配，更具体的规则应放在前面
   - 检查是否有其他防火墙工具（如firewalld）冲突

3. **端口转发不工作**
   - 确认IP转发已启用：`cat /proc/sys/net/ipv4/ip_forward`（应返回1）
   - 检查转发规则是否正确：`sudo iptables -t nat -L PREROUTING`
   - 验证目标服务是否正常运行

#### 4.7.2 最佳实践

1. **定期检查开放端口**：至少每月审计一次系统开放端口
2. **采用分层防御**：结合网络防火墙、主机防火墙和应用层防护
3. **记录端口变更**：对端口配置变更进行文档记录和版本控制
4. **使用工具自动化管理**：对于复杂环境，考虑使用Ansible、Puppet等工具管理防火墙规则
5. **监控端口流量**：使用iftop、nload等工具监控异常端口流量
6. **避免使用特权端口运行非必要服务**：非系统服务应使用1024以上端口

## 5. 综合案例分析

### 5.1 案例一：Web服务器多端口配置

场景：在Ubuntu服务器上部署多个Web应用，需要通过不同端口访问。

解决方案：
1. 配置Nginx作为反向代理，监听80和443端口
2. 根据域名或路径转发到不同后端端口的应用

```nginx
# /etc/nginx/sites-available/default
server {
    listen 80;
    server_name app1.example.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
    }
}

server {
    listen 80;
    server_name app2.example.com;
    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
    }
}
```

### 5.2 案例二：远程访问家庭服务器

场景：需要从互联网安全访问家庭Ubuntu服务器上的服务。

解决方案：
1. 在路由器配置端口转发，将外部端口映射到服务器内部端口
2. 服务器配置SSH密钥登录，禁用密码登录
3. 使用非标准SSH端口增强安全性
4. 配置UFW只允许必要端口和IP访问

```bash
# 服务器端配置SSH
sudo nano /etc/ssh/sshd_config
PasswordAuthentication no
Port 2222

# 重启SSH服务
sudo systemctl restart sshd

# 配置UFW
sudo ufw allow 2222/tcp
sudo ufw allow 443/tcp
sudo ufw default deny incoming
sudo ufw enable
```
