# ubuntu24.04压缩与解压详解

## 1. 引言

在Ubuntu 24.04系统中，压缩与解压是日常操作中不可或缺的技能。无论是处理软件包、备份文件，还是传输数据，都需要用到压缩与解压工具。本文档将详细介绍Ubuntu 24.04系统中常用的压缩解压工具、命令用法、高级技巧以及底层原理，帮助用户全面掌握这一重要技能。

## 2. 压缩解压基础

### 2.1 压缩的概念与作用

文件压缩是指通过特定的算法将文件或目录转换成更小的存储空间的过程。压缩的主要作用包括：
- 节省磁盘空间
- 加快文件传输速度
- 便于文件和目录的管理与分发
- 减少备份时间和存储需求

### 2.2 常见压缩格式

Ubuntu系统支持多种压缩格式，常见的包括：
- .tar: 归档文件，通常与压缩算法结合使用
- .gz: 使用gzip算法压缩的文件
- .bz2: 使用bzip2算法压缩的文件
- .xz: 使用xz算法压缩的文件
- .zip: 一种广泛使用的压缩格式
- .7z: 高压缩率的压缩格式

### 2.3 常用压缩工具

Ubuntu 24.04默认提供了多种压缩解压工具，包括：
- tar: 归档工具，常与压缩算法结合使用
- gzip/gunzip: 使用gzip算法的压缩解压工具
- bzip2/bunzip2: 使用bzip2算法的压缩解压工具
- xz/unxz: 使用xz算法的压缩解压工具
- zip/unzip: 处理zip格式的工具

## 3. tar命令详解

### 3.1 tar命令概述

tar（Tape Archive）是Unix和Linux系统中最常用的归档工具，它可以将多个文件和目录打包成一个单独的归档文件，但本身并不进行压缩。通常，tar会与压缩工具（如gzip、bzip2、xz）结合使用，形成.tar.gz、.tar.bz2、.tar.xz等压缩归档格式。

### 3.2 tar命令基本语法

```bash
# 基本语法
 tar [选项] [归档文件] [文件/目录...]
```

### 3.3 tar常用选项

- **-c**: 创建新的归档文件
- **-x**: 从归档文件中提取文件
- **-t**: 列出归档文件的内容
- **-f**: 指定归档文件名，该选项必须放在所有选项的最后
- **-v**: 显示详细操作过程
- **-z**: 使用gzip压缩或解压
- **-j**: 使用bzip2压缩或解压
- **-J**: 使用xz压缩或解压
- **-C**: 指定解压目录
- **--exclude**: 排除指定文件或目录
- **--wildcards**: 支持通配符匹配

### 3.4 创建归档文件

#### 3.4.1 创建基本tar归档

```bash
# 创建包含file1.txt和dir1目录的归档
 tar -cvf archive.tar file1.txt dir1/
```

#### 3.4.2 创建gzip压缩归档(.tar.gz)

```bash
# 使用gzip压缩创建归档
 tar -czvf archive.tar.gz file1.txt dir1/
```

#### 3.4.3 创建bzip2压缩归档(.tar.bz2)

```bash
# 使用bzip2压缩创建归档
 tar -cjvf archive.tar.bz2 file1.txt dir1/
```

#### 3.4.4 创建xz压缩归档(.tar.xz)

```bash
# 使用xz压缩创建归档
 tar -cJvf archive.tar.xz file1.txt dir1/
```

### 3.5 提取归档文件

#### 3.5.1 提取基本tar归档

```bash
# 提取归档到当前目录
 tar -xvf archive.tar
```

#### 3.5.2 提取到指定目录

```bash
# 提取归档到指定目录/path/to/destination
 tar -xvf archive.tar -C /path/to/destination
```

#### 3.5.3 提取gzip压缩归档

```bash
# 提取.tar.gz归档
 tar -xzvf archive.tar.gz
```

#### 3.5.4 提取bzip2压缩归档

```bash
# 提取.tar.bz2归档
 tar -xjvf archive.tar.bz2
```

#### 3.5.5 提取xz压缩归档

```bash
# 提取.tar.xz归档
 tar -xJvf archive.tar.xz
```

### 3.6 查看归档内容

```bash
# 查看tar归档内容
 tar -tvf archive.tar

# 查看压缩归档内容
 tar -tzvf archive.tar.gz
 tar -tjvf archive.tar.bz2
 tar -tJvf archive.tar.xz
```

### 3.7 向现有归档添加文件

```bash
# 向tar归档添加新文件
 tar -rvf archive.tar newfile.txt
```

> 注意：无法向压缩归档（如.tar.gz、.tar.bz2）添加文件，需要先解压，添加文件后重新压缩

### 3.8 排除文件和目录

```bash
# 排除特定文件
 tar -czvf archive.tar.gz --exclude=*.log dir1/

# 排除多个文件或目录
 tar -czvf archive.tar.gz --exclude=file1.txt --exclude=dir2/ dir1/

# 使用通配符排除
 tar -czvf archive.tar.gz --wildcards --exclude=*~ dir1/
```

### 3.9 增量备份

```bash
# 创建增量备份（基于指定的快照文件）
 tar -czvf backup_incremental.tar.gz --listed-incremental=backup.snar dir1/
```

## 4. gzip命令详解

### 4.1 gzip概述

gzip是GNU压缩程序，使用DEFLATE算法（LZ77和哈夫曼编码的组合）来压缩文件。gzip通常比bzip2更快，但压缩率略低。

### 4.2 gzip基本语法

```bash
# 压缩文件
 gzip [选项] 文件名

# 解压文件
 gunzip [选项] 压缩文件名.gz
# 或
 gzip -d [选项] 压缩文件名.gz
```

### 4.3 gzip常用选项

- **-c**: 将压缩数据输出到标准输出，不改变原文件
- **-d**: 解压文件（与gunzip功能相同）
- **-f**: 强制压缩，即使目标文件已存在
- **-k**: 保留原文件（默认会删除原文件）
- **-l**: 列出压缩文件的压缩信息
- **-r**: 递归压缩目录中的所有文件
- **-t**: 测试压缩文件的完整性
- **-v**: 显示详细信息
- **-1至-9**: 设置压缩级别，1最快（压缩率最低），9最慢（压缩率最高），默认是6

### 4.4 gzip常用命令示例

#### 4.4.1 基本压缩

```bash
# 压缩file.txt，生成file.txt.gz并删除原文件
 gzip file.txt
```

#### 4.4.2 压缩并保留原文件

```bash
# 压缩file.txt，生成file.txt.gz并保留原文件
 gzip -k file.txt
```

#### 4.4.3 设置压缩级别

```bash
# 使用最高压缩级别（9）
 gzip -9 file.txt

# 使用最快压缩（1）
 gzip -1 file.txt
```

#### 4.4.4 解压文件

```bash
# 解压file.txt.gz
 gunzip file.txt.gz
# 或
 gzip -d file.txt.gz
```

#### 4.4.5 查看压缩信息

```bash
# 查看压缩文件信息
 gzip -l file.txt.gz
```

#### 4.4.6 递归压缩目录

```bash
# 递归压缩dir1目录中的所有文件
 gzip -r dir1/
```

## 5. bzip2命令详解

### 5.1 bzip2概述

bzip2是另一种流行的压缩工具，使用Burrows-Wheeler变换算法，通常比gzip提供更高的压缩率，但压缩和解压速度较慢。

### 5.2 bzip2基本语法

```bash
# 压缩文件
 bzip2 [选项] 文件名

# 解压文件
 bunzip2 [选项] 压缩文件名.bz2
# 或
 bzip2 -d [选项] 压缩文件名.bz2
```

### 5.3 bzip2常用选项

- **-c**: 将压缩数据输出到标准输出，不改变原文件
- **-d**: 解压文件（与bunzip2功能相同）
- **-f**: 强制压缩，即使目标文件已存在
- **-k**: 保留原文件（默认会删除原文件）
- **-s**: 小模式，使用更少的内存但速度更慢
- **-t**: 测试压缩文件的完整性
- **-v**: 显示详细信息
- **-1至-9**: 设置压缩级别，1最快（压缩率最低），9最慢（压缩率最高），默认是9

### 5.4 bzip2常用命令示例

#### 5.4.1 基本压缩

```bash
# 压缩file.txt，生成file.txt.bz2并删除原文件
 bzip2 file.txt
```

#### 5.4.2 压缩并保留原文件

```bash
# 压缩file.txt，生成file.txt.bz2并保留原文件
 bzip2 -k file.txt
```

#### 5.4.3 解压文件

```bash
# 解压file.txt.bz2
 bunzip2 file.txt.bz2
# 或
 bzip2 -d file.txt.bz2
```

## 6. xz命令详解

### 6.1 xz概述

xz是相对较新的压缩工具，使用LZMA2算法，通常提供比gzip和bzip2更高的压缩率，但压缩速度也更慢。在Ubuntu 24.04中，xz已成为许多软件包的默认压缩格式。

### 6.2 xz基本语法

```bash
# 压缩文件
 xz [选项] 文件名

# 解压文件
 unxz [选项] 压缩文件名.xz
# 或
 xz -d [选项] 压缩文件名.xz
```

### 6.3 xz常用选项

- **-c**: 将压缩数据输出到标准输出，不改变原文件
- **-d**: 解压文件（与unxz功能相同）
- **-f**: 强制压缩，即使目标文件已存在
- **-k**: 保留原文件（默认会删除原文件）
- **-t**: 测试压缩文件的完整性
- **-v**: 显示详细信息
- **-0至-9**: 设置压缩级别，0最快（压缩率最低），9最慢（压缩率最高），默认是6
- **--threads=N**: 指定使用的线程数，加速压缩

### 6.4 xz常用命令示例

#### 6.4.1 基本压缩

```bash
# 压缩file.txt，生成file.txt.xz并删除原文件
 xz file.txt
```

#### 6.4.2 多线程压缩

```bash
# 使用4个线程进行压缩
 xz -T4 file.txt
```

#### 6.4.3 解压文件

```bash
# 解压file.txt.xz
 unxz file.txt.xz
# 或
 xz -d file.txt.xz
```

## 7. zip和unzip命令详解

### 7.1 zip概述

zip是一种广泛使用的压缩格式，与tar不同，zip可以直接压缩多个文件和目录，而不需要先创建归档。zip格式在跨平台兼容性方面表现出色，可在Windows、macOS和Linux系统之间无缝使用。

### 7.2 zip和unzip安装

在Ubuntu 24.04中，zip和unzip可能未默认安装，可通过以下命令安装：

```bash
 sudo apt update
 sudo apt install zip unzip
```

### 7.3 zip基本语法

```bash
# 创建zip压缩文件
 zip [选项] 压缩文件名.zip 文件/目录...

# 解压zip文件
 unzip [选项] 压缩文件名.zip
```

### 7.4 zip常用选项

- **-r**: 递归压缩目录
- **-q**: 安静模式，不显示压缩过程
- **-v**: 详细模式，显示压缩过程和信息
- **-e**: 创建加密压缩文件
- **-P password**: 使用指定密码（不推荐，密码会显示在命令历史中）
- **-9**: 最高压缩级别
- **-1**: 最快压缩速度
- **-d**: 从压缩文件中删除文件
- **-u**: 更新压缩文件中的文件

### 7.5 unzip常用选项

- **-l**: 列出压缩文件内容
- **-v**: 详细列出压缩文件内容
- **-d 目录**: 指定解压目录
- **-q**: 安静模式
- **-o**: 覆盖现有文件而不提示
- **-x 文件**: 排除指定文件
- **-P password**: 使用密码解压

### 7.6 zip常用命令示例

#### 7.6.1 创建基本zip压缩文件

```bash
# 压缩file1.txt和file2.txt
 zip archive.zip file1.txt file2.txt
```

#### 7.6.2 压缩目录

```bash
# 递归压缩dir1目录
 zip -r archive.zip dir1/
```

#### 7.6.3 创建加密压缩文件

```bash
# 创建需要密码的压缩文件
 zip -e secure_archive.zip sensitive_file.txt
```

#### 7.6.4 更新压缩文件

```bash
# 更新压缩文件中的file1.txt
 zip -u archive.zip file1.txt
```

### 7.7 unzip常用命令示例

#### 7.7.1 基本解压

```bash
# 解压archive.zip到当前目录
 unzip archive.zip
```

#### 7.7.2 解压到指定目录

```bash
# 解压到指定目录/path/to/destination
 unzip archive.zip -d /path/to/destination

#### 7.7.3 列出压缩文件内容

```bash
# 列出压缩文件内容
 unzip -l archive.zip
```

#### 7.7.4 解压加密压缩文件

```bash
# 解压加密的zip文件
 unzip -P password secure_archive.zip
```

## 8. 7z命令详解

### 8.1 7z概述

7z（7-Zip）是一款开源的压缩工具，使用LZMA和LZMA2算法，提供极高的压缩率和强大的加密功能。7z格式在压缩率上通常优于传统的zip和tar.gz格式，同时支持多种压缩算法和格式<mcreference link="https://my.oschina.net/emacs_8875372/blog/17484547" index="5"></mcreference>。

### 8.2 7z安装

在Ubuntu 24.04中，7z工具并未默认安装，需要通过以下命令安装：

```bash
# 更新软件包列表
 sudo apt update
# 安装p7zip-full（包含7z和7za命令）
 sudo apt install p7zip-full
```

> 注：p7zip-full包含完整功能的7z工具，而p7zip只包含基础功能<mcreference link="https://blog.csdn.net/weixin_30702887/article/details/94850484" index="1"></mcreference>。

### 8.3 7z基本语法

```bash
# 压缩文件/目录
 7z [命令] [选项] 压缩文件名 源文件/目录

# 解压文件
 7z [命令] [选项] 压缩文件名
```

### 8.4 7z常用命令

- **a**: 添加文件到压缩包
- **x**: 解压文件（保持目录结构）
- **e**: 解压文件（不保持目录结构）
- **l**: 列出压缩包内容
- **t**: 测试压缩包完整性
- **d**: 从压缩包中删除文件

### 8.5 7z常用选项

- **-t**: 指定压缩格式，如7z、zip、gzip等
- **-r**: 递归处理子目录
- **-mx**: 设置压缩级别（0-9），0表示存储，9表示最高压缩率
- **-m0**: 指定压缩方法，如LZMA、LZMA2、PPMd等
- **-md**: 设置字典大小，如128m表示128MB
- **-p**: 设置密码
- **-o**: 指定解压目录（注意-o后无空格）
- **-v**: 创建分卷压缩

### 8.6 7z常用命令示例

#### 8.6.1 创建7z压缩文件

```bash
# 创建基本7z压缩文件
 7z a archive.7z file1.txt dir1/

# 创建最高压缩率的7z文件
 7z a -mx=9 archive.7z file1.txt dir1/

# 指定LZMA2算法和32MB字典大小
 7z a -t7z -m0=LZMA2 -md=32m archive.7z file1.txt
```

#### 8.6.2 解压7z文件

```bash
# 解压到当前目录（保持结构）
 7z x archive.7z

# 解压到指定目录
 7z x archive.7z -o/path/to/destination

# 解压加密的7z文件
 7z x -ppassword archive.7z
```

#### 8.6.3 创建分卷压缩

```bash
# 创建每个分卷为100MB的压缩包
 7z a -v100m archive.7z large_file.iso
```

#### 8.6.4 列出压缩包内容

```bash
# 列出7z压缩包内容
 7z l archive.7z
```

## 9. 压缩算法原理深度剖析

### 9.1 DEFLATE算法（gzip, zip）

DEFLATE算法是gzip和zip格式的基础，它结合了LZ77算法和哈夫曼编码：

1. **LZ77算法**：通过滑动窗口查找重复数据序列，用(距离,长度)对替换重复序列<mcreference link="https://blog.csdn.net/ljx333/article/details/77447592" index="2"></mcreference>
2. **哈夫曼编码**：对LZ77输出的符号进行熵编码，进一步减小数据体积

DEFLATE的优势在于平衡了压缩率和速度，成为Web传输的标准压缩算法<mcreference link="https://juejin.cn/post/7350501693090283531" index="4"></mcreference>。

### 9.2 Burrows-Wheeler变换（bzip2）

bzip2使用Burrows-Wheeler变换(BWT)算法，工作原理包括：
1. 将数据重排，使相似字符聚集
2. 应用Move-to-Front编码
3. 最后使用哈夫曼编码

bzip2通常比gzip提供更高的压缩率，但速度较慢<mcreference link="https://blog.csdn.net/waverider2012/article/details/8514949" index="1"></mcreference>。

### 9.3 LZMA/LZMA2算法（xz, 7z）

LZMA（Lempel-Ziv-Markov chain-Algorithm）是7z和xz格式的核心算法：
1. 使用改进的LZ77算法，支持更大的滑动窗口
2. 采用范围编码而非哈夫曼编码
3. LZMA2是LZMA的改进版，支持多线程压缩

LZMA提供极高的压缩率，但压缩时间较长，内存占用较高<mcreference link="https://blog.csdn.net/weixin_33483080/article/details/116587380" index="2"></mcreference>。

## 10. 压缩算法性能对比

不同压缩算法在压缩率、速度和资源占用方面有显著差异，以下是主要算法的对比：

### 10.1 压缩率对比（由高到低）

1. LZMA2/xz > BZIP2 > DEFLATE(gzip) > LZ4

### 10.2 压缩速度对比（由快到慢）

1. LZ4 > DEFLATE(gzip) > BZIP2 > LZMA2/xz

### 10.3 解压速度对比（由快到慢）

1. LZ4 > DEFLATE(gzip) > LZMA2/xz > BZIP2

### 10.4 内存占用对比（由高到低）

1. LZMA2/xz > BZIP2 > DEFLATE(gzip) > LZ4

### 10.5 综合性能表格

| 算法 | 压缩率 | 压缩速度 | 解压速度 | 内存占用 | 适用场景 |
|------|--------|----------|----------|----------|----------|
| gzip | 中等 | 快 | 快 | 低 | 日常文件压缩、Web传输 |
| bzip2 | 高 | 中 | 慢 | 中 | 归档存储、备份 |
| xz | 最高 | 慢 | 中 | 高 | 长期归档、软件包 |
| 7z | 最高 | 慢 | 中 | 高 | 高压缩需求、加密归档 |
| zip | 中 | 快 | 快 | 低 | 跨平台文件交换 |

> 数据来源：综合多个测试结果，实际性能可能因文件类型和内容有所不同<mcreference link="https://blog.csdn.net/waverider2012/article/details/8514949" index="1"></mcreference><mcreference link="https://blog.csdn.net/weixin_39593469/article/details/110240911" index="5"></mcreference>。

## 11. 高级压缩技巧

### 11.1 分卷压缩

当需要将大文件分割成多个小文件时，可以使用分卷压缩：

#### 11.1.1 使用tar和split分卷

```bash
# 创建每个分卷为1GB的tar.gz压缩包
 tar -czvf - large_dir/ | split -b 1G - large_dir.tar.gz.

# 解压分卷压缩包
 cat large_dir.tar.gz.* | tar -xzvf -
```

#### 11.1.2 使用7z分卷

```bash
# 创建每个分卷为500MB的7z压缩包
 7z a -v500m archive.7z large_file/

# 解压7z分卷（只需指定第一个分卷）
 7z x archive.7z.001
```

#### 11.1.3 使用rar分卷（需安装rar）

```bash
# 安装rar
 sudo apt install rar
# 创建每个分卷为50MB的rar压缩包
 rar a -v50m archive.rar large_file/
```

### 11.2 加密压缩

保护敏感数据，可使用加密压缩：

#### 11.2.1 zip加密

```bash
# 创建加密zip压缩包
 zip -e secure.zip sensitive_files/
```

#### 11.2.2 7z加密

```bash
# 创建加密7z压缩包（包含文件名加密）
 7z a -pPassword -mhe=on secure.7z sensitive_files/
```

> 注意：-mhe选项启用文件头加密，防止他人查看压缩包内文件列表<mcreference link="https://my.oschina.net/emacs_8888227/blog/17514359" index="5"></mcreference>。

### 11.3 多线程压缩

利用多核CPU加速压缩：

#### 11.3.1 xz多线程

```bash
# 使用4线程压缩
 xz -T4 large_file
```

#### 11.3.2 7z多线程

```bash
# 使用4线程压缩
 7z a -mx=9 -mmt=4 archive.7z large_dir/
```

### 11.4 压缩级别调优

不同压缩级别平衡压缩率和速度：

```bash
# gzip快速压缩（级别1）
 gzip -1 file.txt

# gzip最高压缩率（级别9）
 gzip -9 file.txt

# xz最高压缩率（级别9）
 xz -9 file.txt

# 7z最高压缩率（级别9）
 7z a -mx=9 archive.7z file.txt
```

## 12. 不同文件类型的压缩策略

### 12.1 文本文件

- **特点**：高度可压缩，重复内容多
- **推荐算法**：xz或7z（最高压缩率）
- **示例**：日志文件、代码文件、文档

### 12.2 二进制文件

- **特点**：压缩率中等，取决于数据结构
- **推荐算法**：gzip或xz
- **示例**：可执行文件、库文件

### 12.3 媒体文件

- **特点**：已压缩格式（JPEG、MP3、MP4等）
- **推荐策略**：仅打包不压缩（使用tar -cf）
- **示例**：图片、音频、视频文件

> 注意：对已压缩的媒体文件进行二次压缩通常效果甚微，甚至可能增大文件大小<mcreference link="https://blog.csdn.net/axutongxue/article/details/118614579" index="3"></mcreference>。

### 12.4 数据库文件

- **特点**：大型文件，结构化数据
- **推荐算法**：lz4（速度优先）或xz（压缩率优先）
- **最佳实践**：先导出为文本格式（如SQL）再压缩

## 13. 常见问题与解决方案

### 13.1 压缩文件损坏

**症状**：解压时提示CRC错误或文件损坏
**解决方案**：
1. 使用测试命令验证：`tar -tvf archive.tar.gz` 或 `7z t archive.7z`
2. 尝试修复：`7z r archive.7z`
3. 对于tar文件，使用`tar --ignore-zeros -xvf archive.tar`跳过损坏块

### 13.2 解压中文乱码

**症状**：解压含中文文件名的zip文件出现乱码
**解决方案**：
```bash
# 使用unar工具（支持编码转换）
 sudo apt install unar
 unar -e GBK archive.zip
```

### 13.3 压缩速度过慢

**症状**：大型目录压缩耗时过长
**解决方案**：
1. 降低压缩级别：`tar -czf -1 archive.tar.gz dir`
2. 改用更快的算法：`tar -cf - dir | lz4 - archive.tar.lz4`
3. 使用多线程压缩：`xz -T0 file`（-T0表示使用所有CPU核心）

### 13.4 内存占用过高

**症状**：压缩大文件时系统卡顿或OOM
**解决方案**：
1. 减小字典大小：`7z a -md=8m archive.7z large_file`
2. 使用低内存算法：gzip替代xz/7z
3. 增加虚拟内存（swap）

## 14. 经验总结与最佳实践

### 14.1 日常使用推荐

- **快速备份**：`tar -czf backup.tar.gz dir`（平衡速度和压缩率）
- **最大压缩**：`7z a -mx=9 -m0=LZMA2 -md=64m archive.7z dir`
- **跨平台传输**：`zip -r archive.zip dir`（Windows兼容性好）
- **大文件归档**：`tar -cJf archive.tar.xz dir`（高压缩率，适合长期存储）

### 14.2 服务器环境优化

- **日志压缩**：`logrotate`配合gzip自动压缩日志
- **备份策略**：重要数据使用xz压缩，临时备份使用lz4
- **传输优化**：scp传输前用gzip压缩，通过管道直接传输：`tar -czf - dir | ssh user@host "tar -xzf - -C /path"`

### 14.3 常见陷阱规避

1. **不要压缩已压缩文件**：对图片、视频等媒体文件仅打包不压缩
2. **注意压缩级别**：最高压缩级别通常收益有限，但耗时显著增加
3. **分卷大小选择**：根据存储介质（如CD/DVD容量）选择合适的分卷大小
4. **加密安全性**：使用强密码并启用文件头加密保护敏感数据
5. **测试压缩包**：创建重要压缩包后使用测试命令验证完整性

### 14.4 性能优化建议

- **选择合适算法**：根据需求在压缩率和速度间权衡
- **利用多线程**：多核CPU环境下启用多线程压缩
- **合理设置字典大小**：大字典提高压缩率但增加内存占用
- **增量备份**：使用`tar --listed-incremental`创建增量备份减少重复工作

## 15. 总结

Ubuntu 24.04提供了丰富的压缩解压工具，从基础的tar、gzip到高级的7z、xz，每种工具都有其适用场景。选择合适的工具和参数可以显著提高工作效率，节省存储空间和网络带宽。

掌握压缩解压技术不仅是日常使用的基本技能，也是系统管理和数据处理的重要基础。通过理解不同压缩算法的原理和特性，我们可以针对不同场景做出最优选择，平衡压缩率、速度和资源占用。
