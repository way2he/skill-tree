# Windows使用助手 - 项目规则

## 1. 项目规范

### 1.1 文档规范
- 所有文档使用简体中文编写
- 使用Markdown格式
- 保持文档结构清晰，使用适当的标题层级
- 代码块需指定语言类型
- 重要信息使用加粗或高亮标记

### 1.2 命名规范
- 文件名使用小写字母，单词间用下划线连接
- 目录名使用小写字母，单词间用下划线连接
- 配置文件使用`.md`或`.yaml`格式
- 代码文件根据具体语言使用对应扩展名

### 1.3 版本控制
- 使用Git进行版本控制
- 提交信息需清晰描述更改内容
- 重要功能开发使用独立分支
- 定期同步主分支更新
